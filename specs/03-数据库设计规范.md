# 数据库设计规范

> 版本：v1.0
> 日期：2026-06-11
> 归属：摸鱼圈开发规范
> 技术栈：SQLite 3 + Prisma 5

---

## 1. 总体原则

### 1.1 技术选型

| 维度 | 选型 | 说明 |
|------|------|------|
| **数据库** | SQLite 3 | 零配置、文件存储、无需安装 |
| **ORM** | Prisma 5 | 类型安全、自动迁移、Schema-first |
| **存储位置** | `server/prisma/dev.db` | 开发环境本地文件 |

### 1.2 设计原则

- **Schema-first**：所有数据模型在 `schema.prisma` 中定义，通过 `npx prisma migrate dev` 生成数据库
- **不做过度设计**：SQLite 适合轻量场景，不建视图、存储过程、触发器
- **字符串主键**：使用 `cuid()` 或 `uuid()` 作为主键，兼容分布式场景
- **逻辑删除**：关键业务表使用 `deletedAt` 软删除，非关键表直接物理删除
- **时间字段**：统一使用 `DateTime`，由 Prisma 管理 `createdAt` / `updatedAt`

## 2. 命名规范

### 2.1 模型命名（Prisma Model）

| 规则 | 说明 | 示例 |
|------|------|------|
| PascalCase | 模型名使用大驼峰 | `User`, `Circle`, `UnoCard` |
| 单数名词 | 使用单数形式 | `User` 而非 `Users` |
| 语义清晰 | 表达业务含义 | `MoyuStat` 而非 `Stat` |

### 2.2 字段命名

| 规则 | 说明 | 示例 |
|------|------|------|
| camelCase | 字段名使用小驼峰 | `userName`, `createdAt` |
| 外键字段 | `{关联模型}Id` 格式 | `creatorId`, `circleId` |
| 布尔字段 | `is` / `has` 前缀 | `isBanned`, `isPrivate` |

### 2.3 数据库表名（SQLite）

Prisma 默认使用模型名作为表名。如需自定义，在 `@@map` 中指定：

```prisma
model User {
  id String @id @default(cuid())
  // ...

  @@map("users")
}
```

> 本项目使用 `@@map` 将 Prisma 模型映射为小写复数表名，符合 SQLite 惯例。

## 3. 数据模型设计

### 3.1 用户表 User

```prisma
model User {
  id              String   @id @default(cuid())
  email           String   @unique
  password        String                       // bcrypt 加密后的密码
  userName        String                       // 摸鱼昵称
  avatar          String   @default("moyu_otter") // 头像ID
  joinedCircleId  String?                      // 当前加入的鱼圈ID
  privateCircleId String   @unique             // 私有鱼圈ID（不可退出）
  salary          Int      @default(250)       // 日薪（元）
  workStart       String   @default("09:00")   // 上班时间 HH:mm
  workEnd         String   @default("18:00")   // 下班时间 HH:mm
  isBanned        Boolean  @default(false)     // 是否被封禁
  role            String   @default("USER")    // USER | ADMIN
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt

  // 关联
  ownedCircles    Circle[] @relation("CircleOwner")
  memberOfCircles CircleMember[]
  moyuStats       MoyuStat[]
  unoCards        UserCard[]

  @@map("users")
}
```

### 3.2 鱼圈表 Circle

```prisma
model Circle {
  id            String   @id @default(cuid())
  name          String                       // 鱼圈名称（最大50字符）
  inviteCode    String   @unique             // 6位数字邀请码
  isPrivate     Boolean  @default(false)     // 是否为私有鱼圈
  creatorId     String                       // 创建者ID
  ownerId       String                       // 当前群主ID
  memberCount   Int      @default(1)         // 当前成员数
  // 宠物鱼数据（鱼圈共享）
  petFishName   String   @default("懵懂胖金鱼")
  petFishLevel  Int      @default(1)
  petFishExp    Int      @default(0)
  petFishType   String   @default("肥嘟嘟胖金鱼")
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
  deletedAt     DateTime?                    // 软删除时间

  // 关联
  creator       User     @relation("CircleOwner", fields: [creatorId], references: [id])
  members       CircleMember[]
  messages      ChatMessage[]

  @@map("circles")
}
```

### 3.3 鱼圈成员关系表 CircleMember

```prisma
model CircleMember {
  id         String   @id @default(cuid())
  circleId   String
  userId     String
  role       String   @default("MEMBER")     // OWNER | MEMBER
  joinedAt   DateTime @default(now())
  createdAt  DateTime @default(now())

  // 关联
  circle     Circle   @relation(fields: [circleId], references: [id])
  user       User     @relation(fields: [userId], references: [id])

  @@unique([circleId, userId])               // 一个用户在一个鱼圈中只能有一条记录
  @@map("circle_members")
}
```

### 3.4 聊天消息表 ChatMessage

> **重要**：消息会在 5 分钟后从数据库物理删除。这是核心业务规则，不是bug。

```prisma
model ChatMessage {
  id           String   @id @default(cuid())
  circleId     String
  authorId     String
  authorName   String                       // 冗余存储，避免 JOIN 查询
  authorAvatar String                       // 冗余存储，避免 JOIN 查询
  text         String                       // 消息内容（最大500字符）
  createdAt    DateTime @default(now())

  // 关联
  circle       Circle   @relation(fields: [circleId], references: [id])

  @@index([circleId, createdAt])            // 按鱼圈+时间查询
  @@map("chat_messages")
}
```

### 3.5 UNO 卡片定义表 UnoCard

```prisma
model UnoCard {
```

### 3.6 用户卡片持有表 UserCard

```prisma
model UserCard {
  id         String   @id @default(cuid())
  userId     String
  cardId     String                           // 关联 UnoCard
  count      Int      @default(1)             // 持有数量
  createdAt  DateTime @default(now())
  updatedAt  DateTime @updatedAt

  // 关联
  user       User     @relation(fields: [userId], references: [id])
  card       UnoCard  @relation(fields: [cardId], references: [id])

  @@map("user_cards")

```

### 3.7 摸鱼统计表 MoyuStat

```prisma
model MoyuStat {
  id            String   @id @default(cuid())
  userId        String
  circleId      String
  todayCount    Int      @default(0)          // 今日摸鱼次数
  totalCount    Int      @default(0)          // 历史总摸鱼次数

  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  // 关联
  user          User     @relation(fields: [userId], references: [id])

  @@unique([userId, circleId])                // 每个用户在每个鱼圈中有一条统计
  @@map("moyu_stats")
}
```

### 3.8 管理员表 AdminUser

```prisma
model AdminUser {
  id            String   @id @default(cuid())
  username      String   @unique              // 管理员账号

  nickname      String?                      // 昵称
  role          String   @default("ADMIN")   // SUPER_ADMIN | ADMIN

  @@map("admin_users")

```

### 3.9 系统配置表 SystemConfig

```prisma
model SystemConfig {
  id            String   @id @default(cuid())
```

## 5. SQLite 特定注意事项

### 5.1 SQLite 与 MySQL/PostgreSQL 的差异

| 特性 | SQLite | MySQL/PostgreSQL | 应对策略 |
|------|--------|------------------|---------|
| 无 `AUTO_INCREMENT` | 使用 `@default(cuid())` | `AUTO_INCREMENT` | 已使用 cuid 主键 |
| 无 `ENUM` 类型 | 使用 `String` + 应用层校验 | `ENUM(...)` | 用 `as const` 对象定义枚举 |
| 无 `JSON` 原生查询 | 序列化为字符串 | `JSON` 类型 | 配置值用 JSON 字符串存储 |
| 无 `TIMESTAMP` | 使用 `DateTime` | `TIMESTAMP` | Prisma 统一处理 |
| 无全文搜索 | 不支持 `FULLTEXT` | 支持 | 消息搜索不在范围内（无历史记录） |
| 并发写入限制 | 同一时刻只有一个写入者 | 支持并发写入 | 写操作加排队/重试逻辑 |
| 大小写敏感 | LIKE 默认不敏感 | 视 collation | 邀请码查询注意大小写 |

### 5.2 性能优化

```typescript
// 使用 Prisma 事务处理多个写操作
await prisma.$transaction([
  prisma.circleMember.delete({ where: { id: memberId } }),
  prisma.circle.update({
    where: { id: circleId },
    data: { memberCount: { decrement: 1 } },
  }),
]);
```

### 5.3 备份策略

```bash
# SQLite 备份：直接复制文件
cp server/prisma/dev.db server/prisma/dev.db.bak

# 生产环境定时备份
# cron: 0 2 * * * cp /path/to/prod.db /backup/prod_$(date +%Y%m%d).db
```

## 6. 索引设计

### 6.1 Prisma 索引定义

```prisma
model ChatMessage {
  // ...

  @@index([circleId, createdAt])     // 按鱼圈查询 + 时间排序
  @@index([createdAt])               // 按时间清理过期消息
}

model CircleMember {
  // ...

  @@unique([circleId, userId])       // 防止重复加入
  @@index([userId])                  // 查询用户加入的所有鱼圈
}

model UserCard {
  // ...

  @@unique([userId, cardId])         // 防止重复记录
  @@index([userId])                  // 查询用户所有卡片
}

model MoyuStat {
  // ...

  @@unique([userId, circleId])       // 每用户每鱼圈一条
  @@index([circleId])                // 排行榜按鱼圈查询
}
```

### 6.2 索引设计原则

| 原则 | 说明 |
|------|------|
| **高选择性字段优先** | `userId`、`circleId` 等外键字段 |
| **联合索引遵循最左前缀** | `[circleId, createdAt]` 支持按 circleId 查询或按 circleId + 时间范围查询 |
| **避免过度索引** | SQLite 写入性能受索引数量影响，单表索引不超过 5 个 |
| **唯一约束即索引** | `@unique` 自动创建唯一索引 |

## 7. 查询规范

### 7.1 必须使用 Prisma Client

```typescript
// ✅ 正确：使用 Prisma Client
const user = await prisma.user.findUnique({
  where: { email },
  include: { memberOfCircles: true },
});

// ❌ 禁止：使用 $queryRaw 拼接用户输入
const user = await prisma.$queryRaw`
  SELECT * FROM users WHERE email = ${email}
`;

// ⚠️ 允许：$queryRaw 仅在 Prisma 无法表达的复杂查询时使用，且不得拼接用户输入
const stats = await prisma.$queryRaw`
  SELECT circleId, COUNT(*) as count, SUM(todayCount) as total
  FROM moyu_stats
  GROUP BY circleId
`;
```

### 7.2 常用查询模式

```typescript
// 分页查询（使用 skip + take）
const users = await prisma.user.findMany({
  where: { isBanned: false },
  skip: (page - 1) * pageSize,
  take: pageSize,
  orderBy: { createdAt: 'desc' },
});

// 关联查询（include）
const circle = await prisma.circle.findUnique({
  where: { id: circleId },
  include: {
    members: {
      include: { user: true },
      take: 10,
    },
  },
});

// 计数查询
const total = await prisma.user.count({
  where: { isBanned: false },
});

// 聚合查询（排行榜）
const leaderboard = await prisma.moyuStat.groupBy({
  by: ['userId'],
  where: { circleId },
  _sum: { todayCount: true, totalCount: true },
  orderBy: { todayCount: 'desc' },
  take: 10,
});
```

### 7.3 禁止的查询

| 禁止操作 | 原因 | 替代方案 |
|----------|------|---------|
| `$queryRawUnsafe()` | SQL 注入风险 | 使用 `$queryRaw` 模板字面量 |
| 不带 `where` 的 `deleteMany` | 误删全表数据 | 永远带 where 条件 |
| 不带 `where` 的 `updateMany` | 误更新全表数据 | 永远带 where 条件 |
| 嵌套超过 3 层的 `include` | 性能和可读性差 | 拆分为多次查询 |

## 9. 业务规则与数据约束

### 9.1 业务约束（应用层实现）

| 约束 | 实现方式 | 说明 |
|------|---------|------|
| 邮箱唯一 | `@unique` | Prisma Schema 约束 |
| 鱼圈最多 10 人 | Service 层检查 | `circle.memberCount < 10` 才允许加入 |
| 私有鱼圈不可退出 | Service 层检查 | `circle.isPrivate === true` 时拒绝退出 |
| 消息 5 分钟销毁 | 定时器 + 物理删除 | `DELETE FROM chat_messages WHERE createdAt < datetime('now', '-5 minutes')` |
| 每日摸鱼次数上限 | 哈希确定性 + Service 层检查 | 基于 userId 哈希值生成 15~45 的上限 |
| 邀请码 6 位纯数字 | Service 层生成 | `String(Math.floor(100000 + Math.random() * 900000))` |
| 卡片可重复获得 | `count` 字段累加 | 已有卡片则 `count += 1` |

### 9.2 联合唯一约束

```prisma
// 防止重复加入鱼圈
@@unique([circleId, userId])    // CircleMember

// 防止重复记录卡片持有
@@unique([userId, cardId])       // UserCard

// 每用户每鱼圈一条摸鱼统计
@@unique([userId, circleId])     // MoyuStat
```
