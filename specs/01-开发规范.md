> **适用范围**: 全栈开发（client/ + server/）
> **维护者**: 独立开发者 / 小团队
> **最后更新**: 2026-06-11

## 1. 语言与运行时要求

| 项目 | 要求 | 说明 |
|------|------|------|
| **语言** | TypeScript 5.8+ | 前后端统一，`strict: true` |
| **运行时** | Node.js >= 18 | 后端运行环境 |
| **包管理器** | npm | 已有 `package-lock.json`，禁止使用 yarn/pnpm |
| **模块系统** | ESM | 前端 Vite 原生 ESM，后端 tsx 运行时 |

### tsconfig.json 关键配置

```json
{
  "compilerOptions": {
    "strict": true,
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noFallthroughCasesInSwitch": true
  }
}
```

> **红线**: `strict` 必须为 `true`，不得关闭任何严格检查选项。

## 3. 命名约定

### 3.1 文件命名

| 类型 | 规则 | 示例 |
|------|------|------|
| React 组件文件 | PascalCase + `.tsx` | `ChatRoom.tsx`, `MoyuGame.tsx` |
| React 组件样式文件 | 禁止（使用 Tailwind） | — |
| 非组件 TS 文件 | camelCase + `.ts` | `api.ts`, `avatars.ts`, `authUtils.ts` |
| 目录名 | kebab-case | `components/`, `salary-tracker/` |
| 测试文件 | 与源文件同名 + `.test.ts` | `authUtils.test.ts` |
| 配置文件 | 小写 + 点分隔 | `vite.config.ts`, `.eslintrc.json` |

### 3.2 代码命名

| 类型 | 规则 | 示例 |
|------|------|------|
| **组件** | PascalCase | `ChatRoom`, `SalaryCard`, `MoyuButton` |
| **函数/方法** | camelCase | `sendMessage`, `fetchUserCircles` |
| **常量** | SCREAMING_SNAKE_CASE | `MAX_MESSAGE_LENGTH`, `CARD_DROP_RATE` |
| **类型/接口** | PascalCase | `User`, `Circle`, `UnoCard`, `ChatMessage` |
| **布尔变量** | `is` / `has` 前缀 | `isLoading`, `hasPermission`, `isLoggedIn` |
| **事件处理** | `handle` + 事件名 | `handleClick`, `handleSubmit`, `handleEscPress` |
| **回调 Props** | `on` + 事件名 | `onSubmit`, `onClose`, `onCardDrop` |
| **枚举** | PascalCase，成员 SCREAMING_SNAKE | `UserRole.ADMIN` |

### 3.3 路由命名

| 规则 | 示例 |
|------|------|
| kebab-case | `/chat-room`, `/salary-tracker`, `/circle-manage` |
| 动态参数用冒号 | `/circle/:circleId`, `/user/:userId` |
| 嵌套路由按模块 | `/admin/users`, `/admin/circles` |

## 4. TypeScript 特定规范

### 4.1 `import type` 使用场景

当导入内容仅用于类型标注（不产生运行时代码）时，必须使用 `import type`。

```typescript
// ✅ 正确：仅类型引用
import type { User, Circle } from '../types';
import type { RequestHandler } from 'express';

// ❌ 错误：混用运行时和纯类型导入
import { User, fetchUser } from '../types'; // User 仅用于类型标注时应分开导入
```

### 4.2 `interface` vs `type` 使用场景

| 场景 | 推荐 | 示例 |
|------|------|------|
| 对象结构定义 | `interface` | `interface User { id: string; name: string; }` |
| 联合类型 | `type` | `type CardColor = 'red' \| 'green' \| 'blue' \| 'yellow';` |
| 函数签名 | `type` | `type MessageHandler = (msg: ChatMessage) => void;` |
| Props 定义 | `interface` | `interface ChatRoomProps { circleId: string; }` |
| 工具类型 | `type` | `type UserPreview = Pick<User, 'id' \| 'name' \| 'avatar'>;` |
| 可扩展的类实现 | `interface` | `interface CircleMember extends User { role: MemberRole; }` |

### 4.3 禁止的写法及替代方案

| 禁止 | 替代方案 |
|------|---------|
| `any` | `unknown` + 类型守卫 |
| 非空断言 `!` | 可选链 `?.` + 空值合并 `??`，或显式类型守卫 |
| `as` 类型断言 | 类
