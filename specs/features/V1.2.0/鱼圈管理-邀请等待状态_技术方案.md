# 鱼圈管理-邀请等待状态 技术方案

## 1. 设计概要

**功能描述**：在侧边栏展示"等待中"的鱼圈，包含过期倒计时、已邀请人数、状态标签，支持点击打开邀请弹窗

**现有基础**：Circle.isActive、Invite 表、创建鱼圈+邀请码生成、加入+自动激活、过期清理 均已实现

**本次新增**：
- `GET /api/circles/pending` — 获取当前用户的"等待中"鱼圈列表
- 创建鱼圈时校验3个上限
- 前端 `SidebarPending.tsx` 组件
- `CircleSidebar.tsx` 集成展示

---

## 2. 数据库设计

**无需修改**，现有表已满足需求：

- `Circle.isActive` (Boolean, default false) — 区分等待中/已激活
- `Invite` 表 — 存储邀请码、过期时间、状态
- `UserCircle` 表 — 关联用户和鱼圈

---

## 3. API 设计

### `GET /api/circles/pending`

**描述**：获取当前用户的"等待中"鱼圈列表

**鉴权**：需要JWT

**处理逻辑**：
1. 查询当前用户创建的 `isActive=false` 的鱼圈
2. 关联查询 Invite 表获取邀请码和过期时间（取 status=active 的最新邀请）
3. 按创建时间倒序排列

**Response**：
```json
{
  "success": true,
  "data": {
    "pendingCircles": [
      {
        "circleId": "uuid",
        "circleName": "第五工位躺平分会",
        "circleIcon": "🐟",
        "inviteCode": "123456",
        "expiresAt": "2026-06-22T12:00:00.000Z",
        "memberCount": 1,
        "createdAt": "2026-06-22T11:00:00.000Z"
      }
    ]
  }
}
```

### `POST /api/circles` — 修改

**新增校验**：创建前检查当前用户"等待中"鱼圈数量 < 3

**新增异常响应**：
| 场景 | 状态码 | message |
|------|--------|---------|
| 达到3个上限 | 400 | "你已经有3个等待中的鱼圈了，等它们过期或有人加入后再创建吧！" |

## 4. 核心逻辑

### 4.1 获取等待中鱼圈

```
1. 查询 UserCircle WHERE userId = currentUser
2. 关联 Circle WHERE Circle.isActive = false AND Circle.ownerId = currentUser
3. 关联 Invite WHERE Invite.circleId = Circle.id AND Invite.status = 'active'
4. 按 Circle.createdAt DESC 排列
5. 返回列表
```

### 4.2 创建鱼圈数量校验

```
1. 在创建鱼圈前，查询 COUNT(Circle WHERE ownerId = userId AND isActive = false)
2. 如果 count >= 3，返回错误
3. 否则继续创建流程
```

## 7. 安全与性能

**性能**：
- `GET /api/circles/pending` 查询使用索引（ownerId, isActive）
- 前端30秒轮询，不频繁请求
- 倒计时使用前端 setInterval，不依赖服务器

**安全**：
- 仅返回当前用户创建的等待中鱼圈
- 3个上限防止滥用
