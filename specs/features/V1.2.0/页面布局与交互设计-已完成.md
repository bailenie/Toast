# 页面布局与交互设计 PRD (V1.1.0)

## 1. 背景与目标

### 为什么做

V1.0.0版本的页面布局存在以下问题：
- 功能层级混乱：窝囊费、蛐蛐蛐、摸鱼鱼在导航栏是平级的，但蛐蛐蛐和摸鱼鱼是鱼圈功能，窝囊费是个人功能
- 鱼圈上下文缺失：进入摸鱼鱼页面后，看不到当前是哪个鱼圈的宠物鱼、排行榜
- 功能归属不明确：宠物鱼是鱼圈共有的，但显示在独立的"摸鱼鱼"页面，用户可能误以为是个人功能
- 切换鱼圈不便捷：需要回到蛐蛐蛐页面才能切换鱼圈

### 用户价值

- 左侧常驻鱼圈栏让用户一眼看到所有已加入的鱼圈
- 点击直接切换聊天，操作更便捷
- 功能归属清晰：所有鱼圈功能都在鱼圈上下文中展示
- 窝囊费作为个人功能独立存在，不与鱼圈功能混淆

### 成功标准

- 用户能清晰区分个人功能和鱼圈功能
- 鱼圈切换操作步骤从3步减少到1步
- 用户能直观感知当前在哪个鱼圈的哪个功能中

---

## 2. 范围定义

### 本次做

- 重新定义导航结构，区分个人功能和鱼圈功能
- 新增左侧常驻鱼圈栏（200px宽）
- 切换鱼圈后默认进入聊天室（蛐蛐间）
- 底部Tab栏切换到摸鱼鱼
- 摸鱼鱼页面左右布局，直接展示所有功能
- 窝囊费放在左侧栏底部（个人功能）
- 窝囊费页面独立布局：隐藏左侧鱼圈栏，新增返回按钮（点击返回首页 /home）

### 本次不做

- 鱼圈名称修改
- 鱼圈公告功能
- 鱼圈头像/封面自定义
- 成员权限管理（除群主外）
- 鱼圈搜索/发现
- 鱼圈二维码邀请

## 3. 用户与场景

| 角色 | 说明 |
| ---- | ---- |
| 普通成员 | 加入鱼圈，使用聊天、摸鱼等功能 |

| 新用户 | 刚注册，需要创建或加入鱼圈 |

| 编号 | 用户故事 |

### 4.2 左侧常驻鱼圈栏

左侧常驻鱼圈栏是用户查看和切换鱼圈的主要入口，解决入口发现性和多鱼圈导航问题。

```
左侧鱼圈栏 (200px)
│
├── ① 操作区 (顶部)
│   └── [+] 按钮 (点击展开创建/加入鱼圈表单)
│
├── ② 鱼圈列表 (滚动)
│   ├── 🐟 技术部摸鱼会 (当前高亮)
│   │   ├── 图标 (系统默认图标)
│   │   ├── 名称: "技术部摸鱼会"
│   │   └── 未读消息数角标 (右上角红色数字)
│   │
│   ├── 🐠 市场部早退党
│   │   ├── 图标 (系统默认图标)
│   │   ├── 名称: "市场部早退党"
│   │   └── 无未读消息
│   │
│   └── 🐟 第五工位躺平分会
│       ├── 图标 (系统默认图标)
│       ├── 名称: "第五工位躺平分会"
│       └── 未读消息数角标 (右上角红色数字)
│
├── ────────── 分隔线
│
└── ③ 个人功能区 (底部)
    └── 💰 我的窝囊费 (个人功能，与鱼圈无关)
```

#### 左侧栏设计细节

| 元素 | 说明 |
| ---- | ---- |
| 栏位宽度 | 约200px，显示完整名称 |
| 鱼圈图标 | 系统提供默认图标，创建鱼圈时选择 |
| 未读消息数 | 鱼圈名称右上角红色数字角标 |
| 当前鱼圈 | 高亮显示（背景色或边框） |
| 切换行为 | 点击直接切换聊天内容到该鱼圈 |
| 创建/加入入口 | 顶部 [+] 按钮，点击展开表单 |
| 个人功能 | 底部"我的窝囊费"，与鱼圈功能分开 |

### 4.3 主内容区

主内容区显示当前鱼圈的功能，默认进入聊天室（蛐蛐间），底部Tab切换到摸鱼鱼或鱼圈管理。

```
主内容区
│
├── 顶部信息栏
│   ├── 鱼圈名称 (可点击打开鱼圈管理弹窗)
│   ├── 功能标签 (如"蛐蛐间"、"摸鱼鱼"、"鱼圈管理")
│   └── 状态信息 (如"正在划水中")

    ├── [🎮 摸鱼鱼]
    └── [⚙️ 鱼圈管理] (始终显示)
```

### 5.4 创建/加入鱼圈

#### 功能说明

功能目标：提供创建和加入鱼圈的入口，支持实时邀请机制

#### 页面设计

功能入口：左侧鱼圈栏顶部 [+] 按钮

页面组成：

*   Tab切换：加入同事会合点 / 组建全新鱼圈
*   加入表单：输入6位邀请码
*   创建表单：输入鱼圈名称
*   邀请状态：显示邀请码、已邀请人数、有效期

#### 创建鱼圈流程

```
输入鱼圈名称
    ↓
生成邀请码（1小时有效期）
    ↓
显示邀请码和邀请链接
    ↓
等待至少1人加入
    ↓
达到2人，自动激活鱼圈
    ↓
进入鱼圈
```

#### 交互设计

| 操作 | 反馈 |
| ---- | ---- |
| 点击[+]按钮 | 展开创建/加入鱼圈表单 |
| 输入鱼圈名称 | 显示字符计数（限50字） |
| 点击"建立安全通道" | 生成邀请码，显示邀请状态 |
| 分享邀请码 | 同事输入邀请码加入 |
| 达到2人 | 自动激活鱼圈，进入鱼圈 |
| 超过1小时 | 邀请失效，自动删除未激活的鱼圈 |

#### 业务规则

| 编号 | 规则 |
| ---- | ---- |
| BR-016 | 创建鱼圈至少需要2人（创建者+1人） |
| BR-017 | 邀请码有效期1小时 |
| BR-018 | 超过1小时未激活，鱼圈自动删除 |
| BR-019 | 邀请码为6位数字 |
| BR-020 | 鱼圈名称限50字以内 |

功能目标：窝囊费作为个人功能，进入后隐藏左侧鱼圈栏和底部Tab栏，全宽展示窝囊费内容，提供返回按钮回到首页

#### 页面设计

功能入口：左侧栏底部"我的窝囊费"

页面组成：

*   导航栏返回按钮：`[← 返回]`，位于 Logo 右侧
*   标题区："带薪躺平·实时窝囊费"
*   防窥按钮
*   收入展示区
*   进度条区
*   趣味换算区
*   参数编辑表单（折叠态）

#### 返回按钮设计

| 属性 | 说明 |
| ---- | ---- |
| 位置 | 顶部导航栏 Logo 右侧 |
| 样式 | `← 返回`，带箭头的文本按钮，遵循潮玩风格（粗边框 + 硬偏移阴影） |
| 点击行为 | 跳转到 `/home`（首页/蛐蛐间） |
| 显示条件 | 仅在窝囊费页面显示，其他页面不显示 |

#### 交互设计

| 操作 | 反馈 |
| ---- | ---- |
| 进入窝囊费页面 | 左侧鱼圈栏隐藏，底部Tab栏隐藏，内容区全宽展示 |
| 点击返回按钮 | 跳转到 `/home`，恢复左侧鱼圈栏和底部Tab栏 |

#### 业务规则

| 编号 | 规则 |
| ---- | ---- |
| BR-024 | 窝囊费页面不显示左侧鱼圈栏 |
| BR-025 | 窝囊费页面不显示底部Tab栏（蛐蛐间/摸鱼鱼） |
| BR-026 | 返回按钮点击后跳转到 `/home` |

## 6. 验收标准

### Happy Path

*   AC-001
    *   Given 用户在首页页面
    *   When 查看左侧栏
    *   Then 显示所有已加入的鱼圈，底部显示"我的窝囊费"

*   AC-002
    *   Given 用户在左侧栏看到多个鱼圈
    *   When 点击某个鱼圈
    *   Then 聊天内容切换到该鱼圈，主内容区显示该鱼圈的聊天室

*   AC-003
    *   Given 用户在主内容区
    *   When 点击底部Tab"摸鱼鱼"
    *   Then 显示当前鱼圈的摸鱼鱼页面，包含签到、宠物鱼、排行榜、图鉴

*   AC-004
    *   Given 用户在摸鱼鱼页面
    *   When 点击"签到领鱼币"
    *   Then 签到成功，鱼币余额+1，按钮变为"已签到"

*   AC-005
    *   Given 用户在摸鱼鱼页面
    *   When 点击宠物鱼
    *   Then 触发1秒摸鱼动画，动画结束后弹出卡片掉落弹窗

*   AC-006
    *   Given 用户在左侧栏底部
    *   When 点击"我的窝囊费"
    *   Then 跳转到窝囊费页面（个人功能）

*   AC-007
    *   Given 新用户注册成功
    *   When 查看引导页面
    *   Then 显示"加入同事的鱼圈"和"创建新鱼圈"选项

*   AC-008
    *   Given 用户创建新鱼圈
    *   When 输入鱼圈名称并点击"建立安全通道"
    *   Then 生成邀请码，显示邀请状态，等待至少1人加入

*   AC-009
    *   Given 用户在邀请状态页面
    *   When 超过1小时未有人加入
    *   Then 邀请失效，自动删除未激活的鱼圈

*   AC-010
    *   Given 用户在摸鱼鱼页面
    *   When 点击"装扮商城"
    *   Then 打开装饰商店弹窗，显示装饰列表和购买功能

*   AC-011
    *   Given 用户在窝囊费页面
    *   When 查看页面布局
    *   Then 左侧鱼圈栏隐藏，页面全宽展示，导航栏显示返回按钮

*   AC-012
    *   Given 用户在窝囊费页面
    *   When 点击导航栏返回按钮
    *   Then 跳转到 /home（首页/蛐蛐间），左侧鱼圈栏和底部Tab栏恢复显示

### Edge Case

*   AC-101
    *   Given 用户只加入了一个鱼圈
    *   When 查看左侧栏
    *   Then 只显示一个鱼圈和"我的窝囊费"

*   AC-102
    *   Given 鱼圈名称过长
    *   When 在左侧栏显示
    *   Then 名称截断并显示省略号

*   AC-103
    *   Given 用户有未读消息
    *   When 查看左侧栏
    *   Then 鱼圈右上角显示红色数字角标

*   AC-104
    *   Given 用户没有鱼圈
    *   When 尝试访问摸鱼鱼
    *   Then 显示"加入鱼圈解锁"提示

*   AC-105
    *   Given 用户在创建鱼圈邀请状态
    *   When 达到2人
    *   Then 自动激活鱼圈，进入鱼圈

*   AC-106
    *   Given 用户在摸鱼鱼页面
    *   When 今日摸鱼次数已达上限
    *   Then 宠物鱼不可点击，显示"你已触及今日防沉迷保护网！"

*   AC-107
    *   Given 用户在窝囊费页面
    *   When 查看导航栏
    *   Then 显示返回按钮，不显示底部Tab栏（蛐蛐间/摸鱼鱼）

### Business Rule

*   AC-201
    *   Given 用户切换鱼圈
    *   When 系统处理
    *   Then 默认进入新鱼圈的聊天室（蛐蛐间）

*   AC-202
    *   Given 用户创建新鱼圈
    *   When 系统处理
    *   Then 邀请码有效期1小时，至少需要1人加入才能激活

*   AC-203
    *   Given 用户签到
    *   When 系统处理
    *   Then 鱼币进入当前鱼圈的公共账户

*   AC-204
    *   Given 用户摸鱼
    *   When 系统处理
    *   Then 每次摸鱼固定获得1点成长值

*   AC-205
    *   Given 用户购买装饰
    *   When 系统处理
    *   Then 装饰自动显示在鱼缸，所有鱼圈成员可见

*   AC-206
    *   Given 用户进入窝囊费页面
    *   When 系统渲染布局
    *   Then 左侧鱼圈栏和底部Tab栏均隐藏，页面全宽展示，导航栏显示返回按钮

## 8. 非功能需求

### 性能

*   左侧栏加载时间 < 1秒
*   鱼圈切换响应时间 < 500ms
*   摸鱼动画播放流畅，无卡顿
*   签到响应时间 < 1秒
*   装饰购买响应时间 < 2秒

### 可用性

*   鱼圈切换成功率 ≥ 99%
*   左侧栏显示正确率 100%
*   摸鱼操作成功率 ≥ 99%
*   签到成功率 ≥ 99%
*   装饰购买成功率 ≥ 99%

### 安全性

*   只有加入的鱼圈才会显示在左侧栏
*   未读消息数基于实际消息统计
*   每日配额基于服务端统计，防止客户端篡改
*   鱼币余额基于服务端计算，防止客户端篡改
*   装饰购买基于服务端验证，防止重复购买

## 9. 数据模型

### 鱼圈 (Circle)

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| id | string | 鱼圈唯一标识 |
| name | string | 鱼圈名称 |
| icon | string | 鱼圈图标（系统默认图标标识符） |
| code | string | 6位数字邀请码 |
| creatorId | string | 创建者用户ID |
| ownerId | string | 群主用户ID |
| memberCount | number | 成员数量 |
| createdAt | Timestamp | 创建时间 |
| petFishName | string | 宠物鱼名称 |
| petFishLevel | number | 宠物鱼等级 |
| petFishGrowth | number | 宠物鱼成长值 |
| petFishType | string | 宠物鱼品类 |
| coinBalance | number | 鱼币余额 |

### 存储路径

*   鱼圈文档：`circles/{circleId}`
*   用户文档：`users/{userId}` 中的 `joinedCircleId` 字段
