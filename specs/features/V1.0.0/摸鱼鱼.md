# 摸鱼鱼（游戏系统）PRD

## 1. 背景与目标

### 为什么做

"摸鱼"是职场人日常的高频行为，但传统摸鱼缺乏正向反馈和仪式感。摸鱼鱼将"摸鱼"行为游戏化，通过UNO卡片收集和宠物鱼养成两条主线，让枯燥的工作间隙变得有趣。每次"摸鱼"都能获得随机卡片奖励和宠物鱼成长反馈，形成正向激励循环。

### 用户价值

- 将"摸鱼"从偷偷摸摸的行为转化为有仪式感的收集体验
- UNO卡片收集系统提供长期目标（集齐108张兑换实体卡牌）
- 宠物鱼养成提供即时视觉反馈和团队归属感
- 排行榜激发小团体内的良性竞争

### 成功标准

- 用户日均摸鱼次数 ≥ 10次
- UNO卡片收集参与率 ≥ 80%
- 宠物鱼养成系统可正常升级和进化

***

## 2. 范围定义

### 本次做

- 摸鱼点击按钮（主交互入口）
- 每次摸鱼随机获得1\~2张UNO卡片
- 每日摸鱼次数限制（基于用户ID的确定性随机，15\~45次）
- UNO卡片收集图鉴（54种卡片，含N/R/SR稀有度）
- 鱼圈共享宠物鱼养成（经验值、等级、进化形态）
- 带薪战斗排行榜（今日摸鱼次数 + 历史总次数）
- 集满108张卡片的兑换入口

### 本次不做

- 卡片交易/赠送功能
- 卡片合成/升级功能
- 宠物鱼自定义命名
- 宠物鱼装扮/道具系统
- 跨鱼圈排行榜
- 线上支付兑换

## 5. 功能详细设计

### 5.1 摸鱼点击按钮

#### 功能说明

功能目标：提供核心交互入口，用户每次点击触发卡片掉落和宠物鱼喂养

#### 页面设计

功能入口：摸鱼主界面中央区域

页面组成：

- 标题："带薪疯狂摸鱼 🦥"
- 副标题："点击一键带薪抽卡！每戳一次增加鱼池 XP 且掉落 1\~2 张精美卡片。"
- 今日配额显示："{已用次数} / {上限次数}"
- 大按钮：橙色背景，显示"点击划水带薪抽卡"，带水獭emoji
- 按钮标签："+5 XP"

#### 交互设计

| 操作     | 反馈                                    |
| ------ | ------------------------------------- |
| 点击摸鱼按钮 | 按钮显示"摸鱼中..."，弹出卡片掉落弹窗                 |
| 摸鱼按钮禁用 | 灰色背景，无法点击                             |
| 已达今日上限 | 显示"你已触及今日防沉迷保护网！强制进入'高效装勤快'模式，明天早上见！" |

#### 状态流转

| 状态   | 说明                |
| ---- | ----------------- |
| 可用   | 橙色按钮，显示"点击划水带薪抽卡" |
| 摸鱼中  | 灰色按钮，显示"摸鱼中..."   |
| 已达上限 | 灰色禁用按钮，显示上限提示     |

#### 业务规则

| 编号     | 规则                                                            |
| ------ | ------------------------------------------------------------- |
| BR-001 | 每次摸鱼概率获得1张卡片（50%概率没有，30%概率获得1张普通卡,10%概率获得1张稀有卡，5%概率获得一张超级稀有卡） |
| BR-002 | 每次摸鱼增加鱼池5点经验值（每张卡片）                                           |
| BR-003 | 每日摸鱼次数上限基于用户ID哈希确定性生成，范围15\~45次                               |
| BR-004 | 每日配额在当天重置（基于todayCount统计）                                     |
| BR-005 | 达到上限后按钮禁用，无法继续摸鱼                                              |

#### 异常处理

| 场景     | 处理方式            |
| ------ | --------------- |
| 卡片保存失败 | 控制台输出错误，不影响用户体验 |
| 网络异常   | 显示错误提示，允许重试     |

### 5.2 UNO卡片掉落

#### 功能说明

功能目标：每次摸鱼随机掉落UNO卡片，展示卡片详情

#### 卡片池设计

| 类别     | 数量      | 说明                 |
| ------ | ------- | ------------------ |
| 红色卡片   | 13张     | 0-9 + 跳过 + 反转 + +2 |
| 蓝色卡片   | 13张     | 0-9 + 跳过 + 反转 + +2 |
| 绿色卡片   | 13张     | 0-9 + 跳过 + 反转 + +2 |
| 黄色卡片   | 13张     | 0-9 + 跳过 + 反转 + +2 |
| 万能卡片   | 2张      | 变色 + +4            |
| **总计** | **54种** | -                  |

#### 稀有度设计

| 稀有度      | 标签 | 对应卡片     | 说明  |
| -------- | -- | -------- | --- |
| N（普通）    | N  | 0-9数字卡   | 最常见 |
| R（稀有）    | R  | 跳过、反转、+2 | 较稀有 |
| SR（超级稀有） | SR | 变色、+4    | 最稀有 |

#### 卡片掉落弹窗

页面组成：

- 标题："摸鱼成功！抽得神牌"
- 副标题："今日工作时间在此凝固，转化为可爱的能量 card！"
- 卡片展示区：并排展示1\~2张卡片
- 每张卡片包含：稀有度标签、数字/符号、卡片名称、彩蛋文字
- 按钮："继续摸鱼 / 收下卡牌 🦦"

#### 交互设计

| 操作       | 反馈                                   |
| -------- | ------------------------------------ |
| 摸鱼成功     | 弹出卡片掉落弹窗                             |
| 点击"继续摸鱼" | 关闭弹窗，返回摸鱼主界面                         |
| 卡片颜色区分   | Red红色、Blue蓝色、Green绿色、Yellow黄色、Wild紫色 |

#### 业务规则

| 编号     | 规则                      |
| ------ | ----------------------- |
| BR-006 | 卡片从54种卡片池中随机抽取          |
| BR-007 | 卡片可重复获得，重复卡片count累加     |
| BR-008 | 每张卡片包含独特彩蛋文字（bonusText） |

## 6. 验收标准

### Happy Path

- AC-001
  - Given 用户今日摸鱼次数未达上限
  - When 用户点击摸鱼按钮
  - Then 弹出卡片掉落弹窗，显示1\~2张随机UNO卡片
- AC-002
  - Given 用户获得UNO卡片
  - When 卡片掉落
  - Then 卡片保存到鱼圈公共收集库，数量累加，并记录掉落日志
- AC-003
  - Given 用户摸鱼成功
  - When 鱼圈宠物鱼获得经验值
  - Then 经验条实时更新，显示当前经验/升级所需经验
- AC-004
  - Given 鱼圈宠物鱼经验达到升级条件
  - When 自动升级
  - Then 等级+1，品类可能变化，显示"进化成功！"动画
- AC-005
  - Given 用户在摸鱼主界面
  - When 查看带薪战斗排行榜
  - Then 显示鱼圈成员按今日次数降序排列的排行榜

### Edge Case

- AC-101
  - Given 用户今日摸鱼次数已达上限
  - When 用户尝试点击摸鱼按钮
  - Then 按钮禁用，显示"你已触及今日防沉迷保护网！"提示
- AC-102
  - Given 用户获得重复卡片
  - When 卡片保存
  - Then 卡片count累加，不新增种类
- AC-103
  - Given 宠物鱼经验溢出升级所需
  - When 自动升级
  - Then 溢出经验保留到下一级
- AC-104
  - Given 用户收集满108张卡片
  - When 点击兑换按钮
  - Then 显示"满足兑换资格"，提供"发送邮件申请"按钮
- AC-105
  - Given 鱼圈成员A获得卡片
  - When 成员B打开图鉴
  - Then 成员B能看到A获得的卡片，图鉴显示鱼圈公共收集数据
- AC-106
  - Given 鱼圈成员A获得卡片
  - When 卡片掉落
  - Then 系统自动记录掉落日志（谁、什么卡片、什么时间）
- AC-107
  - Given 用户打开图鉴页面
  - When 切换到「掉落记录」Tab
  - Then 显示最近50条掉落记录，按时间倒序，每条显示用户昵称、卡片名称和颜色、时间

### Business Rule

- AC-201
  - Given 用户点击摸鱼按钮
  - When 系统决定掉落卡片数量
  - Then 60%概率掉1张，40%概率掉2张
- AC-202
  - Given 用户ID已确定
  - When 计算每日摸鱼上限
  - Then 基于用户ID哈希值确定性生成15\~45之间的上限
- AC-203
  - Given 用户摸鱼成功
  - When 计算宠物鱼经验
  - Then 每张卡片增加5点经验值
- AC-204
  - Given 宠物鱼达到新的等级阶段
  - When 更新品类
  - Then 1-4级"肥嘟嘟胖金鱼"，5-9级"带薪发愣神游鳌"，10-14级"太极双休太公鱼"，15级+"极品七彩锦鲤皇"
