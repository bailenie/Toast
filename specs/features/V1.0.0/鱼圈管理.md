# 鱼圈管理 PRD

## 1. 背景与目标

### 为什么做

鱼圈是摸鱼圈的核心社交单元，承载着聊天、摸鱼游戏、宠物鱼养成等功能。用户需要一个直观的管理界面来创建、加入、退出鱼圈，以及管理鱼圈成员。私有鱼圈则为每个用户提供独处空间。

### 用户价值

- 通过6位数字邀请码快速加入同事的鱼圈
- 创建专属鱼圈，定制名称和宠物鱼
- 私有鱼圈提供个人独处空间，无法被解散或退出
- 群主可踢出不活跃成员

### 成功标准

- 创建鱼圈流程可在 10 秒内完成
- 通过邀请码加入鱼圈成功率 ≥ 99%
- 成员列表实时更新

***

## 2. 范围定义

### 本次做

- 创建鱼圈（自定义名称，自动生成6位邀请码）
- 通过6位数字邀请码加入鱼圈
- 查看当前鱼圈信息（名称、ID、邀请码）
- 查看鱼圈成员列表
- 退出当前鱼圈（回到私有鱼圈）
- 群主踢出成员
- 复制邀请码到剪贴板
- 私有鱼圈展示
- **邀请等待状态展示**：创建鱼圈后在侧边栏显示"等待中"的鱼圈，包含过期倒计时、已邀请人数、状态标签，支持点击打开邀请弹窗

### 本次不做

- 鱼圈名称修改
- 鱼圈公告功能
- 鱼圈头像/封面设置
- 成员权限管理（除群主外）
- 鱼圈搜索/发现
- 鱼圈二维码邀请

***

## 4. 页面结构

### 4.1 页面入口

| 功能模块    | 入口位置                 | 交互方式        |
| ------- | -------------------- | ----------- |
| 鱼圈列表与切换 | 蛐蛐蛐页 → **左侧常驻鱼圈栏**   | 点击鱼圈直接切换聊天  |
| 创建/加入鱼圈 | 左侧常驻栏 → **顶部操作区**    | 点击展开创建/加入表单 |
| 鱼圈管理面板  | 蛐蛐蛐页 → **当前聊天窗口右上角** | 点击打开独立弹窗    |

### 4.2 左侧常驻鱼圈栏

左侧常驻鱼圈栏是用户查看和切换鱼圈的主要入口，解决入口发现性和多鱼圈导航问题。

```
蛐蛐蛐页面布局
│
├── 左侧常驻鱼圈栏 (CircleSidebar / 200px宽 / 固定显示)
│   │
│   ├── ① 操作区 (顶部)
│   │   └── [+] 按钮 (点击展开创建/加入鱼圈表单)
│   │
│   ├── ② 私有鱼圈 (固定在顶部，不可移除)
│   │   ├── 图标 (系统默认图标)
│   │   ├── 名称: "我的专属安全水箱 🐠"
│   │   └── 未读消息数角标 (右上角红色数字)
│   │
│   └── ③ 公共鱼圈列表 (滚动)
│       ├── 图标 (系统默认图标)
│       ├── 名称 (完整显示)
│       ├── 在线人数 (可选)
│       └── 未读消息数角标 (右上角红色数字)
│
├── 中间聊天区
│   └── 当前鱼圈聊天内容
│
└── 右侧功能区 (可选)
```

#### 左侧栏设计细节

| 元素      | 说明                |
| ------- | ----------------- |
| 栏位宽度    | 约200px，显示完整名称     |
| 私有鱼圈    | 固定在顶部，始终显示，不可移除   |
| 鱼圈图标    | 系统提供默认图标，创建鱼圈时选择  |
| 未读消息数   | 鱼圈名称右上角红色数字角标     |
| 当前鱼圈    | 高亮显示（背景色或边框）      |
| 切换行为    | 点击直接切换聊天内容到该鱼圈    |
| 创建/加入入口 | 顶部 \[+] 按钮，点击展开表单 |

### 4.3 鱼圈管理弹窗

鱼圈管理功能（成员列表、邀请码、退出、踢人等）以**独立弹窗**形式展示，从当前聊天窗口右上角打开。

```
鱼圈管理弹窗 (CircleModal / 居中弹窗)
│
├── ① 当前鱼圈信息区 (顶部)
│   ├── 标签: "个人专属静室" / "同事划水会合点"
│   ├── 鱼圈名称
│   ├── 通道ID
│   ├── 邀请码 (可复制，仅公共鱼圈)
│   └── 退出按钮 (仅公共鱼圈)
│
├── ② 成员列表区 (中部)
│   ├── 标题: "会合点战友 Roster (X/10)"
│   └── 成员列表 (滚动)
│       ├── 头像 emoji
│       ├── 昵称
│       ├── "群主" / "我" 标签
│       ├── 邮箱
│       └── 踢出按钮 (仅群主可见)
│
└── ③ 关闭按钮 (右上角)
```

## 5. 功能详细设计

### 5.1 当前鱼圈信息展示

#### 功能说明

功能目标：展示用户当前所在鱼圈的基本信息

#### 页面设计

功能入口：**左侧常驻鱼圈栏**（当前鱼圈高亮显示）+ **鱼圈管理弹窗**（点击聊天窗口右上角打开）

页面组成：

- 标签："个人专属静室" 或 "同事划水会合点"
- 鱼圈名称
- 通道ID（灰色小字）

#### 私有鱼圈显示

- 标签：🔒 个人专属静室
- 提示文字："这是你的私有水池。当你无心合群或想要独自发呆、默默记录属于自己的带薪窝囊费、悄悄抽UNO卡片时，就在这里安然度过吧！"
- **左侧栏展示**：固定在顶部，显示图标和名称

#### 公共鱼圈显示

- 标签：⚓️ 同事划水会合点
- 6位分享秘钥（可复制）
- "退出此圈"按钮

#### 业务规则

| 编号     | 规则                           |
| ------ | ---------------------------- |
| BR-001 | 私有鱼圈ID格式为"private\_{userId}" |
| BR-002 | 私有鱼圈邀请码固定为"000000"           |
| BR-003 | 私有鱼圈不可退出、不可解散                |

#### 功能说明

功能目标：展示当前鱼圈的所有成员

| 元素    | 说明                  |

### 5.7 邀请等待状态展示

#### 功能说明

功能目标：创建鱼圈后，在侧边栏持续展示"等待中"的鱼圈，让群主随时掌握邀请状态，避免关闭弹窗后找不到自己的群邀请

#### 页面设计

功能入口：**左侧常驻鱼圈栏**

页面组成：

- "等待中"的鱼圈在侧边栏中以特殊样式展示（与已激活鱼圈视觉区分）
- 显示信息：鱼圈名称、状态标签（"⏳ 等待加入"）、过期倒计时、已邀请人数
- 点击"等待中"鱼圈 → 打开邀请弹窗（与创建鱼圈时的弹窗一致）

#### 左侧栏"等待中"鱼圈样式

```
左侧鱼圈栏 (200px)
│
├── ① 操作区 (顶部)
│   └── [+] 按钮
│
├── ② 已激活鱼圈列表
│   ├── 🐟 技术部摸鱼会 (高亮/未读角标)
│   └── 🐠 市场部早退党
│
├── ③ "等待中"鱼圈列表 (特殊样式)
│   └── ⏳ 第五工位躺平分会
│       ├── 状态标签: "⏳ 等待加入"
│       ├── 倒计时: "剩余 45:23"
│       └── 已邀请人数: "0/1 人加入"
│
├── ────────── 分隔线
│
└── ④ 个人功能区 (底部)
    └── 💰 我的窝囊费
```

#### "等待中"鱼圈视觉设计

| 元素    | 说明                       |
| ----- | ------------------------ |
| 背景色   | 浅灰色/半透明，与已激活鱼圈区分         |
| 状态标签  | 黄色标签 "⏳ 等待加入"，位于名称下方     |
| 倒计时   | 等宽数字字体，每秒更新，红色警示（<10分钟时） |
| 已邀请人数 | 灰色小字 "X/1 人加入"           |
| 图标    | 半透明/灰色鱼圈图标               |
| 整体样式  | 稍微降低透明度，表示"未激活"状态        |

#### 交互设计

| 操作           | 反馈                              |
| ------------ | ------------------------------- |
| 点击"等待中"鱼圈    | 打开邀请弹窗，显示邀请码和倒计时                |
| 邀请弹窗关闭       | 返回侧边栏，"等待中"鱼圈仍显示                |
| 倒计时归零        | "等待中"鱼圈从侧边栏消失，提示"邀请已过期，鱼圈已自动删除" |
| 第2人加入成功      | 自动激活鱼圈，"等待中"样式变为正常鱼圈样式          |
| 同时有多个"等待中"鱼圈 | 按创建时间倒序排列，最多显示3个                |

#### 邀请弹窗内容

点击"等待中"鱼圈后打开的弹窗，与创建鱼圈时的弹窗一致：

- 标题：鱼圈名称
- 6位邀请码（大号显示，可复制）
- 过期倒计时（实时更新）
- 已邀请人数："X/1 人加入"
- 复制邀请码按钮
- 关闭按钮

#### 业务规则

| 编号     | 规则                                         |
| ------ | ------------------------------------------ |
| BR-021 | 邀请码有效期1小时，过期后鱼圈自动删除                        |
| BR-022 | 每个用户同时最多有3个"等待中"的鱼圈                        |
| BR-023 | 达到3个上限时，提示"你已经有3个等待中的鱼圈了，等它们过期或有人加入后再创建吧！" |
| BR-024 | "等待中"鱼圈按创建时间倒序排列（最新的在最上面）                  |
| BR-025 | 至少1人加入后鱼圈自动激活，"等待中"样式消失                    |
| BR-026 | 倒计时<10分钟时，倒计时文字变为红色警示                      |
| BR-027 | 倒计时每秒更新，使用等宽数字字体                           |

#### 异常处理

| 场景     | 处理方式                               |
| ------ | ---------------------------------- |
| 达到3个上限 | 提示"你已经有3个等待中的鱼圈了，等它们过期或有人加入后再创建吧！" |
| 邀请码过期  | 鱼圈从侧边栏自动消失，如用户正在查看弹窗则关闭弹窗并提示       |
| 网络异常   | 侧边栏保留本地缓存的"等待中"鱼圈信息                |

## 6. 验收标准

### Happy Path

- AC-001
  - Given 用户在鱼圈管理页
  - When 输入鱼圈名称并点击"建立安全通道"
  - Then 鱼圈创建成功，用户成为群主，自动切换到新鱼圈
- AC-002
  - Given 用户在鱼圈管理页
  - When 输入正确的6位邀请码并点击"加入"
  - Then 成功加入鱼圈，切换到新鱼圈
- AC-003
  - Given 用户在公共鱼圈
  - When 点击"退出此圈"并确认
  - Then 退出成功，回到私有鱼圈
- AC-004
  - Given 群主在鱼圈管理页
  - When 点击某成员的踢出按钮并确认
  - Then 成员被踢出，从列表移除
- AC-005
  - Given 用户在公共鱼圈
  - When 点击邀请码旁的复制图标
  - Then 邀请码复制到剪贴板，显示成功提示
- AC-006
  - Given 用户创建了新鱼圈（未激活）
  - When 查看左侧鱼圈栏
  - Then 显示"⏳ 等待加入"状态标签、过期倒计时、已邀请人数
- AC-007
  - Given 用户点击左侧栏"等待中"鱼圈
  - When 邀请弹窗打开
  - Then 显示邀请码、过期倒计时（实时更新）、已邀请人数、复制按钮
- AC-008
  - Given "等待中"鱼圈的邀请码倒计时归零
  - When 系统检查过期
  - Then 鱼圈从侧边栏自动消失
- AC-009
  - Given 第2人通过邀请码加入"等待中"鱼圈
  - When 系统激活鱼圈
  - Then "等待中"样式变为正常鱼圈样式，倒计时消失
- AC-010
  - Given 用户已有3个"等待中"的鱼圈
  - When 尝试创建第4个鱼圈
  - Then 提示"你已经有3个等待中的鱼圈了，等它们过期或有人加入后再创建吧！"
- AC-011
  - Given "等待中"鱼圈倒计时<10分钟
  - When 查看侧边栏
  - Then 倒计时文字变为红色警示

### Edge Case

- AC-101
  - Given 用户尝试加入鱼圈
  - When 输入的邀请码不存在
  - Then 显示"找不到匹配的鱼圈！请核对同事给的6位分享秘钥是否正确。"
- AC-102
  - Given 用户尝试加入鱼圈
  - When 目标鱼圈已满10人
  - Then 显示"这个划水小分队已经达到10人满负荷啦！"
- AC-103
  - Given 用户已在某鱼圈
  - When 尝试再次加入同一鱼圈
  - Then 显示"你已经在这只划水队伍中啦！"
- AC-104
  - Given 用户尝试创建鱼圈
  - When 鱼圈名称超过50字符
  - Then 显示"名称超长，鱼圈名限50字以内哦\~"
- AC-105
  - Given 用户在私有鱼圈
  - When 查看鱼圈管理页
  - Then 显示"个人专属静室"标签，无退出按钮

### Business Rule

- AC-201
  - Given 新用户注册
  - When 系统创建私有鱼圈
  - Then 私有鱼圈名称为"我的专属安全水箱 🐠"，邀请码为"000000"
- AC-202
  - Given 用户创建新鱼圈
  - When 系统生成邀请码
  - Then 自动生成6位纯数字邀请码
- AC-203
  - Given 用户加入新鱼圈
  - When 系统更新用户资料
  - Then joinedCircleId 更新为新鱼圈ID
- AC-204
  - Given 群主踢出成员
  - When 系统更新鱼圈数据
- AC-205
  - Given 用户创建新鱼圈
  - When 系统生成邀请码
  - Then 邀请码有效期1小时，鱼圈 isActive=false，需至少1人加入才激活
- AC-206
  - Given 用户尝试创建第4个"等待中"的鱼圈
  - When 系统检查数量上限
  - Then 返回错误，提示已达3个上限
  - Then 鱼圈成员数减1，被踢用户的 joinedCircleId 重置

## 9. 数据模型

### 鱼圈 (Circle)

| 字段           | 类型          | 说明                              |
| ------------ | ----------- | ------------------------------- |
| id           | string      | 鱼圈唯一标识                          |
| name         | string      | 鱼圈名称                            |
| icon         | string      | 鱼圈图标（系统默认图标标识符）                 |
| code         | string      | 6位数字邀请码                         |
| creatorId    | string      | 创建者用户ID                         |
| ownerId      | string      | 群主用户ID                          |
| memberCount  | number      | 成员数量                            |
| **isActive** | **boolean** | **鱼圈是否已激活（false=等待中，true=已激活）** |
| createdAt    | Timestamp   | 创建时间                            |
| petFishName  | string      | 宠物鱼名称                           |
| petFishLevel | number      | 宠物鱼等级                           |
| petFishExp   | number      | 宠物鱼经验值                          |
| petFishType  | string      | 宠物鱼品类                           |

### 邀请码 (Invite)

| 字段        | 类型        | 说明                  |
| --------- | --------- | ------------------- |
| id        | string    | 邀请记录唯一标识            |
| circleId  | string    | 关联的鱼圈ID             |
| code      | string    | 6位数字邀请码（唯一）         |
| createdBy | string    | 创建者用户ID             |
| status    | string    | 状态：active / expired |
| expiresAt | Timestamp | 过期时间（创建后1小时）        |
| createdAt | Timestamp | 创建时间                |

- 邀请码文档：`invites/{inviteId}`（按 circleId 索引）
- 用户文档：`users/{userId}` 中的 `joinedCircleId` 字段
