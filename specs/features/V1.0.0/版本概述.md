# 产品概述

**产品名称**：吐司（吐槽公司的谐音，打工人的情绪排解平台）

## 1. 项目愿景 (Vision)
> 为职场小团体提供一个安全、有趣的情绪释放空间，让"摸鱼"从偷偷摸摸变成一种有仪式感的社交行为。

## 2. 核心价值 (Value Proposition)
*   **用户价值**：
    *   在不被监控的安全环境中释放职场压力和负面情绪
    *   通过游戏化机制将枯燥的工作时间转化为有趣的收集体验
    *   通过"窝囊费"可视化，让每一天的付出都有即时的情绪回报
*   **商业/技术价值**：
    *   以极低的开发成本验证"办公室情绪工具"这一细分市场
    *   通过实体UNO卡牌周边探索轻量变现路径
    *   积累职场情绪数据（匿名聚合），为后续商业化提供基础

## 3. 核心板块 (Core Modules)

### 3.1 蛐蛐蛐（聊天系统）
*   **描述**：面向鱼圈成员的即时文字聊天系统，强调短时、高频、无留存的情绪表达
*   **关键特性**：
    *   纯文字消息，单条最大 500 字符
    *   消息生命周期 5 分钟自动销毁，无历史记录
    *   每个鱼圈最多 10 人
    *   每个用户默认拥有一个不可退出的「私有鱼圈」

### 3.2 摸鱼鱼（游戏系统）
*   **描述**：将"摸鱼"行为游戏化的收集系统，包含 UNO 卡片收集和宠物鱼养成两条主线
*   **关键特性**：
    *   总计 108 张 UNO 卡片，通过每日摸鱼行为随机掉落（每次 1~2 张）
    *   每日摸鱼次数随机 1~50 次，制造不确定性与惊喜感
    *   每个鱼圈共享一只宠物鱼，承载摸鱼行为的视觉反馈
    *   鱼圈摸鱼统计面板（今日总次数 + 个人次数）

### 3.3 窝囊费系统（时间价值可视化）
*   **描述**：将工作时间实时映射为金钱收益的情绪补偿系统
*   **关键特性**：
    *   用户自定义上班时间、下班时间、单日工资
    *   实时计算并展示"今日已赚窝囊费"
    *   将时间消耗转化为可见的情绪补偿指标

### 3.4 鱼圈管理
*   **描述**：支撑社交关系的基础管理模块
*   **关键特性**：
    *   创建鱼圈、加入鱼圈（邀请制）
    *   成员管理（查看成员、移除成员）
    *   鱼圈解散 / 退出（私有鱼圈不可退出）

### 3.5 老板键（快捷键 `Esc`）

#### 1、核心功能
按下键盘上的 **`Esc`** 键，可将当前页面**瞬间切换为伪装的工作界面**，再次按下则恢复原界面。让用户在他人接近时快速隐藏“摸鱼”内容，避免被发现。

#### 2、触发与生效条件
- **快捷键**：`Esc`（全局监听，页面获得焦点时有效）
- **生效范围**：仅影响当前浏览器标签页内的内容切换，不隐藏浏览器窗口或任务栏
- **防误触**：连续快速按 `Esc` 3 次不触发（避免与浏览器默认行为冲突）

#### 3、触发后的变化

| 变化项 | 真实界面（摸鱼鱼） | 伪装界面（工作界面） |
|--------|-------------------|---------------------|
| **页面内容** | 鱼圈聊天、摸鱼按钮、卡片收集、窝囊费面板 | 预置的“正经”模板（如在线表格、邮件列表、代码编辑器） |
| **页面标题** | `摸鱼鱼 - 鱼圈名称` | `工作周报 - 在线文档` |
| **浏览器图标 (Favicon)** | 🐟 | 📄 或 📧 |
| **输入框草稿** | 保留（恢复时重现） | 清空（防止敏感内容暴露） |
| **弹窗/侧边栏** | 关闭所有二级界面 | 无 |

#### 4、伪装界面模板（用户可自行选择）
- **在线表格**（类 Excel / Google Sheets）

#### 5、恢复方式
- 再次按下 **`Esc`** → 立即恢复真实界面，**所有状态保留**（聊天未发送内容、摸鱼倒计时、收集进度等不变）
- 点击伪装界面角落的 **“返回摸鱼鱼”** 小按钮（备选）


### 3.5 管理后台 (Admin Dashboard)
*   **描述**：面向运营人员的后台管理系统，用于用户管理、内容监管和数据统计
*   **关键特性**：
    *   管理员登录与权限控制
    *   用户管理：查看用户列表、禁用/启用用户
    *   鱼圈管理：查看所有鱼圈、强制解散违规鱼圈
    *   数据统计：用户增长、活跃度、摸鱼数据概览
    *   系统配置：游戏参数调整（摸鱼次数范围、卡片掉落率等）

## 4. 目标用户 & 核心场景 (User & Scenes)
### 4.1 目标用户
*   **主要用户：办公室小团体成员**：同一部门或相邻工位的同事（3~10人），长期共处但缺乏安全的情绪宣泄渠道，核心诉求是"在不被发现的前提下释放压力"
*   **次要用户：有稳定工作节奏的职场人**：朝九晚五、重复性工作中存在高频情绪压力的人群，核心诉求是"让上班这件事变得不那么无聊"

### 4.2 核心场景 (Key Use Cases)
*   **场景 1：午休吐槽**：小 A 被领导批评后，在鱼圈里发了一条吐槽消息，同事们纷纷附和，5分钟后消息自动消失，不留痕迹
*   **场景 2：上班摸鱼**：小 B 在工作间隙打开吐司，点击摸鱼获得 2 张 UNO 卡片，看着宠物鱼的成长反馈，获得短暂的愉悦感
*   **场景 3：工资倒计时**：小 C 下午 3 点打开窝囊费系统，看到"今日已赚 ¥280"，心想"再撑 2 小时就赚满了"，获得继续工作的动力

## 5. 关键业务流程 (Core Workflows)

### 用户注册 → 加入鱼圈 → 日常使用链路
> 用户注册账号 → 系统自动创建私有鱼圈 → 通过邀请码加入同事鱼圈 → 在鱼圈中发消息吐槽 / 点击摸鱼收集卡片 / 查看窝囊费进度 → 每日重复使用

### UNO 卡片收集 → 实物兑换链路
> 用户每日摸鱼 → 随机获得 UNO 卡片 → 查看收集进度 → 集齐 108 张 → 联系客服购买实体定制 UNO 卡牌

## 6. 验收标准 (Acceptance Criteria)
*   **功能完备性**：
    *   注册登录流程完整（邮箱验证码 + 密码两种方式）
    *   鱼圈创建、加入、退出、解散流程跑通
    *   蛐蛐蛐聊天消息发送、接收、5分钟自动消失正常工作
    *   摸鱼鱼系统每日摸鱼次数随机、卡片掉落、收集进度查看正常
    *   窝囊费实时计算和展示正常
    *   无严重 Bug，核心流程可完整走通
*   **技术指标**：
    *   页面首次加载 < 3 秒
    *   聊天消息延迟 < 1 秒
    *   支持同时在线 50+ 用户（初期目标）

## 7. 约束条件 (Constraints)
*   一期仅支持 PC 端 Web 浏览器
*   不支持匿名机制，所有用户需注册登录
*   不支持线上支付，商业化通过线下客服对接
*   消息不持久化，5分钟后物理删除

## 8. 项目边界 (Out of Scope)
*   暂不支持移动端（App / H5 / 小程序）
*   暂不支持图片、语音、视频等富媒体消息
*   暂不支持聊天记录留存和搜索
*   暂不支持开放社交（陌生人发现、广场等）
*   暂不支持多语言和国际化
