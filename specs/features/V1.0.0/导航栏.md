# 导航栏 PRD

## 1. 背景与目标

### 为什么做

导航栏是摸鱼圈应用的核心组件，位于首页层顶部，负责在三个功能页面（窝囊费、蛐蛐蛐、摸鱼鱼）之间快速切换，并提供用户账号操作入口。它是用户日常使用中最频繁交互的元素，直接影响应用的易用性和操作效率。

### 用户价值

- 提供清晰的功能入口，让用户能够快速定位和切换到所需功能
- 固定在页面顶部，确保在任何功能页面都能快速访问其他功能
- 通过用户头像菜单提供便捷的账号管理入口

### 成功标准

- Tab切换响应时间 ≤ 200ms
- 页面切换动画流畅，无明显卡顿
- URL同步正确，支持浏览器前进/后退导航

***

## 2. 范围定义

### 本次做

- Logo + 应用名称显示
- 3个功能Tab（窝囊费、蛐蛐蛐、摸鱼鱼）
- Tab高亮状态显示
- Tab切换与URL同步
- 用户头像显示
- 用户头像下拉菜单（个人信息、头像选择、登出）
- 登出确认弹窗
- 固定顶部布局

### 本次不做

- 响应式导航栏（移动端适配）
- 消息通知提醒
- 鱼圈名称/状态显示
- 搜索功能
- 面包屑导航

***

## 5. 功能详细设计

### 5.1 Logo + 应用名称

#### 功能说明

功能目标：展示应用品牌标识，提供回到首页的快捷入口

#### 组件设计

| 组件     | 类型         | 说明      |
| ------ | ---------- | ------- |
| Logo图标 | Image/Icon | 摸鱼圈Logo |
| 应用名称   | Text       | 显示"摸鱼鱼" |

#### 交互设计

| 操作       | 反馈            |
| -------- | ------------- |
| 点击Logo区域 | 跳转到窝囊费页（默认首页） |

***

### 5.2 功能Tab

#### 功能说明

功能目标：提供三个功能页面的切换入口，支持URL同步

#### 组件设计

| 组件     | 类型  | 说明             |
| ------ | --- | -------------- |
| 窝囊费Tab | Tab | 图标💰 + 文字"窝囊费" |
| 蛐蛐蛐Tab | Tab | 图标💬 + 文字"蛐蛐蛐" |
| 摸鱼鱼Tab | Tab | 图标🎮 + 文字"摸鱼鱼" |

#### 路由映射

| Tab | 路由路径      | 说明          |
| --- | --------- | ----------- |
| 窝囊费 | `/salary` | 默认首页        |
| 蛐蛐蛐 | `/chat`   | 聊天功能 + 鱼圈管理 |
| 摸鱼鱼 | `/game`   | 游戏收集功能      |

#### 交互设计

| 操作       | 反馈                      |
| -------- | ----------------------- |
| 点击Tab    | 高亮当前Tab，切换到对应页面，URL同步更新 |
| 鼠标悬停Tab  | 显示悬停状态（背景色变化）           |
| 浏览器前进/后退 | 根据URL切换对应Tab            |

#### 样式规范

| 状态   | 说明               |
| ---- | ---------------- |
| 默认状态 | 未选中Tab，灰色文字      |
| 悬停状态 | 背景色变化，文字颜色变化     |
| 选中状态 | 高亮背景，强调色文字，底部指示条 |

***

### 5.3 用户头像菜单

#### 功能说明

功能目标：提供用户账号操作入口

#### 组件设计

| 组件   | 类型       | 说明                 |
| ---- | -------- | ------------------ |
| 用户头像 | Avatar   | 显示用户选择的头像emoji     |
| 下拉菜单 | Dropdown | 包含个人信息、修改密码、登出三个选项 |

#### 交互设计

| 操作       | 反馈        |
| -------- | --------- |
| 点击头像     | 显示/隐藏下拉菜单 |
| 点击"个人信息" | 显示个人信息弹窗  |
| 点击"修改密码" | 显示修改密码弹窗  |
| 点击"下班跑路" | 显示登出确认弹窗  |
| 点击菜单外部区域 | 关闭下拉菜单    |

***

### 5.4 登出确认弹窗

#### 功能说明

功能目标：防止用户误操作登出，提供二次确认

#### 组件设计

| 组件   | 类型     | 说明                        |
| ---- | ------ | ------------------------- |
| 弹窗标题 | Text   | "确认操作"                    |
| 确认文案 | Text   | "你确信要收拾公文包下线，离开今天的带薪阵地吗？" |
| 取消按钮 | Button | 关闭弹窗                      |
| 确认按钮 | Button | 执行登出操作                    |

#### 交互设计

| 操作     | 反馈             |
| ------ | -------------- |
| 点击"取消" | 关闭弹窗，保持当前状态    |
| 点击"确认" | 清除Token，跳转到登录页 |
| 点击遮罩层  | 关闭弹窗，保持当前状态    |

## 6. 验收标准

### Happy Path

- AC-001
  - Given 用户已登录并在窝囊费页
  - When 用户点击"蛐蛐蛐"Tab
  - Then 导航栏高亮"蛐蛐蛐"Tab，页面切换到蛐蛐蛐页，URL更新为`/chat`
- AC-002
  - Given 用户已在某个功能页面
  - When 用户点击Logo区域
  - Then 页面跳转到窝囊费页，高亮"窝囊费"Tab
- AC-003
  - Given 用户已登录
  - When 用户点击头像，点击"下班跑路"
  - Then 显示登出确认弹窗，文案为"你确信要收拾公文包下线，离开今天的带薪阵地吗？"
- AC-004
  - Given 登出确认弹窗已显示
  - When 用户点击"确认"
  - Then 用户成功登出，跳转到登录页

### Edge Case

- AC-101
  - Given 用户在蛐蛐蛐页输入了聊天草稿
  - When 用户切换到窝囊费页再切回蛐蛐蛐页
  - Then 聊天草稿保持不变
- AC-102
  - Given 用户通过浏览器后退按钮返回
  - When URL变化触发路由切换
  - Then 导航栏高亮Tab与当前页面保持一致
- AC-103
  - Given 下拉菜单已展开
  - When 用户点击菜单外部区域
  - Then 下拉菜单关闭

### Business Rule

- AC-201
  - Given 用户登录成功
  - When 首次进入首页
  - Then 默认显示窝囊费页，高亮"窝囊费"Tab
- AC-202
  - Given Tab切换
  - When 页面切换
  - Then 切换动画时长为200ms淡入淡出

## 7. 埋点需求

| 事件                 | 触发条件        |
| ------------------ | ----------- |
| tab\_switch        | 用户点击Tab切换页面 |
| logo\_click        | 用户点击Logo区域  |
| avatar\_menu\_open | 用户点击头像展开菜单  |
| logout\_start      | 用户点击"下班跑路"  |
| logout\_confirm    | 用户确认登出      |
| logout\_cancel     | 用户取消登出      |

***

## 8. 非功能需求

### 性能

- Tab切换响应时间 ≤ 200ms
- 页面切换动画流畅，帧率 ≥ 30fps

### 可用性

- Tab切换成功率 ≥ 99.9%
- 导航栏在所有功能页面均正常显示

### 兼容性

- 支持主流浏览器（Chrome、Firefox、Edge、Safari）
