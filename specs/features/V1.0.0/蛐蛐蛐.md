# 蛐蛐蛐（聊天系统）PRD

## 1. 背景与目标

### 为什么做

职场小团体需要一个安全、无痕的情绪宣泄渠道。传统的聊天工具（微信、钉钉）消息会永久留存，用户担心被截图或被领导看到。蛐蛐蛐提供"阅后即焚"的即时聊天体验，消息在30分钟后自动物理删除，让用户可以放心吐槽。

### 用户价值

*   消息5分钟后自动销毁，无任何历史记录，彻底消除"被抓包"的顾虑
*   提供快捷常用语，一键发送职场吐槽，降低表达门槛
*   实时倒计时显示消息剩余存活时间，增强紧迫感和仪式感

### 成功标准

*   消息发送延迟 < 1秒
*   消息在30分钟后100%从服务器物理删除
*   用户可在聊天中看到实时倒计时

## 2. 范围定义

### 本次做

*   纯文字消息发送与接收
*   消息30分钟自动销毁（客户端过滤+服务器物理删除）
*   消息实时倒计时显示
*   快捷常用语一键发送
*   消息字数限制500字符
*   发送者/接收者消息气泡区分（自己靠右，他人靠左）

### 本次不做

*   图片、语音、视频等富媒体消息
*   消息撤回功能
*   消息转发功能
*   聊天记录搜索
*   @提及功能
*   消息已读状态

## 5. 功能详细设计

### 5.1 消息列表展示

#### 功能说明

功能目标：展示当前鱼圈的实时聊天消息，消息按时间顺序排列，新消息自动滚动到底部

#### 页面设计

功能入口：导航栏"蛐蛐间"Tab

页面组成：

*   顶部信息栏：鱼圈名称 + "蛐蛐间"标题 + "5MIN 瞬时阅后即焚"标签 + "正在划水中"状态
*   消息列表区域：展示消息气泡，按时间倒序排列
*   快捷常用语栏：横向滚动的快捷按钮
*   输入区域：文字输入框 + 发送按钮

#### 组件设计

| 组件 | 类型 | 说明 |
| ---- | ---- | ---- |
| 消息气泡 | Card | 包含头像、昵称、倒计时、消息内容 |
| 头像展示 | Avatar | 40x40px，圆角，显示用户选择的emoji头像 |
| 倒计时标签 | Badge | 红色背景，显示"X分X秒后毁灭"，每秒更新 |
| 空状态提示 | EmptyState | 无消息时显示"寂静无声的秘密基地"提示 |

#### 字段设计

| 字段 | 类型 | 必填 | 默认值 | 规则 | 说明 |
| ---- | ---- | ---- | ---- | ---- | ---- |
| circleId | string | 是 | - | - | 所属鱼圈ID |
| authorId | string | 是 | - | - | 发送者ID |
| authorName | string | 是 | - | - | 发送者昵称 |
| authorAvatar | string | 是 | - | - | 发送者头像emoji |
| text | string | 是 | - | 最大500字符 | 消息内容 |
| createdAt | Timestamp | 是 | - | - | 发送时间 |

#### 交互设计

| 操作 | 反馈 |
| ---- | ---- |
| 新消息到达 | 自动滚动到消息底部 |
| 消息超过5分钟 | 从列表中移除，并从服务器物理删除 |
| 无消息时 | 显示空状态插图和提示文字 |

#### 业务规则

| 编号 | 规则 |
| ---- | ---- |
| BR-001 | 消息按时间正序排列（最早的在上方） |
| BR-002 | 新消息自动滚动到列表底部 |
| BR-003 | 自己的消息靠右对齐，他人消息靠左对齐 |
| BR-004 | 自己消息气泡背景为浅橙色，他人消息气泡背景为白色 |

#### 异常处理

| 场景 | 处理方式 |
| ---- | ---- |
| 消息加载失败 | 控制台输出错误，不影响页面展示 |
| 消息删除失败 | 静默处理（可能已被其他客户端删除） |

### 5.2 消息发送

#### 功能说明

功能目标：让用户输入文字并发送到当前鱼圈

#### 页面设计

功能入口：聊天室底部输入区域

页面组成：

*   文字输入框：placeholder: "打字蛐蛐同事领导...🤫 单条消息会在5分钟后物理火化销毁哦"
*   字数统计：右下角显示"X/500"，超过450字变红
*   发送按钮：橙色背景，显示"碎碎念"，含发送图标

#### 交互设计

| 操作 | 反馈 |
| ---- | ---- |
| 输入文字 | 实时更新字数统计 |
| 字数超过450 | 字数统计变红 |
| 字数达到500 | 无法继续输入 |
| 点击"碎碎念" | 清空输入框，消息发送到服务器 |
| 输入框为空 | 发送按钮禁用 |
| 发送中 | 发送按钮禁用 |
| 发送失败 | 恢复输入框内容，显示错误 |

#### 状态流转

| 状态 | 说明 |
| ---- | ---- |
| 待输入 | 输入框为空，发送按钮禁用 |
| 输入中 | 用户正在输入，显示字数统计 |
| 发送中 | 点击发送后的加载状态 |
| 成功 | 消息出现在列表中，输入框清空 |
| 失败 | 恢复输入内容，显示错误提示 |

#### 业务规则

| 编号 | 规则 |
| ---- | ---- |
| BR-005 | 单条消息最大500字符 |
| BR-006 | 空白消息不可发送 |
| BR-007 | 发送成功后清空输入框和字数统计 |
| BR-008 | 发送失败时恢复输入框内容 |

#### 异常处理

| 场景 | 处理方式 |
| ---- | ---- |
| 发送失败 | 恢复输入框内容，控制台输出错误 |
| 网络异常 | 静默处理，消息不丢失 |

### 5.4 消息自动销毁

#### 功能说明

功能目标：确保消息在30分钟后从服务器物理删除，不留任何痕迹

#### 处理流程

1.  客户端实时监听消息列表
2.  计算每条消息的创建时间与当前时间的差值
3.  超过30分钟（18000秒）的消息：
    *   从客户端显示列表中移除
    *   从服务器物理删除文档
4.  每条消息显示实时倒计时："X分X秒后毁灭"

#### 业务规则

| 编号 | 规则 |
| ---- | ---- |
| BR-011 | 消息生命周期为30分钟（18000秒） |
| BR-012 | 超过30分钟的消息从服务器物理删除（deleteDoc） |
| BR-013 | 倒计时每秒更新一次 |
| BR-014 | 倒计时显示格式为"🔥 X分X秒后毁灭" |
| BR-015 | 消息销毁时显示"物理销毁中..." |

#### 异常处理

| 场景 | 处理方式 |
| ---- | ---- |
| 删除失败 | 静默处理，可能已被其他客户端删除 |
| 时间计算异常 | 默认显示"刚刚" |

## 6. 验收标准

### Happy Path

*   AC-001

    *   Given 用户在鱼圈聊天室
    *   When 用户输入文字并点击"碎碎念"发送
    *   Then 消息出现在列表底部，显示用户头像、昵称、倒计时和消息内容

*   AC-002

    *   Given 用户在鱼圈聊天室
    *   When 其他成员发送新消息
    *   Then 消息实时出现在列表底部，自动滚动到最新位置

*   AC-003

    *   Given 用户在鱼圈聊天室
    *   When 点击任意快捷常用语按钮
    *   Then 该条消息立即发送到聊天室

*   AC-004

    *   Given 聊天室中有消息
    *   When 消息创建时间超过30分钟
    *   Then 消息从列表中消失，并从服务器物理删除

### Edge Case

*   AC-101

    *   Given 用户正在输入消息
    *   When 字数达到500
    *   Then 无法继续输入

*   AC-102

    *   Given 用户正在输入消息
    *   When 字数超过450
    *   Then 字数统计显示为红色

*   AC-103

    *   Given 用户尝试发送消息
    *   When 输入框为空
    *   Then 发送按钮禁用，无法发送

*   AC-104

    *   Given 用户尝试发送消息
    *   When 网络连接异常导致发送失败
    *   Then 输入框恢复发送前的内容，控制台输出错误

*   AC-105

    *   Given 聊天室中暂无消息
    *   When 用户进入聊天室
    *   Then 显示空状态插图"寂静无声的秘密基地"及提示文字

### Business Rule

*   AC-201

    *   Given 消息已创建
    *   When 计算消息存活时间
    *   Then 倒计时每秒更新，格式为"🔥 X分X秒后毁灭"

*   AC-202

    *   Given 消息已创建超过5分钟
    *   When 系统检测到过期消息
    *   Then 从服务器物理删除该消息文档（deleteDoc）

*   AC-203

    *   Given 用户发送消息
    *   When 消息内容为空白
    *   Then 发送失败，提示"内容超长，碎碎念不可超过500字！"

*   AC-204

    *   Given 自己发送的消息和他人发送的消息
    *   When 在消息列表中展示
    *   Then 自己的消息靠右对齐、浅橙色背景，他人消息靠左对齐、白色背景

## 8. 非功能需求

### 性能

*   消息发送延迟 < 1秒
*   消息接收延迟 < 1秒
*   倒计时更新流畅，无卡顿

### 可用性

*   消息发送成功率 ≥ 99%
*   消息自动销毁准确率 100%

### 安全性

*   消息在服务器端5分钟后物理删除（不可恢复）
*   消息内容不持久化到任何日志系统
