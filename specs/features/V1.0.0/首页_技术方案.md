# 首页 — 技术设计文档

## 1. 设计概要

**功能描述**：实现吐司应用的落地首页，采用全屏滚动设计，包含Hero区域、功能亮点、热梗墙、功能预览四个板块，根据登录状态切换导航栏和CTA跳转目标。

**影响范围**：前端路由模块、首页模块、导航栏组件、认证模块

**技术难点**：全屏滚动交互、弹幕式热梗墙动画、Lottie动画懒加载、首屏性能优化

**外部依赖**：热梗墙语录接口（后端新增，前端保留默认预设兜底）

## 5. 核心逻辑

### 5.1 全屏滚动实现 → AC-001, AC-002, AC-003, AC-004

**触发条件**：用户滚动鼠标滚轮或点击导航指示器

**处理流程**：
1. 监听 `wheel` 事件，判断滚动方向
2. 使用 `scrollIntoView({ behavior: 'smooth' })` 或 CSS `scroll-snap` 实现平滑滚动
3. 更新当前活跃板块索引
4. 导航指示器高亮对应板块

**实现方案选择**：

| 方案 | 优点 | 缺点 |
|------|------|------|
| CSS scroll-snap | 原生支持，性能好，无需JS | 兼容性略差，自定义程度有限 |
| JS手动控制 | 完全可控，兼容性好 | 代码量大，需要处理边界情况 |
| react-fullpage库 | 开箱即用，功能完整 | 增加依赖包体积 |

**结论**：选择CSS scroll-snap方案，原生性能最佳，且现代浏览器支持良好。

**伪代码**：
```css
/* 全屏滚动容器 */
.fullscreen-container {
  height: 100vh;
  overflow-y: scroll;
  scroll-snap-type: y mandatory;
}

/* 每个板块 */
.section {
  height: 100vh;
  scroll-snap-align: start;
}
```

### 5.2 弹幕式热梗墙 → AC-003

**触发条件**：页面滚动到热梗墙板块时开始播放

**处理流程**：
1. 组件挂载时获取语录数据（接口或兜底数据）
2. 将语录分配到不同轨道（垂直位置）
3. 使用CSS animation实现从右向左滚动
4. 每条语录速度略有差异（8-15秒完成滚动）
5. 动画结束后重新开始，形成循环

**伪代码**：
```typescript
// 语录配置
interface MemeQuote {
  id: number;
  text: string;
  speed: number;      // 8-15秒
  track: number;      // 0-4，对应5个垂直轨道
  delay: number;      // 动画延迟，错开启动时间
}

// 分配轨道和速度
const assignTrackAndSpeed = (quotes: string[]): MemeQuote[] => {
  return quotes.map((text, index) => ({
    id: index,
    text,
    speed: 8 + Math.random() * 7,  // 8-15秒
    track: index % 5,               // 5个轨道循环
    delay: Math.random() * 3,       // 0-3秒随机延迟
  }));
};
```

**CSS动画**：
```css
@keyframes danmaku-scroll {
  from { transform: translateX(100%); }
  to { transform: translateX(-100%); }
}

.danmaku-item {
  animation: danmaku-scroll var(--speed) linear var(--delay) infinite;
}
```

### 5.3 功能预览Tab切换 → AC-004, AC-005

**触发条件**：用户点击Tab或自动轮播

**处理流程**：
1. 默认选中"蛐蛐蛐"Tab
2. 点击Tab切换对应Lottie动画
3. 动画文件懒加载，首屏不加载
4. 切换时使用淡入淡出过渡动画

### 5.4 CTA按钮跳转逻辑 → AC-006, AC-007

**触发条件**：用户点击CTA按钮

**处理流程**：
1. 检查用户登录状态（通过useAuth Hook）
2. 未登录 → 跳转 `/login`
3. 已登录 → 跳转 `/salary`

**伪代码**：
```typescript
const handleCTAClick = () => {
  if (isLoggedIn) {
    navigate('/salary');
  } else {
    navigate('/login');
  }
};
```

**处理流程**：

2. 或触发全屏滚动组件滚动到第一个section

## 7. 技术决策

### 7.1 全屏滚动方案选择

**背景**：需要实现整页滚动，每个板块占满一屏

**选项**：
- A: CSS scroll-snap — 原生CSS方案，性能最佳，代码量少
- B: react-fullpage 第三方库 — 功能完整，但增加依赖
- C: 手动JS控制 — 完全可控，但代码复杂

**结论**：选择方案A，CSS scroll-snap在现代浏览器支持良好，性能最优，且无需引入额外依赖。对于不支持的浏览器（极少数），降级为普通滚动。

### 7.2 弹幕实现方案

**背景**：热梗墙需要实现弹幕式滚动效果

**选项**：
- A: CSS animation — 纯CSS实现，性能好，但控制粒度有限
- B: Canvas绘制 — 性能好，可处理大量弹幕，但实现复杂
- C: requestAnimationFrame — JS控制，灵活但需要手动优化

**结论**：选择方案A，20-30条语录数量较少，CSS animation完全够用，且代码简洁易维护。

### 7.3 Lottie动画方案

**背景**：功能预览需要展示动画效果

**选项**：
- A: Lottie + lottie-react — 业界标准，文件小，支持交互
- B: GIF动图 — 兼容性好，但文件大，无法交互
- C: CSS/SVG动画 — 无需额外依赖，但复杂动画难以实现

**结论**：选择方案A，Lottie是动画展示的行业标准，文件体积小，且支持暂停、循环等控制。使用动态import实现懒加载。

## 8. 安全与性能

**性能考量**：
- 首屏（Hero区域）优先加载，其他板块懒加载
- Lottie动画文件使用动态import，不阻塞首屏渲染
- 图片资源使用WebP格式，减小体积
- 弹幕动画使用CSS transform，触发GPU加速
- 使用 `loading="lazy"` 属性懒加载图片

**缓存策略**：
- 热梗墙语录接口设置Cache-Control，缓存1小时
- Lottie动画文件设置长期缓存（文件名带hash）
