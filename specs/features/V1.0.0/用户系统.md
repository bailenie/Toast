# 用户系统 PRD

## 1. 背景与目标

### 为什么做

摸鱼圈是一个面向职场小团体的安全情绪释放空间。用户系统是整个应用的基础，负责用户身份的注册、登录、登出以及个人资料管理。没有稳定的用户系统，其他功能（聊天、摸鱼游戏、窝囊费）均无法正常运作。

### 用户价值

*   提供简洁的邮箱+密码注册登录方式，无需复杂的手机号验证
*   注册时即可选择个性化昵称和萌系头像，增强归属感
*   自动创建个人专属私有鱼圈，确保每个用户都有独立空间

### 成功标准

*   注册到首次进入主页的流程可在 30 秒内完成
*   登录成功率达到 99% 以上
*   用户资料在注册后可正常加载和展示

---

## 2. 范围定义

### 本次做

*   邮箱+密码注册（含昵称、头像选择）
*   邮箱+密码登录
*   登出功能
*   用户资料加载与展示（含导航栏头像显示）
*   用户资料编辑（头像、昵称、个人简介）
*   修改密码
*   被禁用用户的封禁提示页
*   注册时自动创建私有鱼圈和摸鱼统计占位文档

### 本次不做

*   手机号注册/验证码登录
*   第三方登录（微信、Google 等）
*   忘记密码/重置密码流程
*   用户注销/账号删除

## 4. 页面结构

| 页面 | 入口 | 目标 |
| ---- | ---- | ---- |
| 登录页 | 应用首页（未登录状态） | 输入邮箱密码进行登录 |
| 注册页 | 登录页底部切换链接 | 填写邮箱、密码、昵称、选择头像进行注册 |

## 5. 功能详细设计

### 5.1 注册功能

#### 功能说明

功能目标：让新用户通过邮箱+密码注册账号，同时设置昵称和选择头像

#### 页面设计

功能入口：登录页底部"没有注册过工号？注册一个带薪化身账号"链接

页面组成：

*   标题区域：显示"册就任工号 (注册账号)"及副标题
*   邮箱输入框
*   密码输入框（不少于6位）
*   昵称输入框（注册时展开）
*   头像选择网格（注册时展开）
*   提交按钮："创建账号并滑入避难所"
*   底部切换链接

#### 组件设计

| 组件 | 类型 | 说明 |
| ---- | ---- | ---- |
| 邮箱输入框 | Input[type=email] | 必填，placeholder: example@yourco.com |
| 密码输入框 | Input[type=password] | 必填，最少6位，placeholder: •••••• |
| 昵称输入框 | Input[type=text] | 注册时必填，最大40字符，placeholder: 例: 第五工室如厕家 / 下班预言师 |
| 头像选择器 | Grid[4列] | 展示12个萌系头像，点击选中，选中状态高亮 |
| 提交按钮 | Button | 注册时文案"创建账号并滑入避难所 🦦"，加载中显示"飞速处理中..." |
| 错误提示 | Alert | 红色背景，含图标，展示具体错误信息 |

#### 字段设计

| 字段 | 类型 | 必填 | 默认值 | 规则 | 说明 |
| ---- | ---- | ---- | ---- | ---- | ---- |
| email | string | 是 | - | 有效邮箱格式 | 注册/绑定的邮箱账号 |
| password | string | 是 | - | 不少于6位 | 登录邮箱密码 |
| nickname | string | 是（注册时） | - | 最大40字符 | 摸鱼专属花名 |
| avatar | string | 是（注册时） | moyu_otter | 从预设列表选择 | 头像ID |

#### 交互设计

| 操作 | 反馈 |
| ---- | ---- |
| 点击"创建账号并滑入避难所" | 按钮显示"飞速处理中..."，禁用按钮 |
| 注册成功 | 自动进入主页 |
| 邮箱已被使用 | 显示"该邮箱已在职场划水中！请尝试直接登录。" |
| 昵称为空 | 显示"注册需要填写一个萌新新昵称哦~" |
| 网络异常 | 显示"认证失败：密码过短或网络超时" |

#### 状态流转

| 状态 | 说明 |
| ---- | ---- |
| 待填写 | 初始状态，表单为空 |
| 填写中 | 用户正在输入信息 |
| 提交中 | 点击注册按钮后的加载状态 |
| 成功 | 跳转至主页 |
| 失败 | 显示错误信息，保持在注册页 |

状态流转：

待填写 → 填写中 → 提交中 → 成功/失败

#### 输入输出

输入：

| 项 | 说明 |
| ---- | ---- |
| 邮箱 | 用户邮箱地址 |
| 密码 | 不少于6位的密码 |
| 昵称 | 最大40字符的花名 |
| 头像ID | 从12个预设头像中选择 |

输出：

| 项 | 说明 |
| ---- | ---- |
| 用户文档 | Firestore users/{uid} 文档 |
| 私有鱼圈 | Firestore circles/private_{uid} 文档 |
| 摸鱼统计 | Firestore circles/private_{uid}/moyu_stats/{uid} 文档 |

#### 业务规则

| 编号 | 规则 |
| ---- | ---- |
| BR-001 | 注册时自动创建名为"我的专属安全水箱 🐠"的私有鱼圈，邀请码为"000000" |
| BR-002 | 私有鱼圈的宠物鱼默认为"懵懂胖金鱼"，等级1，经验0，品类"肥嘟嘟胖金鱼" |
| BR-003 | 用户默认工资250元/天，上班时间09:00，下班时间18:00 |
| BR-004 | 同一邮箱不可重复注册 |
| BR-005 | 昵称最大长度为40字符 |

#### 异常处理

| 场景 | 处理方式 |
| ---- | ---- |
| 邮箱已存在 | 提示"该邮箱已在职场划水中！请尝试直接登录。" |
| 密码过短 | 提示"认证失败：密码过短或网络超时" |
| 网络超时 | 提示"认证失败：密码过短或网络超时" |
| 用户未找到 | 提示"找不到该雇员信息，请确认邮箱或切换为注册页面！" |

### 5.5 个人资料编辑功能

#### 功能说明

功能目标：让已登录用户修改头像、昵称和个人简介，保持个人形象更新

#### 页面设计

功能入口：导航栏右上角头像 → 下拉菜单 → "个人资料"

页面组成：

*   居中弹窗，标题"编辑个人资料"
*   头像选择区域：12 个预设头像网格，当前头像预选中
*   昵称输入框：显示当前昵称，可修改
*   个人简介输入框：显示当前简介（可为空），可修改
*   底部按钮："保存"和"取消"

#### 组件设计

| 组件 | 类型 | 说明 |
| ---- | ---- | ---- |
| 头像选择器 | Grid[4列] | 展示12个萌系头像，当前头像预选中高亮，点击切换 |
| 昵称输入框 | Input[type=text] | 必填，最大40字符，预填当前昵称 |
| 个人简介输入框 | Textarea | 选填，最大200字符，预填当前简介 |
| 保存按钮 | Button | 文案"保存修改"，加载中显示"飞速处理中..." |
| 取消按钮 | Button | 文案"取消"，关闭弹窗不保存 |

#### 字段设计

| 字段 | 类型 | 必填 | 默认值 | 规则 | 说明 |
| ---- | ---- | ---- | ---- | ---- | ---- |
| avatar | string | 是 | 当前值 | 从预设列表选择 | 头像ID |
| nickname | string | 是 | 当前值 | 最大40字符 | 摸鱼专属花名 |
| bio | string | 否 | 空字符串 | 最大200字符 | 个人简介/签名 |

#### 交互设计

| 操作 | 反馈 |
| ---- | ---- |
| 打开弹窗 | 加载当前头像、昵称、简介，头像预选中 |
| 点击"保存修改" | 按钮显示"飞速处理中..."，禁用按钮 |
| 保存成功 | 弹窗关闭，导航栏头像和昵称即时刷新 |
| 昵称为空时保存 | 显示"昵称不能为空，给职场花名留个位置~" |
| 网络异常 | 显示"保存失败，请稍后重试" |
| 点击"取消"或遮罩层或 Esc | 关闭弹窗，不保存修改 |

#### 状态流转

| 状态 | 说明 |
| ---- | ---- |
| 待编辑 | 弹窗打开，显示当前资料 |
| 编辑中 | 用户修改了任意字段 |
| 提交中 | 点击保存后的加载状态 |
| 成功 | 弹窗关闭，页面资料刷新 |
| 失败 | 显示错误信息，保持在弹窗中 |

状态流转：

待编辑 → 编辑中 → 提交中 → 成功/失败

#### 业务规则

| 编号 | 规则 |
| ---- | ---- |
| BR-012 | 昵称不能为空，最大 40 字符 |
| BR-013 | 个人简介选填，最大 200 字符，默认为空 |
| BR-014 | 保存成功后，导航栏头像和页面中所有显示用户信息的位置需即时刷新 |

#### 异常处理

| 场景 | 处理方式 |
| ---- | ---- |
| 昵称为空 | 提示"昵称不能为空，给职场花名留个位置~"，保存失败 |
| 网络异常 | 提示"保存失败，请稍后重试" |

### 5.6 修改密码功能

#### 功能说明

功能目标：让已登录用户修改登录密码，保障账号安全

#### 页面设计

功能入口：导航栏右上角头像 → 下拉菜单 → "修改密码"

页面组成：

*   居中弹窗，标题"修改密码"
*   旧密码输入框
*   新密码输入框（不少于6位）
*   确认新密码输入框
*   底部按钮："确认修改"和"取消"

#### 组件设计

| 组件 | 类型 | 说明 |
| ---- | ---- | ---- |
| 旧密码输入框 | Input[type=password] | 必填，placeholder: •••••• |
| 新密码输入框 | Input[type=password] | 必填，最少6位，placeholder: •••••• |
| 确认新密码输入框 | Input[type=password] | 必填，需与新密码一致，placeholder: •••••• |
| 确认按钮 | Button | 文案"确认修改"，加载中显示"飞速处理中..." |
| 取消按钮 | Button | 文案"取消" |

#### 字段设计

| 字段 | 类型 | 必填 | 规则 | 说明 |
| ---- | ---- | ---- | ---- | ---- |
| oldPassword | string | 是 | 当前使用的密码 | 用于验证身份 |
| newPassword | string | 是 | 不少于6位 | 新密码 |
| confirmPassword | string | 是 | 必须与 newPassword 一致 | 确认新密码 |

#### 交互设计

| 操作 | 反馈 |
| ---- | ---- |
| 点击"确认修改" | 校验三个字段，通过后按钮显示"飞速处理中..." |
| 修改成功 | 清除本地认证状态，跳转到登录页，用户需用新密码重新登录 |
| 旧密码错误 | 显示"原密码输入有误，请核实后再试！" |
| 新密码少于6位 | 显示"新密码不少于6位" |
| 两次新密码不一致 | 显示"两次输入的新密码不一致" |
| 点击"取消"或遮罩层或 Esc | 关闭弹窗 |
| 网络异常 | 显示"修改失败，请稍后重试" |

#### 状态流转

| 状态 | 说明 |
| ---- | ---- |
| 待填写 | 弹窗打开，三个输入框为空 |
| 填写中 | 用户正在输入 |
| 提交中 | 点击确认后的加载状态 |
| 成功 | 清除认证，跳转登录页 |
| 失败 | 显示错误信息，保持在弹窗中 |

状态流转：

待填写 → 填写中 → 提交中 → 成功/失败

#### 业务规则

| 编号 | 规则 |
| ---- | ---- |
| BR-015 | 修改密码必须验证旧密码正确 |
| BR-016 | 新密码不少于 6 位 |
| BR-017 | 修改密码成功后清除本地认证状态，强制跳转登录页 |
| BR-018 | 修改密码成功后，旧密码立即失效 |

#### 异常处理

| 场景 | 处理方式 |
| ---- | ---- |
| 旧密码错误 | 提示"原密码输入有误，请核实后再试！" |
| 新密码少于6位 | 提示"新密码不少于6位" |
| 两次新密码不一致 | 提示"两次输入的新密码不一致" |
| 网络异常 | 提示"修改失败，请稍后重试" |

## 6. 验收标准

### Happy Path

*   AC-001

    *   Given 用户未注册过摸鱼圈账号
    *   When 用户填写有效邮箱、密码(≥6位)、昵称，选择头像，点击注册
    *   Then 账号创建成功，自动登录并进入主页，私有鱼圈自动创建

*   AC-002

    *   Given 用户已注册摸鱼圈账号
    *   When 用户输入正确邮箱和密码，点击登录
    *   Then 登录成功，进入主页，显示用户资料和当前鱼圈信息

*   AC-003

    *   Given 用户已登录
    *   When 用户点击头像下拉菜单中的"退出登录"，并在确认框中点击确认
    *   Then 用户成功登出，返回登录页

*   AC-004

    *   Given 用户已登录
    *   When 点击导航栏右上角头像
    *   Then 弹出下拉菜单，显示"个人资料"、"修改密码"、"退出登录"三个选项

*   AC-005

    *   Given 用户点击下拉菜单中的"个人资料"
    *   When 弹窗打开
    *   Then 显示当前头像已选中、当前昵称和简介已预填

*   AC-006

    *   Given 用户在个人资料弹窗中修改了头像/昵称/简介
    *   When 点击"保存修改"
    *   Then 资料更新成功，弹窗关闭，导航栏头像和昵称即时刷新

*   AC-007

    *   Given 用户点击下拉菜单中的"修改密码"
    *   When 弹窗打开
    *   Then 显示旧密码、新密码、确认新密码三个输入框

*   AC-008

    *   Given 用户在修改密码弹窗中输入正确旧密码、有效新密码（≥6位）和匹配的确认密码
    *   When 点击"确认修改"
    *   Then 密码修改成功，清除本地认证状态，跳转到登录页

### Edge Case

*   AC-101

    *   Given 用户尝试注册
    *   When 输入的邮箱已被其他用户注册
    *   Then 显示错误提示"该邮箱已在职场划水中！请尝试直接登录。"，注册失败

*   AC-102

    *   Given 用户尝试注册
    *   When 未填写昵称直接提交
    *   Then 显示错误提示"注册需要填写一个萌新新昵称哦~"，注册失败

*   AC-103

    *   Given 用户尝试登录
    *   When 输入的密码错误
    *   Then 显示错误提示"密码输入有误，请核实后再敲门！"，登录失败

*   AC-104

    *   Given 用户尝试登录
    *   When 输入的邮箱未注册
    *   Then 显示错误提示"找不到该雇员信息，请确认邮箱或切换为注册页面！"

*   AC-105

    *   Given 被封禁用户尝试登录
    *   When 输入正确邮箱和密码
    *   Then 登录成功但显示封禁提示页"你已被管理员关进【冷冻鱼缸】！"，无法访问其他功能

*   AC-106

    *   Given 用户尝试注册
    *   When 网络连接异常
    *   Then 显示错误提示"认证失败：密码过短或网络超时"

*   AC-107

    *   Given 用户在个人资料弹窗中
    *   When 清空昵称后点击保存
    *   Then 提示"昵称不能为空，给职场花名留个位置~"，保存失败

*   AC-108

    *   Given 用户在修改密码弹窗中
    *   When 输入错误的旧密码后点击确认
    *   Then 提示"原密码输入有误，请核实后再试！"，修改失败

*   AC-109

    *   Given 用户在修改密码弹窗中
    *   When 新密码少于 6 位
    *   Then 提示"新密码不少于6位"

*   AC-110

    *   Given 用户在修改密码弹窗中
    *   When 两次输入的新密码不一致
    *   Then 提示"两次输入的新密码不一致"

*   AC-111

    *   Given 用户保存个人资料或修改密码
    *   When 网络异常
    *   Then 提示"保存失败，请稍后重试"或"修改失败，请稍后重试"

*   AC-112

    *   Given 用户在个人资料弹窗中
    *   When 个人简介超过 200 字符
    *   Then 输入框限制无法继续输入

*   AC-113

    *   Given 用户点击头像弹出下拉菜单
    *   When 点击菜单外部区域
    *   Then 下拉菜单关闭

### Business Rule

*   AC-201

    *   Given 新用户注册成功
    *   When 系统创建用户资料
    *   Then 自动创建名为"我的专属安全水箱 🐠"的私有鱼圈，邀请码为"000000"，宠物鱼为"懵懂胖金鱼"等级1

*   AC-202

    *   Given 新用户注册成功
    *   When 系统创建用户资料
    *   Then 用户默认工资250元/天，上班时间09:00，下班时间18:00

*   AC-203

    *   Given 用户尝试注册
    *   When 密码少于6位
    *   Then 注册失败，提示认证失败

*   AC-204

    *   Given 用户尝试注册
    *   When 昵称超过40字符
    *   Then 输入框限制无法继续输入

*   AC-205

    *   Given 用户编辑个人资料
    *   When 简介为空时保存
    *   Then 保存成功，简介字段留空（选填）

*   AC-206

    *   Given 用户修改密码成功
    *   When 跳转到登录页
    *   Then 本地认证状态已清除，原密码失效，必须用新密码登录

### 性能

*   注册/登录请求响应时间 ≤ 3秒
*   登录后用户资料加载 ≤ 2秒

### 可用性

*   注册成功率 ≥ 99%
*   登录成功率 ≥ 99.5%

*   密码传输必须加密（HTTPS）
*   密码存储必须加密（Firebase Auth 自动处理）
