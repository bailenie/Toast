> 更新时间：2026.06.18

## 2. 关键文件说明 (Key Files)

### 配置文件

| 文件                   | 路径                            | 用途                                                  |
| -------------------- | ----------------------------- | --------------------------------------------------- |
| `vite.config.ts`     | `client/vite.config.ts`       | Vite 构建配置：端口 3000、React 插件、Tailwind CSS 插件、代理后端 API |
| `tsconfig.json` (前端) | `client/tsconfig.json`        | 前端 TypeScript 编译配置：严格模式、JSX、路径别名                    |
| `tsconfig.json` (后端) | `server/tsconfig.json`        | 后端 TypeScript 编译配置：严格模式、Node 目标                     |
| `package.json` (前端)  | `client/package.json`         | 前端依赖声明与脚本命令                                         |
| `package.json` (后端)  | `server/package.json`         | 后端依赖声明与脚本命令                                         |
| `schema.prisma`      | `server/prisma/schema.prisma` | Prisma 数据库模型定义（User、Circle、Card、Salary 等）           |
| `.env.example`       | `.env.example`                | 环境变量模板，说明前后端所需的所有环境变量                               |

### 入口文件

| 文件           | 路径                    | 用途                                               |
| ------------ | --------------------- | ------------------------------------------------ |
| `index.html` | `client/index.html`   | 前端 HTML 入口，挂载 React 根节点                          |
| `main.tsx`   | `client/src/main.tsx` | React 应用入口：挂载 App 组件、引入全局样式                      |
| `App.tsx`    | `client/src/App.tsx`  | 主应用组件：路由配置、全局 Provider（Auth、Socket）、老板键监听        |
| `index.ts`   | `server/src/index.ts` | 后端入口：Express 应用初始化、中间件注册、路由挂载、Socket.io 启动、数据库连接 |

### 核心业务文件

| 文件           | 路径                           | 用途                                       |
| ------------ | ---------------------------- | ---------------------------------------- |
| `api.ts`     | `client/src/utils/api.ts`    | Axios 实例封装：统一 baseURL、Token 注入、响应拦截、错误处理 |
| `socket.ts`  | `client/src/utils/socket.ts` | Socket.io 客户端实例：连接管理、自动重连、事件订阅           |
| `types.ts`   | `client/src/types.ts`        | 全局 TypeScript 类型定义：用户、鱼圈、消息、卡片、窝囊费等接口类型  |
| `avatars.ts` | `client/src/avatars.ts`      | 预置头像数据：头像 ID 与图片路径映射                     |

## 3. 模块说明 (Module Description)

### 3.1 Auth - 用户认证模块

| 维度        | 说明                                                            |
| --------- | ------------------------------------------------------------- |
| **前端目录**  | `client/src/components/auth/` + `client/src/hooks/useAuth.ts` |
| **后端路由**  | `server/src/routes/auth.routes.ts`                            |
| **后端服务**  | `server/src/services/auth.service.ts`                         |
| **后端中间件** | `server/src/middleware/auth.ts`                               |
| **职责**    | 用户注册（邮箱 + 密码）、登录、JWT Token 签发/刷新/验证、路由守卫、密码加密（bcrypt）         |
| **数据模型**  | `User`（Prisma schema）                                         |

### 3.2 Chat - 蛐蛐蛐聊天模块

| 维度            | 说明                                                                                             |
| ------------- | ---------------------------------------------------------------------------------------------- |
| **前端目录**      | `client/src/components/chat/` + `client/src/hooks/useChat.ts` + `client/src/hooks/useDraft.ts` |
| **后端路由**      | `server/src/routes/chat.routes.ts`                                                             |
| **后端服务**      | `server/src/services/chat.service.ts`                                                          |
| **Socket 事件** | `server/src/socket/index.ts`                                                                   |
| **职责**        | 实时文字消息发送/接收、单条最大 500 字符、消息 5 分钟自动销毁、无历史记录、聊天草稿保存与恢复                                            |
| **通信方式**      | Socket.io（消息收发为实时事件，销毁定时器在服务端执行）                                                               |

### 3.3 Game - 摸鱼鱼游戏模块

| 维度       | 说明                                                              |
| -------- | --------------------------------------------------------------- |
| **前端目录** | `client/src/components/game/` + `client/src/hooks/useMoyu.ts`   |
| **后端路由** | `server/src/routes/game.routes.ts`                              |
| **后端服务** | `server/src/services/game.service.ts`                           |
| **职责**   | 每日摸鱼次数随机（1\~50 次）、每次 UNO 卡片掉落（1\~2 张）、108 张 UNO 卡片收集进度、宠物鱼养成与展示 |
| **数据模型** | `FishCircle`（宠物鱼状态）、`UserCard`（用户卡片持有）、`DailyMoyu`（每日摸鱼记录）      |

### 3.4 Salary - 窝囊费模块

| 维度       | 说明                                                                |
| -------- | ----------------------------------------------------------------- |
| **前端目录** | `client/src/components/salary/` + `client/src/hooks/useSalary.ts` |
| **后端路由** | `server/src/routes/salary.routes.ts`                              |
| **后端服务** | `server/src/services/salary.service.ts`                           |
| **职责**   | 用户自定义上班/下班时间与单日工资、实时计算"今日已赚窝囊费"、时间价值可视化展示                         |
| **计算逻辑** | 前端定时器每秒更新；后端持久化配置参数，首次加载时返回基准数据                                   |

### 3.5 Circle - 鱼圈管理模块

| 维度       | 说明                                                                |
| -------- | ----------------------------------------------------------------- |
| **前端目录** | `client/src/components/circle/` + `client/src/hooks/useCircle.ts` |
| **后端路由** | `server/src/routes/circle.routes.ts`                              |
| **后端服务** | `server/src/services/circle.service.ts`                           |
| **职责**   | 创建鱼圈、邀请码生成与加入、成员管理（查看/移除）、退出/解散（私有鱼圈不可退出）、鱼圈内人数上限 10 人            |
| **数据模型** | `Circle`（鱼圈）、`CircleMember`（成员关系）                                 |

### 3.6 Boss - 老板键模块

| 维度       | 说明                                                               |
| -------- | ---------------------------------------------------------------- |
| **前端目录** | `client/src/components/boss/` + `client/src/hooks/useBossKey.ts` |
| **后端**   | 无（纯前端功能）                                                         |
| **职责**   | Esc 键全局监听、伪装界面切换（在线表格模板）、页面标题与 Favicon 动态变更、关闭所有弹窗/侧边栏、聊天草稿保留与恢复 |
| **注意**   | 连续快速按 Esc 3 次不触发（防误触），仅页面获得焦点时有效                                 |

### 3.7 Admin - 管理后台模块

| 维度        | 说明                                                        |
| --------- | --------------------------------------------------------- |
| **前端目录**  | `client/src/components/admin/`                            |
| **后端路由**  | `server/src/routes/admin.routes.ts`                       |
| **后端服务**  | `server/src/services/admin.service.ts`                    |
| **后端中间件** | `server/src/middleware/adminAuth.ts`                      |
| **职责**    | 管理员登录、用户管理（查看列表/禁用启用）、鱼圈管理（查看/强制解散）、数据统计概览（用户增长/活跃度/摸鱼数据） |
| **权限控制**  | 通过 `adminAuth` 中间件校验管理员角色，独立于普通用户认证流程                     |

### 3.8 Navbar - 导航栏模块

| 维度        | 说明                                                                                 |
| --------- | ---------------------------------------------------------------------------------- |
| **前端目录**  | `client/src/components/common/Navbar.tsx` + `client/src/components/user/`（下拉菜单子组件） |
| **后端**    | 无（纯前端功能）                                                                           |
| **职责**    | Logo+应用名称展示、4个功能Tab切换（窝囊费/蛐蛐蛐/摸鱼鱼/鱼圈管理）、URL同步、固定顶部布局、用户头像下拉菜单（个人信息/头像选择/登出）、登出确认弹窗 |
| **路由映射**  | `/salary`（默认）、`/chat`、`/game`、`/circle`                                            |
| **依赖**    | `common`（通用组件）、`auth`（认证状态）、`user`（用户信息）                                           |
| **PRD文档** | `specs/features/V1.0.0/导航栏.md`                                                     |

### 3.10 CircleSidebar - 左侧鱼圈栏模块（V1.1.0新增）

| 维度        | 说明                                               |
| --------- | ------------------------------------------------ |
| **前端目录**  | `client/src/components/circle/CircleSidebar.tsx` |
| **后端**    | 无（纯前端功能）                                         |
| **职责**    | 显示所有已加入的鱼圈列表、快速切换鱼圈、创建/加入鱼圈入口、个人功能区（窝囊费）         |

## 4. 文件放置规则 (Placement Rules)

### 4.1 页面/路由级组件

> 放置位置：`client/src/components/<模块名>/`

- 每个业务模块一个子目录，模块主界面组件以模块名命名（如 `ChatRoom.tsx`）
- 页面内使用的子组件放在同一模块目录下（如 `MessageBubble.tsx`）
- 不同模块之间**不允许直接引用**对方目录下的组件，必须通过 Hook 或全局状态通信

### 4.2 业务逻辑

> 前端业务逻辑：`client/src/hooks/`

- 所有跨组件共享的状态管理、API 调用、Socket 事件监听都封装为自定义 Hook
- Hook 命名格式：`use<模块名>`（如 `useChat`、`useMoyu`）
- 单个 Hook 职责单一，不混合多个不相关模块的逻辑

> 后端业务逻辑：`server/src/services/`

- 所有数据库操作、业务规则计算放在 Service 层
- Service 文件命名格式：`<模块名>.service.ts`（如 `chat.service.ts`）
- Route 层只负责参数解析和响应返回，不写业务逻辑

### 4.3 通用组件

> 放置位置：`client/src/components/common/`

- 按钮、卡片、弹窗、提示、加载状态等与业务无关的 UI 组件
- 命名采用 PascalCase，文件名即组件名
- 通用组件**不得引入**任何业务模块的类型或逻辑

### 4.4 工具函数

> 前端工具函数：`client/src/utils/`

- API 封装（`api.ts`）、Socket 实例（`socket.ts`）、Token 管理（`token.ts`）、格式化（`format.ts`）
- 纯函数优先，避免副作用

> 后端工具函数：`server/src/utils/`

- 统一响应格式（`response.ts`）、JWT 操作（`jwt.ts`）、邀请码生成（`inviteCode.ts`）

### 4.5 类型定义

> 前端类型：`client/src/types.ts`

- 所有与后端通信的数据结构类型（接口、枚举）统一定义在此文件
- 按模块分组，添加注释区块分隔
- 组件内部使用的局部类型可定义在组件文件顶部

> 后端类型：Prisma 自动生成（`@prisma/client`）

- 数据库模型类型由 Prisma Schema 定义并自动生成
- Service 层中需要的辅助类型可在对应 Service 文件顶部定义

### 4.6 测试文件

> 测试文件位置：与被测文件同目录，以 `.test.ts` / `.test.tsx` 后缀命名

```
client/src/hooks/useChat.test.ts          # Hook 单元测试
client/src/components/chat/ChatRoom.test.tsx # 组件测试
server/src/services/chat.service.test.ts   # Service 单元测试
server/src/routes/chat.routes.test.ts      # API 集成测试
```

### 4.7 中间件

> 放置位置：`server/src/middleware/`

| 文件                | 职责                                              |
| ----------------- | ----------------------------------------------- |
| `auth.ts`         | 解析 Authorization 头的 JWT，注入 `req.user`，未认证返回 401 |
| `adminAuth.ts`    | 校验 `req.user.role === 'ADMIN'`，非管理员返回 403       |
| `errorHandler.ts` | 捕获所有未处理异常，统一返回错误响应格式                            |
| `validator.ts`    | 使用 Zod 或手动校验请求体/查询参数                            |
| `rateLimiter.ts`  | 接口限流（防刷），基于 IP 或用户 ID                           |

### 4.8 Socket 事件

> 放置位置：`server/src/socket/index.ts`

| 事件方向            | 事件名               | 说明            |
| --------------- | ----------------- | ------------- |
| Client → Server | `join:circle`     | 加入鱼圈聊天房间      |
| Client → Server | `send:message`    | 发送聊天消息        |
| Client → Server | `leave:circle`    | 离开鱼圈聊天房间      |
| Server → Client | `receive:message` | 接收新消息         |
| Server → Client | `destroy:message` | 通知消息已销毁（5分钟后） |
| Server → Client | `user:joined`     | 用户加入房间通知      |
| Server → Client | `user:left`       | 用户离开房间通知      |

## 5. 模块依赖方向 (Module Dependency Direction)

### 前端模块依赖图

```
                        ┌──────────────┐
                        │   common     │  (通用组件+Navbar，被所有模块依赖)
                        └──────┬───────┘
                               │
          ┌────────────────────┼────────────────────┐
          │                    │                    │
          ▼                    ▼                    ▼
   ┌──────────┐        ┌──────────┐         ┌──────────┐
   │   auth   │        │   boss   │         │   admin  │
   │  认证模块 │        │  老板键  │         │ 管理后台  │
   └────┬─────┘        └────┬─────┘         └────┬─────┘
        │                   │                    │
        │         ┌─────────┼─────────┐          │
        │         │         │         │          │
        ▼         ▼         ▼         ▼          │
   ┌──────────┐ ┌──────────┐ ┌──────────┐       │
   │  circle  │ │   chat   │ │   game   │       │
   │ 鱼圈管理  │ │ 蛐蛐蛐  │ │ 摸鱼鱼   │       │
   └──────────┘ └──────────┘ └────┬─────┘       │
        │              │          │              │
        │              ▼          ▼              │
        │         ┌──────────┐ ┌──────────┐     │
        └────────►│  salary  │ │   user   │◄────┘
                    │  窝囊费  │ │ 用户中心  │
                    └──────────┘ └──────────┘
```

### 合法依赖方向（允许）

| 调用方      | 可依赖                      | 说明                           |
| -------- | ------------------------ | ---------------------------- |
| `common` | 无                        | 最底层，不依赖任何业务模块                |
| `auth`   | `common`                 | 认证模块只依赖通用组件                  |
| `boss`   | `common`                 | 老板键只依赖通用组件                   |
| `admin`  | `common`                 | 管理后台只依赖通用组件                  |
| `circle` | `common`、`auth`          | 鱼圈管理需要认证状态                   |
| `chat`   | `common`、`circle`        | 聊天需要知道当前鱼圈                   |
| `game`   | `common`、`circle`        | 游戏需要知道当前鱼圈（宠物鱼为鱼圈级别）         |
| `salary` | `common`、`auth`、`circle` | 窝囊费需要用户身份和鱼圈上下文              |
| `user`   | `common`、`auth`、`game`   | 用户中心需要展示卡片收集（引用 game 的类型/数据） |

### 禁止的依赖方向

| 禁止方向                                            | 原因                         |
| ----------------------------------------------- | -------------------------- |
| `common` → 任何业务模块                               | 通用组件不得引入业务逻辑               |
| `auth` → `chat`/`game`/`salary`/`circle`/`user` | 认证模块不应知道业务模块存在             |
| `chat` → `game`                                 | 聊天与游戏是独立功能，不应耦合            |
| `game` → `chat`                                 | 游戏不应依赖聊天功能                 |
| `boss` → 任何业务模块（除 `common`）                     | 老板键仅做 UI 切换，不应依赖具体业务       |
| `admin` → `chat`/`game`/`salary`/`boss`         | 管理后台通过 API 操作，不应直接引用前端业务组件 |
| 任意模块 → `admin`                                  | 管理后台是独立入口，不应被普通用户界面引用      |

### 后端模块依赖关系

```
routes/  ──→  services/  ──→  Prisma (数据库)
   │              │
   ▼              ▼
middleware/    socket/
(拦截请求)    (实时通信)
```

- **Route 层**：处理 HTTP 请求，调用 Service 层，返回响应
- **Service 层**：纯业务逻辑，通过 Prisma Client 操作数据库
- **Middleware 层**：认证校验、参数校验、错误处理，在 Route 之前执行
- **Socket 层**：实时事件处理，调用 Service 层完成业务操作
- **各 Route/Service 之间不互相调用**，如需跨模块逻辑，在 Service 层注入依赖或提取公共方法


### 6.2 页面层级关系

```
应用根 (App)
├── 🏠 首页 (HomePage) [公开访问]
├── 🔐 认证层 (Auth Layer)
│   ├── 登录页 (LoginPage)
│   ├── 注册页 (RegisterPage)
│   └── 封禁提示页 (BannedPage)
├── 🏠 首页层 (Home Layer)
│   ├── 📱 主布局 (MainLayout)
│   │   ├── 左侧鱼圈栏 (CircleSidebar) [200px]
│   │   ├── 顶部导航栏 (Navbar)
│   │   └── 主内容区 (Content Area)
│   │       ├── 💬 蛐蛐间 (ChatRoom) [默认Tab]
│   │       └── 🎮 摸鱼鱼 (GamePage) [Tab切换]
│   └── 💰 窝囊费页 (SalaryPage) [独立页面]
├── 🎭 伪装层 (Disguise Layer)
└── 🔧 管理后台 (Admin Layer)
```
