# 摸鱼鱼交互优化 PRD (V1.2.0)

## 1. 背景与目标

### 为什么做

V1.0.0版本的摸鱼鱼存在以下问题：

- 抽卡概率设计不合理，用户体验有待优化
- 摸鱼交互方式单一，缺乏趣味性
- 宠物鱼成长体系与卡片数量挂钩，不够直观
- 每日摸鱼次数上限随机，用户感知不明确

### 用户价值

- 点击小鱼触发摸鱼动画，增加趣味性和仪式感
- 抽卡概率更合理，每次摸鱼都有期待感
- 成长值与摸鱼次数挂钩，规则更清晰
- 固定每日上限，用户规划更明确

### 成功标准

- 用户日均摸鱼次数 ≥ 10次
- 摸鱼动画播放流畅，无卡顿
- 成长值获取规则用户理解率 ≥ 90%

***

## 2. 范围定义

### 本次做

- 摸鱼交互方式变更：点击小鱼触发摸鱼动画
- 抽卡概率重新设计
- 每日摸鱼次数上限固定为30次
- 宠物鱼成长体系重构：成长值+1
- 宠物鱼进化形态调整为4个等级
- 摸鱼状态规则明确

### 本次不做

- 卡片交易/赠送功能
- 卡片合成/升级功能
- 宠物鱼自定义命名
- 宠物鱼装扮/道具系统
- 跨鱼圈排行榜
- 线上支付兑换

## 5. 功能详细设计

### 5.1 摸鱼交互方式

#### 功能说明

功能目标：提供核心交互入口，用户点击小鱼触发摸鱼动画，动画结束后获得卡片

#### 页面设计

功能入口：摸鱼主界面鱼缸区域的宠物鱼

页面组成：

- 标题："带薪疯狂摸鱼 🦥"
- 副标题："点击小鱼带薪抽卡！每戳一次增加鱼池成长值且有机会掉落精美UNO卡片。"
- 今日配额显示："{已用次数} / 30"
- 今日获卡显示："{今日获卡数} / 5"
- 鱼缸区域：显示宠物鱼，可点击
- 成长值显示：当前成长值 / 升级所需成长值

#### 交互设计

| 操作     | 反馈                  |
| ------ | ------------------- |
| 点击宠物鱼  | 播放小手摸鱼动画（1秒，左→右→左）  |
| 动画结束   | 弹出卡片掉落弹窗（如果有卡片）     |
| 已达今日上限 | 显示"你已触及今日防沉迷保护网！"提示 |

#### 摸鱼动画

| 元素   | 说明           |
| ---- | ------------ |
| 动画时长 | 1秒           |
| 动画内容 | 小手从左→右→左摸一来回 |
| 触发方式 | 点击宠物鱼        |
| 结束行为 | 弹出卡片掉落弹窗     |

#### 状态流转

| 状态   | 说明                  |
| ---- | ------------------- |
| 可用   | 宠物鱼可点击，显示"点击小鱼带薪抽卡" |
| 摸鱼中  | 动画播放中，宠物鱼不可点击       |
| 已达上限 | 宠物鱼灰色，显示上限提示        |

#### 摸鱼条件

**可以摸鱼的条件**：

- 用户已登录
- 已加入鱼圈
- 在摸鱼主界面
- 未达今日上限

**不可摸鱼的状态**：

- 已达今日上限 → 显示"你已触及今日防沉迷保护网！"
- 未登录状态 → 按钮禁用
- 未加入鱼圈 → 按钮禁用
- 网络异常 → 显示错误提示，允许重试

#### 业务规则

| 编号     | 规则                                     |
| ------ | -------------------------------------- |
| BR-001 | 每次摸鱼触发1秒动画                             |
| BR-002 | 动画结束后判断是否掉卡                            |
| BR-003 | 每日摸鱼次数上限固定30次                          |
| BR-004 | 每日配额在凌晨0点重置，以点击时间为准（23:59点击、0:00结束算当天） |
| BR-005 | 达到上限后宠物鱼不可点击                           |
| BR-015 | 宠物鱼5级为满级，满级后成长值显示"MAX"不再增长             |
| BR-016 | 所有54张卡全收集后，40%新卡概率转为不掉卡，提示"图鉴已满"       |
| BR-017 | 不掉卡时显示轻量提示"这次运气不佳\~"，1.5秒后自动消失         |

#### 异常处理

| 场景     | 处理方式            |
| ------ | --------------- |
| 卡片保存失败 | 控制台输出错误，不影响用户体验 |
| 网络异常   | 显示错误提示，允许重试     |

### 5.2 UNO卡片掉落

#### 功能说明

功能目标：每次摸鱼随机掉落UNO卡片，展示卡片详情

#### 卡片池设计

| 类别     | 数量      | 说明                 |
| ------ | ------- | ------------------ |
| 红色卡片   | 13张     | 0-9 + 跳过 + 反转 + +2 |
| 蓝色卡片   | 13张     | 0-9 + 跳过 + 反转 + +2 |
| 绿色卡片   | 13张     | 0-9 + 跳过 + 反转 + +2 |
| 黄色卡片   | 13张     | 0-9 + 跳过 + 反转 + +2 |
| 万能卡片   | 2张      | 变色 + +4            |
| **总计** | **54种** | -                  |

#### 稀有度设计

| 稀有度      | 标签 | 对应卡片     | 说明  |
| -------- | -- | -------- | --- |
| N（普通）    | N  | 0-9数字卡   | 最常见 |
| R（稀有）    | R  | 跳过、反转、+2 | 较稀有 |
| SR（超级稀有） | SR | 变色、+4    | 最稀有 |

#### 抽卡概率设计

| 结果      | 概率  | 说明              |
| ------- | --- | --------------- |
| 不掉卡     | 60% | 什么都不掉           |
| 已有卡（重复） | 20% | 掉一张用户已收集的卡      |
| 不重复普通卡  | 10% | 掉一张用户未收集的N稀有度卡  |
| 不重复稀有卡  | 7%  | 掉一张用户未收集的R稀有度卡  |
| 不重复超稀有卡 | 3%  | 掉一张用户未收集的SR稀有度卡 |

**数值设计说明**：

- 10人满活跃鱼圈：每天300次摸鱼，约60次掉卡
- 集齐54种卡片预计需要：54÷(60×40%)=2.25天，但实际需要更长时间（因为概率随机）
- 设计目标：让卡片收集成为长期目标，不是几天就集齐

**特殊规则**：

- 稀有卡（R/SR）不会重复，保证每次获得都是新卡
- 当某种稀有度的卡都收集完后，该概率转为重复卡概率
- 重复卡仅累加数量，没有额外价值
- **全收集兜底**：当所有54张卡都收集完后，40%的新卡概率（普通10%+稀有7%+超稀有3%）转为不掉卡概率，最终为80%不掉卡 + 20%重复卡，并提示"图鉴已满！继续摸鱼只能获得重复卡"
- **每日卡片上限**：每个用户每天在每个鱼圈最多获得5张卡，达到上限后不再掉卡（100%不掉卡）

#### 卡片掉落弹窗

页面组成：

- 标题："摸鱼成功！抽得神牌"
- 副标题："今日工作时间在此凝固，转化为可爱的能量 card！"
- 卡片展示区：并排展示1\~2张卡片
- 每张卡片包含：稀有度标签、数字/符号、卡片名称、彩蛋文字
- 按钮："继续摸鱼"（触发下一次摸鱼）和"关闭"（关闭弹窗返回摸鱼状态）

**不掉卡时的反馈**：

- 不弹窗，在鱼缸区域显示轻量提示文字"这次运气不佳\~"
- 提示显示1.5秒后自动消失

#### 交互设计

| 操作       | 反馈                                   |
| -------- | ------------------------------------ |
| 摸鱼成功     | 弹出卡片掉落弹窗                             |
| 点击"继续摸鱼" | 关闭弹窗，返回摸鱼主界面                         |
| 卡片颜色区分   | Red红色、Blue蓝色、Green绿色、Yellow黄色、Wild紫色 |

#### 业务规则

| 编号     | 规则                                  |
| ------ | ----------------------------------- |
| BR-006 | 卡片从54种卡片池中随机抽取                      |
| BR-007 | 卡片可重复获得，重复卡片count累加                 |
| BR-008 | 每张卡片包含独特彩蛋文字（bonusText）             |
| BR-009 | 稀有卡（R/SR）不会重复                       |
| BR-018 | 卡片掉落弹窗按钮为"继续摸鱼"（触发下一次摸鱼）和"关闭"（关闭弹窗） |
| BR-019 | 兑换实物条件：集齐54种不同花色卡片（每种至少1张）          |
| BR-020 | 需记录每个花色的收集数量，用于后续扩展                 |
| BR-021 | 每个用户每天在每个鱼圈最多获得5张卡，达到上限后不再掉卡       |

### 5.3 宠物鱼成长体系

#### 功能说明

功能目标：鱼圈共享宠物鱼，通过摸鱼行为获得成长值成长

#### 页面设计

功能入口：摸鱼主界面左侧

页面组成：

- 标题："治愈金鱼池"
- 副标题："全队共享宠物鱼 · 摸鱼注入能量"
- 鱼缸区域：蓝色背景，显示宠物鱼emoji和名称
- 宠物鱼信息：品类、等级标签
- 成长值条：显示当前成长值/升级所需成长值

#### 宠物鱼进化形态

| 等级 | 品类名称    | emoji | 说明       |
| -- | ------- | ----- | -------- |
| 1级 | 肥嘟嘟胖金鱼  | 🐠    | 初始形态     |
| 2级 | 带薪发愣神游鳌 | 🐙    | 中级形态     |
| 3级 | 太极双休太公鱼 | 🐙    | 高级形态     |
| 4级 | 极品七彩锦鲤皇 | 🎏    | 最终形态     |
| 5级 | 传说级摸鱼之神 | 🐉    | 满级（传说形态） |

**满级规则**：

- 5级为满级，满级后成长值停止增长
- 成长值显示为"MAX"
- 继续摸鱼仍可获得卡片，但成长值不再增加

#### 成长值计算

| 项目        | 计算方式                                             |
| --------- | ------------------------------------------------ |
| 每次摸鱼获得成长值 | 固定1点                                             |
| 升级所需成长值   | 等级1→2: 1000点等级2→3: 2000点等级3→4: 3000点等级4→5: 4000点 |
| 升级后成长值溢出  | 溢出部分保留到下一级                                       |

**数值设计说明**：

- 10人满活跃鱼圈（每人每天30次）：每天300点成长值
- 升级总需求：1000+2000+3000+4000=10000点
- 预计满级时间：10000÷300≈33天（约1个月）

#### 交互设计

| 操作    | 反馈             |
| ----- | -------------- |
| 宠物鱼升级 | 显示"进化成功！"动画    |
| 宠物鱼动画 | 默认浮动动画，升级时弹跳动画 |

#### 业务规则

| 编号     | 规则                             |
| ------ | ------------------------------ |
| BR-010 | 宠物鱼为鱼圈共享，所有成员的摸鱼行为都会增加成长值      |
| BR-011 | 宠物鱼成长值、等级、品类存储在鱼圈文档中           |
| BR-012 | 宠物鱼等级提升后自动更新品类名称和emoji         |
| BR-013 | 成长值条显示格式为"{当前成长值} / {升级所需成长值}" |
| BR-014 | 每次摸鱼固定获得1点成长值，无论是否掉卡           |

#### 异常处理

| 场景       | 处理方式            |
| -------- | --------------- |
| 鱼圈文档更新失败 | 控制台输出错误，不影响用户体验 |

## 6. 验收标准

### Happy Path

- AC-001
  - Given 用户今日摸鱼次数未达上限
  - When 用户点击宠物鱼
  - Then 播放1秒摸鱼动画，动画结束后弹出卡片掉落弹窗
- AC-002
  - Given 用户摸鱼成功
  - When 动画结束
  - Then 宠物鱼成长值+1
- AC-003
  - Given 用户获得UNO卡片
  - When 卡片掉落
  - Then 卡片保存到用户收集库，数量累加
- AC-004
  - Given 宠物鱼成长值达到升级条件
  - When 自动升级
  - Then 等级+1，品类变化，显示"进化成功！"动画
- AC-005
  - Given 用户在摸鱼主界面
  - When 查看今日配额
  - Then 显示"{已用次数} / 30"
- AC-006
  - Given 用户在摸鱼主界面
  - When 查看今日获卡数
  - Then 显示"{今日获卡数} / 5"

### Edge Case

- AC-101
  - Given 用户今日摸鱼次数已达上限
  - When 用户尝试点击宠物鱼
  - Then 宠物鱼不可点击，显示"你已触及今日防沉迷保护网！"提示
- AC-102
  - Given 用户获得重复卡片
  - When 卡片保存
  - Then 卡片count累加，不新增种类
- AC-103
  - Given 宠物鱼成长值溢出升级所需
  - When 自动升级
  - Then 溢出成长值保留到下一级
- AC-104
  - Given 用户未登录或未加入鱼圈
  - When 尝试点击宠物鱼
  - Then 宠物鱼不可点击
- AC-105
  - Given 所有54张卡都已收集
  - When 用户摸鱼触发抽卡
  - Then 80%概率不掉卡并提示"图鉴已满！继续摸鱼只能获得重复卡"，20%概率掉重复卡
- AC-106
  - Given 摸鱼结果为不掉卡（30%概率）
  - When 动画结束
  - Then 不弹窗，在鱼缸区域显示"这次运气不佳\~"提示，1.5秒后自动消失
- AC-107
  - Given 宠物鱼已达到5级（满级）
  - When 用户继续摸鱼
  - Then 成长值显示"MAX"不再增长，但仍可获得卡片
- AC-108
  - Given 用户在23:59点击摸鱼，动画在0:00结束
  - When 系统判断配额归属
  - Then 以点击时间为准，算作当天的配额
- AC-109
  - Given 用户今日在该鱼圈已获得5张卡
  - When 用户继续摸鱼
  - Then 100%不掉卡，显示"这次运气不佳\~"提示

### Business Rule

- AC-201
  - Given 用户点击宠物鱼
  - When 系统判断是否掉卡
  - Then 60%概率不掉卡，20%概率掉已有卡，10%概率掉不重复普通卡，7%概率掉不重复稀有卡，3%概率掉不重复超稀有卡
- AC-202
  - Given 用户摸鱼成功
  - When 计算成长值
  - Then 每次摸鱼固定获得1点成长值
- AC-203
  - Given 宠物鱼达到新的等级
  - When 更新品类
  - Then 1级"肥嘟嘟胖金鱼"，2级"带薪发愣神游鳌"，3级"太极双休太公鱼"，4级"极品七彩锦鲤皇"，5级"传说级摸鱼之神"
- AC-204
  - Given 用户ID已确定
  - When 计算每日摸鱼上限
  - Then 固定30次，凌晨0点重置，以点击时间为准
- AC-205
  - Given 卡片掉落弹窗显示
  - When 用户点击"继续摸鱼"
  - Then 关闭弹窗并自动触发下一次摸鱼
- AC-206
  - Given 卡片掉落弹窗显示
  - When 用户点击"关闭"
  - Then 关闭弹窗返回摸鱼主界面
- AC-207
  - Given 用户集齐54种不同花色卡片
  - When 查看兑换资格
  - Then 显示"已集齐全套图鉴，可兑换实物"
- AC-208
  - Given 用户收集卡片
  - When 卡片入库
  - Then 记录该花色的收集数量（用于后续扩展）
- AC-209
  - Given 用户ID和鱼圈ID已确定
  - When 计算每日获卡上限
  - Then 每个用户每天在每个鱼圈最多获得5张卡，凌晨0点重置，以卡片掉落时间为准

## 8. 非功能需求

### 性能

- 摸鱼动画播放流畅，无卡顿
- 卡片掉落弹窗显示延迟 < 200ms
- 成长值更新实时

### 可用性

- 摸鱼操作成功率 ≥ 99%
- 卡片数据一致性 100%
- 动画播放成功率 100%

### 安全性

- 每日配额基于服务端统计，防止客户端篡改
- 卡片收集数据存储在用户私有目录，其他用户无法访问

## 9. 数据模型

### UNO卡片定义 (UnoCardInfo)

| 字段        | 类型     | 说明                             |
| --------- | ------ | ------------------------------ |
| id        | string | 卡片唯一标识（如 "R\_0", "B\_Skip"）    |
| name      | string | 卡片名称（如 "红色 0"）                 |
| color     | string | 颜色（Red/Blue/Green/Yellow/Wild） |
| value     | string | 显示值（如 "0", "🚫", "⇄", "+2"）    |
| rarity    | string | 稀有度（N/R/SR）                    |
| bonusText | string | 彩蛋文字                           |

### 用户卡片收集 (UnoCard)

| 字段          | 类型        | 说明     |
| ----------- | --------- | ------ |
| id          | string    | 卡片ID   |
| cardName    | string    | 卡片名称   |
| count       | number    | 持有数量   |
| rarity      | string    | 稀有度    |
| color       | string    | 颜色     |
| collectedAt | Timestamp | 首次收集时间 |
| updatedAt   | Timestamp | 最后更新时间 |

### 摸鱼统计 (MoyuStat)

| 字段           | 类型        | 说明      |
| ------------ | --------- | ------- |
| userId       | string    | 用户ID    |
| userName     | string    | 用户昵称    |
| todayCount   | number    | 今日摸鱼次数  |
| todayCardCount | number    | 今日获卡数量（该鱼圈） |
| totalCount   | number    | 历史总摸鱼次数 |
| lastMoyuTime | Timestamp | 最后摸鱼时间  |

### 宠物鱼数据（存储在Circle文档中）

| 字段            | 类型     | 说明    |
| ------------- | ------ | ----- |
| petFishName   | string | 宠物鱼名称 |
| petFishLevel  | number | 当前等级  |
| petFishGrowth | number | 当前成长值 |
| petFishType   | string | 品类名称  |

### 存储路径

- 用户卡片：`users/{userId}/cards/{cardId}`
- 摸鱼统计：`circles/{circleId}/moyu_stats/{userId}`
- 宠物鱼数据：`circles/{circleId}` 文档内
