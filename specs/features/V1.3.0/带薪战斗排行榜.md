# 带薪战斗排行榜 PRD (V1.1.0)

## 1. 背景与目标

### 为什么做

V1.0.0版本的摸鱼鱼缺乏竞争激励机制，用户容易产生疲劳感。通过引入排行榜，可以：

- 增加用户之间的竞争意识
- 提供可视化的摸鱼成就展示
- 促进鱼圈成员之间的互动

### 用户价值

- 看到自己的摸鱼排名，获得成就感
- 与鱼圈成员竞争，增加使用动力
- 查看历史总榜，了解长期摸鱼成果

### 成功标准

- 排行榜加载时间 < 1秒
- 用户日均查看排行榜次数 ≥ 3次

***

## 2. 范围定义

### 本次做

- 今日摸鱼排行榜
- 历史总摸鱼排行榜
- 排行榜显示用户头像、昵称、摸鱼次数

### 本次不做

- 排行榜奖励机制
- 排行榜分享功能
- 跨鱼圈排行榜

***

## 5. 功能详细设计

### 5.1 带薪战斗排行榜

#### 功能说明

功能目标：展示鱼圈成员的摸鱼排名，增加竞争意识

#### 页面设计

功能入口：摸鱼主界面

页面组成：

- 标题："带薪战斗排行榜"
- Tab切换：今日排行 / 历史总榜
- 排行榜列表：显示用户头像、昵称、摸鱼次数
- 当前用户高亮显示

#### 排行榜规则

| 规则   | 说明             |
| ---- | -------------- |
| 今日排行 | 按今日摸鱼次数从高到低排序  |
| 历史总榜 | 按历史总摸鱼次数从高到低排序 |
| 显示数量 | 显示所有鱼圈成员       |
| 重置时间 | 今日排行榜在凌晨0点重置   |

#### 排行榜显示信息

| 字段   | 说明               |
| ---- | ---------------- |
| 排名   | 数字排名（1, 2, 3...） |
| 头像   | 用户头像             |
| 昵称   | 用户昵称             |
| 摸鱼次数 | 今日/历史总摸鱼次数       |

#### 交互设计

| 操作          | 反馈            |
| ----------- | ------------- |
| 点击"今日排行"Tab | 显示今日摸鱼排行榜。    |
| 点击"历史总榜"Tab | 显示历史总摸鱼排行榜    |
| 当前用户        | 高亮显示，方便用户快速定位 |

#### 业务规则

| 编号     | 规则                                |
| ------ | --------------------------------- |
| BR-001 | 今日排行榜按今日摸鱼次数从高到低排序（摸鱼次数不是获得卡片的次数） |
| BR-002 | 历史总榜按历史总摸鱼次数从高到低排序                |
| BR-003 | 排行榜显示所有鱼圈成员                       |
| BR-004 | 今日排行榜在凌晨0点重置                      |
| BR-005 | 当前用户在排行榜中高亮显示                     |
| BR-006 | 摸鱼次数相同时，按最后摸鱼时间排序                 |

#### 异常处理

| 场景    | 处理方式        |
| ----- | ----------- |
| 鱼圈无成员 | 显示"暂无排行数据"  |
| 网络异常  | 显示错误提示，允许重试 |

## 6. 验收标准

### Happy Path

- AC-001
  - Given 用户在摸鱼主界面
  - When 用户查看排行榜
  - Then 显示今日摸鱼排行榜，包含所有鱼圈成员的头像、昵称、摸鱼次数
- AC-002
  - Given 用户在摸鱼主界面
  - When 用户点击"历史总榜"Tab
  - Then 显示历史总摸鱼排行榜，包含所有鱼圈成员的头像、昵称、摸鱼次数
- AC-003
  - Given 用户在排行榜中
  - When 用户查看当前用户
  - Then 当前用户高亮显示

### Edge Case

- AC-101
  - Given 鱼圈无成员
  - When 用户查看排行榜
  - Then 显示"暂无排行数据"
- AC-102
  - Given 今日排行榜
  - When 凌晨0点
  - Then 排行榜重置，所有成员今日摸鱼次数清零
- AC-103
  - Given 多个成员摸鱼次数相同
  - When 系统排序
  - Then 按最后摸鱼时间排序，先摸鱼的排在前面

### Business Rule

- AC-201
  - Given 用户查看今日排行榜
  - When 系统排序
  - Then 按今日摸鱼次数从高到低排序
- AC-202
  - Given 用户查看历史总榜
  - When 系统排序
  - Then 按历史总摸鱼次数从高到低排序
- AC-203
  - Given 用户查看排行榜
  - When 系统显示
  - Then 显示所有鱼圈成员，包括今日未摸鱼的成员

## 8. 非功能需求

### 性能

- 排行榜加载时间 < 1秒
- Tab切换响应时间 < 200ms

### 可用性

- 排行榜显示正确率 100%
- 当前用户高亮显示正确率 100%

### 安全性

- 排行榜数据基于服务端统计，防止客户端篡改

## 9. 数据模型

### 排行榜数据

| 字段           | 类型        | 说明      |
| ------------ | --------- | ------- |
| userId       | string    | 用户ID    |
| userName     | string    | 用户昵称    |
| userAvatar   | string    | 用户头像    |
| todayCount   | number    | 今日摸鱼次数  |
| totalCount   | number    | 历史总摸鱼次数 |
| lastMoyuTime | Timestamp | 最后摸鱼时间  |

### 存储路径

- 摸鱼统计：`circles/{circleId}/moyu_stats/{userId}`
