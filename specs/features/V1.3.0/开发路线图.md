# V1.3.0 开发路线图 (Development Roadmap)

## 0. 策略概览

*   **开发策略**: 以资产归属规则为地基 → 摸鱼鱼交互优化为核心业务重构 → 签到与装饰系统为经济体系扩展 → 带薪战斗排行榜为竞争激励补充。遵循"规则先行、核心优先"原则，先明确资产归属定义，再重构摸鱼鱼交互（为排行榜提供数据基础），然后构建签到装饰经济体系，最后上线排行榜。
*   **MVP 目标**: 完成鱼圈资产归属规则落地 + 摸鱼鱼交互优化（核心业务闭环），确保摸鱼动画、抽卡概率、成长体系、每日上限全部按新规则运行。

---

## 1. 依赖分析

### 地基模块（必须最先完成）
- **鱼圈资产归属规则** — 定义鱼币/宠物鱼/装饰/卡片四类资产的归属逻辑，是签到系统、装饰系统、排行榜数据存储的前提

### 核心业务模块（MVP核心）
- **摸鱼鱼交互优化** — 重构摸鱼动画、抽卡概率（60%不掉/20%重复/10%普通/7%稀有/3%超稀有）、每日上限30次、成长值+1/次、5级进化形态、每日卡片上限5张。为排行榜提供 `moyu_stats` 数据基础

### 新增功能模块（功能完善）
- **签到与装饰系统** — 依赖资产归属规则定义鱼币归属，新增每日签到（1鱼币/鱼圈/天）、鱼币公共账户、装饰商店（5款/5~20鱼币）、兑换记录
- **带薪战斗排行榜** — 依赖摸鱼鱼交互优化产生的 `moyu_stats` 统计数据，新增今日/历史排行榜

## 2. 里程碑规划

### Milestone 1: 规则定义 + 核心交互重构（MVP）

> 目标：完成资产归属规则落地和摸鱼鱼交互优化，实现核心摸鱼业务闭环。

#### Phase 1.1: 鱼圈资产归属规则落地
*   **模块**: `Server/CircleService` / `Server/GameService` / `Server/AuthService`
*   **说明**:
    - 确认所有资产归属逻辑：鱼币→鱼圈公共、宠物鱼→鱼圈共享、装饰→鱼圈公共、卡片→用户私有
    - 用户退出鱼圈：装饰保留、卡片保留、历史记录保留
    - 鱼圈解散：鱼币/宠物鱼/装饰删除、卡片保留
    - 新成员加入：共享鱼圈资产，卡片从零开始
*   **开发顺序**: 1（所有后续功能的规则前提）
*   **提示词**: `"落地鱼圈资产归属规则，确保鱼币/宠物鱼/装饰属于鱼圈公共，卡片属于用户私有；实现用户退出鱼圈时装饰和卡片保留、鱼圈解散时公共资产清零但用户卡片保留、新成员加入时共享鱼圈资产但卡片从零开始的逻辑"`
*   **详细文档**: `specs/features/V1.3.0/鱼圈资产归属规则.md`

#### Phase 1.2: 摸鱼鱼交互优化
*   **模块**: `Game/PetFish` / `Game/CardDropModal` / `Game/FishTank` / `Server/GameService`
*   **说明**:
    - 摸鱼交互变更：点击小鱼触发1秒摸鱼动画（左→右→左）
    - 状态流转：可用→摸鱼中→已达上限
    - 抽卡概率重新设计：60%不掉卡、20%重复卡、10%不重复普通(N)、7%不重复稀有(R)、3%不重复超稀有(SR)
    - 每日摸鱼次数上限固定30次，凌晨0点重置（以点击时间为准）
    - 每日卡片上限5张/鱼圈
    - 成长值固定+1/次，升级所需1000→2000→3000→4000
    - 宠物鱼5级进化形态（新增5级传说级摸鱼之神 🐉）
    - 满级后成长值显示"MAX"不再增长
    - 全收集兜底：54张全收后80%不掉卡+20%重复卡
    - 不掉卡时显示"这次运气不佳~"提示1.5秒
*   **开发顺序**: 2（为排行榜提供moyu_stats数据）
*   **提示词**: `"重构摸鱼鱼交互，点击小鱼触发1秒摸鱼动画（左→右→左），抽卡概率改为60%不掉/20%重复/10%普通N/7%稀有R/3%超稀有SR，每日上限固定30次凌晨重置，每日卡片上限5张，成长值固定+1/次，宠物鱼升级所需1000→2000→3000→4000，新增5级传说级摸鱼之神🐉，满级显示MAX，全收集后80%不掉卡+20%重复卡"`
*   **详细文档**: `specs/features/V1.3.0/摸鱼鱼交互优化.md`

### Milestone 2: 签到与装饰系统（经济体系）

> 目标：构建鱼币经济体系和装饰系统，增加长期激励。

#### Phase 2.1: 签到系统 + 鱼币系统
*   **模块**: `Game/SignCalendar` / `Server/CircleService`
*   **说明**:
    - 每日签到：每个用户每天在每个鱼圈可签到一次，获得1鱼币
    - 鱼币归属：鱼币进入当前鱼圈的公共账户
    - 鱼币余额显示：摸鱼主界面 + 装饰商店
    - 签到状态凌晨0点重置，以点击签到按钮时间为准
    - 签到成功提示"签到成功！+1鱼币"，1.5秒后自动消失
*   **开发顺序**: 3（装饰系统的经济基础）
*   **提示词**: `"开发签到系统，每个用户每天在每个鱼圈可签到一次获得1鱼币，鱼币进入鱼圈公共账户，签到状态凌晨0点重置以点击时间为准，签到成功提示1.5秒后自动消失，在摸鱼主界面和装饰商店显示鱼币余额"`
*   **详细文档**: `specs/features/V1.3.0/签到与装饰系统.md`

#### Phase 2.2: 装饰商店 + 装饰显示
*   **模块**: `Game/DecorationShop` / `Game/FishTank` / `Server/CircleService`
*   **说明**:
    - 装饰商店：预置5款装饰（水草5鱼币、气泡10鱼币、石头10鱼币、海星20鱼币、珊瑚20鱼币）
    - 购买确认弹窗："确认花费{价格}鱼币购买{装饰名称}？"
    - 装饰是鱼圈公共资源，每个鱼圈的装饰只能购买一次
    - 购买后装饰自动显示在鱼缸固定位置（写死在代码中）
    - 装饰不存在穿上/卸下操作
    - 已购买装饰显示"已拥有"标签
*   **开发顺序**: 4
*   **提示词**: `"开发装饰商店，预置5款装饰（水草5币/气泡10币/石头10币/海星20币/珊瑚20币），装饰是鱼圈公共资源只能购买一次，购买后自动显示在鱼缸固定位置，支持购买确认弹窗和鱼币不足提示"`
*   **详细文档**: `specs/features/V1.3.0/签到与装饰系统.md`

#### Phase 2.3: 兑换记录
*   **模块**: `Game/DecorationShop` / `Server/CircleService`
*   **说明**:
    - 兑换记录入口：装饰商店内
    - 记录内容：兑换时间、兑换用户、兑换装饰、消耗鱼币
    - 按时间倒序排列
    - 记录为空时显示"暂无兑换记录"
*   **开发顺序**: 5
*   **提示词**: `"开发兑换记录功能，在装饰商店内展示鱼币兑换历史，记录包含兑换时间、用户、装饰、鱼币数，按时间倒序排列，为空时显示暂无兑换记录"`
*   **详细文档**: `specs/features/V1.3.0/签到与装饰系统.md`

### Milestone 3: 带薪战斗排行榜（竞争激励）

> 目标：增加用户之间的竞争意识，提供摸鱼成就展示。

#### Phase 3.1: 排行榜数据服务
*   **模块**: `Server/GameService`
*   **说明**:
    - 摸鱼统计存储：`circles/{circleId}/moyu_stats/{userId}`
    - 统计字段：userId、userName、userAvatar、todayCount、totalCount、lastMoyuTime
    - 今日排行榜：按今日摸鱼次数从高到低排序
    - 历史总榜：按历史总摸鱼次数从高到低排序
    - 摸鱼次数相同时按最后摸鱼时间排序（先摸鱼的排前面）
    - 今日排行榜凌晨0点重置
*   **开发顺序**: 6（排行榜UI的数据基础）
*   **提示词**: `"开发排行榜数据服务，存储摸鱼统计到circles/{circleId}/moyu_stats/{userId}，支持今日排行和历史总榜查询，今日排行按今日摸鱼次数排序，历史总榜按总摸鱼次数排序，次数相同时按最后摸鱼时间排序"`
*   **详细文档**: `specs/features/V1.3.0/带薪战斗排行榜.md`

#### Phase 3.2: 排行榜UI
*   **模块**: `Game/Leaderboard`
*   **说明**:
    - 排行榜入口：摸鱼主界面
    - Tab切换：今日排行 / 历史总榜
    - 排行榜列表：排名、用户头像、昵称、摸鱼次数
    - 当前用户高亮显示
    - 显示所有鱼圈成员（包括今日未摸鱼的）
    - 鱼圈无成员时显示"暂无排行数据"
    - 加载时间 < 1秒，Tab切换响应 < 200ms
*   **开发顺序**: 7
*   **提示词**: `"开发排行榜UI，在摸鱼主界面展示带薪战斗排行榜，支持今日排行和历史总榜Tab切换，显示所有鱼圈成员的排名、头像、昵称、摸鱼次数，当前用户高亮显示，鱼圈无成员时显示暂无排行数据"`
*   **详细文档**: `specs/features/V1.3.0/带薪战斗排行榜.md`

## 3. 开发顺序总览

```
Phase 1.1 鱼圈资产归属规则 ──→ Phase 1.2 摸鱼鱼交互优化
                                      │
                      ┌───────────────┼───────────────┐
                      ▼                               ▼
        Phase 2.1 签到+鱼币系统              Phase 3.1 排行榜数据服务
                │                               │
                ▼                               ▼
        Phase 2.2 装饰商店+显示              Phase 3.2 排行榜UI
                │
                ▼
        Phase 2.3 兑换记录
```

**依赖关系说明**:
- Phase 1.1 是所有后续Phase的规则前提（资产归属影响所有数据存储设计）
- Phase 1.2 是排行榜和签到装饰的数据基础（摸鱼统计→排行榜，摸鱼行为→成长值）
- Phase 2.1 → 2.2 → 2.3 严格串行（签到产出鱼币→鱼币购买装饰→装饰产生兑换记录）
- Phase 3.1 → 3.2 串行（数据服务→UI展示）
- Phase 2.x 和 Phase 3.x 可以并行开发（签到装饰系统与排行榜相互独立）

## 4. 关键技术决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 摸鱼动画实现 | CSS Animation + 1秒时长 | 轻量级方案，不依赖动画库 |
| 抽卡概率实现 | 服务端随机 + 客户端展示 | 防止客户端篡改，安全性优先 |
| 每日配额重置 | 以点击时间为准 | 避免动画期间跨天的歧义 |
| 鱼币存储 | Circle模型 coinBalance字段 | 鱼币是鱼圈公共财产，直接存在鱼圈文档 |
| 装饰位置 | 代码写死固定位置 | 5款装饰位置固定，无需动态配置 |
| 排行榜排序 | 服务端排序 + 客户端展示 | 保证数据一致性 |
| 今日排行重置 | 查询时按日期过滤 | 无需定时任务，查询时实时计算 |
