# 鱼圈资产归属规则 PRD (V1.1.0)

## 1. 背景与目标

### 为什么做

V1.1.0版本引入了多种资产类型（鱼币、装饰、宠物鱼、卡片），需要明确这些资产的归属规则，避免后续开发和运营中出现理解偏差。

### 用户价值

- 用户清楚知道自己的行为（签到、摸鱼、抽卡）是在为鱼圈积累资产
- 用户理解资产是鱼圈公共资源，增强团队协作意识
- 用户退出鱼圈时，资产处理规则明确，避免纠纷

### 成功标准

- 所有资产相关的功能都遵循本规则
- 用户理解率 ≥ 90%

***

## 2. 范围定义

### 本次做

- 明确所有资产的归属规则
- 定义用户退出鱼圈时的资产处理方式
- 定义鱼圈解散时的资产处理方式

### 本次不做

- 资产交易/赠送功能
- 资产跨鱼圈转移功能

***

## 3. 核心原则

### 资产归属原则

**所有资产都属于鱼圈，不属于个人用户。**

用户只是"借手"操作：

- 用户签到 → 为鱼圈赚取鱼币
- 用户摸鱼 → 为鱼圈增加宠物鱼成长值
- 用户抽卡 → 为鱼圈收集卡片
- 用户购买装饰 → 为鱼圈装饰鱼缸

### 资产类型

| 资产类型 | 归属     | 说明                        |
| ---- | ------ | ------------------------- |
| 鱼币   | 鱼圈公共账户 | 用户签到为鱼圈赚取，用于购买装饰          |
| 宠物鱼  | 鱼圈共享   | 所有成员的摸鱼行为都会增加成长值          |
| 装饰   | 鱼圈公共资源 | 用户购买后自动显示在鱼圈鱼缸，所有成员可见     |
| 卡片   | 用户私有   | 用户收集的卡片存储在用户私有目录，其他用户无法访问 |

**注意**：卡片是唯一一个属于用户私有的资产类型，其他资产都属于鱼圈。

***

## 4. 用户角色

| 角色    | 说明             |
| ----- | -------------- |
| 鱼圈创建者 | 创建鱼圈的用户，拥有管理权限 |
| 鱼圈成员  | 加入鱼圈的普通成员      |
| 退出成员  | 退出鱼圈的前成员       |

***

## 5. 资产处理规则

### 5.1 用户退出鱼圈

#### 功能说明

功能目标：定义用户退出鱼圈时各类资产的处理方式

#### 处理规则

| 资产类型 | 处理方式 | 说明                      |
| ---- | ---- | ----------------------- |
| 鱼币   | 不处理  | 鱼币属于鱼圈公共账户，与用户无关        |
| 宠物鱼  | 不处理  | 宠物鱼属于鱼圈，与用户无关           |
| 装饰   | 保留   | 用户购买的装饰保留在鱼圈鱼缸，不影响其他成员  |
| 卡片   | 保留   | 卡片属于用户私有，退出后用户仍可查看自己的卡片 |

#### 业务规则

| 编号     | 规则                         |
| ------ | -------------------------- |
| BR-001 | 用户退出鱼圈后，该用户购买的装饰保留在鱼圈      |
| BR-002 | 用户退出鱼圈后，该用户收集的卡片仍属于用户      |
| BR-003 | 用户退出鱼圈后，该用户的历史摸鱼记录保留在鱼圈排行榜 |
| BR-004 | 用户退出鱼圈后，该用户的签到记录保留在鱼圈      |

#### 异常处理

| 场景        | 处理方式              |
| --------- | ----------------- |
| 用户退出后重新加入 | 装饰仍存在，用户可继续使用鱼圈资产 |

***

### 5.2 鱼圈解散

#### 功能说明

功能目标：定义鱼圈解散时各类资产的处理方式

#### 处理规则

| 资产类型 | 处理方式 | 说明                        |
| ---- | ---- | ------------------------- |
| 鱼币   | 清零   | 鱼圈解散，鱼币消失                 |
| 宠物鱼  | 删除   | 鱼圈解散，宠物鱼数据删除              |
| 装饰   | 删除   | 鱼圈解散，装饰数据删除               |
| 卡片   | 保留   | 卡片属于用户私有，鱼圈解散后用户仍可查看自己的卡片 |

#### 业务规则

| 编号     | 规则                  |
| ------ | ------------------- |
| BR-005 | 鱼圈解散后，鱼币、宠物鱼、装饰数据删除 |
| BR-006 | 鱼圈解散前需确认，防止误操作      |

#### 异常处理

| 场景           | 处理方式               |
| ------------ | ------------------ |
| 鱼圈解散后用户创建新鱼圈 | 新鱼圈资产从零开始，不继承旧鱼圈资产 |

***

### 5.3 用户加入鱼圈

#### 功能说明

功能目标：定义用户加入鱼圈时的资产继承规则

#### 处理规则

| 资产类型 | 处理方式 | 说明                     |
| ---- | ---- | ---------------------- |
| 鱼币   | 共享   | 新成员可使用鱼圈公共账户的鱼币        |
| 宠物鱼  | 共享   | 新成员可看到并摸 鱼圈的宠物鱼        |
| 装饰   | 共享   | 新成员可看到并鱼圈已购买的装饰        |
| 卡片   | 独立   | 新成员的卡片收集从零开始，不继承鱼圈已有卡片 |

#### 业务规则

| 编号     | 规则                    |
| ------ | --------------------- |
| BR-008 | 新成员加入鱼圈后，可使用鱼圈公共账户的鱼币 |
| BR-009 | 新成员加入鱼圈后，可看到鱼圈已购买的装饰  |
| BR-010 | 新成员加入鱼圈后，卡片收集从零开始     |
| BR-011 | 新成员加入鱼圈后，可为鱼圈贡献成长值    |

## 6. 验收标准

### Happy Path

- AC-001
  - Given 用户退出鱼圈
  - When 系统处理该用户购买的装饰
  - Then 装饰保留在鱼圈鱼缸，其他成员仍可看到
- AC-002
  - Given 用户退出鱼圈
  - When 系统处理该用户的卡片
  - Then 卡片保留在用户私有目录，用户仍可查看
- AC-003
  - Given 用户加入鱼圈
  - When 系统处理该用户的卡片
  - Then 卡片收集从零开始，不继承鱼圈已有卡片
- AC-004
  - Given 用户加入鱼圈
  - When 系统处理该用户的鱼币
  - Then 用户可使用鱼圈公共账户的鱼币

### Edge Case

- AC-101
  - Given 鱼圈解散
  - When 系统处理鱼圈资产
  - Then 鱼币、宠物鱼、装饰数据删除，用户私有卡片保留
- AC-102
  - Given 用户退出鱼圈后重新加入
  - When 系统处理该用户的资产
  - Then 装饰仍存在，用户可继续使用鱼圈资产
- AC-103
  - Given 用户同时加入多个鱼圈
  - When 系统处理该用户的资产
  - Then 每个鱼圈的资产独立，互不影响

### Business Rule

- AC-201
  - Given 任何情况下
  - When 系统判断资产归属
  - Then 鱼币、宠物鱼、装饰属于鱼圈，卡片属于用户私有
- AC-202
  - Given 用户退出鱼圈
  - When 系统处理该用户的历史记录
  - Then 摸鱼记录、签到记录保留在鱼圈
- AC-203
  - Given 用户加入鱼圈
  - When 系统处理该用户的贡献
  - Then 用户可为鱼圈贡献成长值、鱼币

## 7. 数据模型

### 资产归属映射

| 资产类型 | 数据表              | 归属字段                        | 说明          |
| ---- | ---------------- | --------------------------- | ----------- |
| 鱼币   | Circle           | coinBalance                 | 存储在鱼圈文档中    |
| 宠物鱼  | Circle           | petFishGrowth, petFishLevel | 存储在鱼圈文档中    |
| 装饰   | CircleDecoration | circleId                    | 存储在鱼圈装饰表中   |
| 卡片   | UnoCard          | userId                      | 存储在用户卡片表中   |
| 摸鱼记录 | MoyuStat         | userId, circleId            | 存储在鱼圈摸鱼统计表中 |
| 签到记录 | SignInRecord     | userId, circleId            | 存储在鱼圈签到记录表中 |

### 存储路径

- 鱼币：`circles/{circleId}.coinBalance`
- 宠物鱼：`circles/{circleId}.petFishGrowth`, `circles/{circleId}.petFishLevel`
- 装饰：`circles/{circleId}/decorations/{decorationId}`
- 卡片：`users/{userId}/cards/{cardId}`
- 摸鱼记录：`circles/{circleId}/moyu_stats/{userId}`
- 签到记录：`circles/{circleId}/sign_records/{userId}_{date}`
