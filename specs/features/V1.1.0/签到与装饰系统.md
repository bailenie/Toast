# 签到与装饰系统 PRD (V1.1.0)

## 1. 背景与目标

### 为什么做

V1.0.0版本的摸鱼鱼缺乏长期激励机制，用户容易产生疲劳感。通过引入签到和装饰系统，可以：
- 增加用户每日登录和使用的动力
- 提供鱼币经济系统，让鱼圈成员有共同目标
- 装饰鱼缸增加团队归属感和成就感

### 用户价值

- 每日签到获得鱼币，增加使用粘性
- 鱼币兑换装饰，装饰鱼缸提升团队归属感
- 装饰是鱼圈公共资源，促进团队协作
- 兑换记录透明，增强信任感

### 成功标准

- 用户日均签到率 ≥ 80%
- 鱼圈装饰兑换率 ≥ 50%
- 用户对装饰系统满意度 ≥ 85%

---

## 2. 范围定义

### 本次做

- 每日签到功能
- 鱼币系统（鱼圈公共账户）
- 装饰商店（预置5款装饰）
- 装饰自动显示在鱼缸
- 鱼币兑换记录

### 本次不做

- 装饰穿上/卸下功能
- 装饰自定义上传
- 装饰交易/赠送
- 鱼币充值/购买
- 装饰限时活动

## 5. 功能详细设计

### 5.1 每日签到功能

#### 功能说明

功能目标：用户每日签到获得鱼币，增加使用粘性

#### 页面设计

功能入口：摸鱼主界面

页面组成：

- 签到按钮：显示"签到领鱼币"
- 签到状态：已签到/未签到
- 鱼圈鱼币余额：显示当前鱼圈的鱼币总数

#### 签到规则

| 规则 | 说明 |
| ---- | ---- |
| 签到奖励 | 固定1鱼币 |
| 签到限制 | 每个用户每天在每个鱼圈可以签到一次 |
| 鱼币归属 | 鱼币进入鱼圈的公共账户 |
| 重置时间 | 凌晨0点 |

#### 签到场景示例

**场景1：单鱼圈用户**
- 用户A只加入了产品组鱼圈
- 用户A在产品组鱼圈签到，产品组鱼圈账户+1鱼币
- 用户A今天不能在产品组鱼圈再次签到

**场景2：多鱼圈用户**
- 用户A有产品组、啦啦队两个鱼圈
- 用户A在产品组鱼圈签到，产品组鱼圈账户+1鱼币
- 用户A在啦啦队鱼圈签到，啦啦队鱼圈账户+1鱼币
- 用户A今天不能在这两个鱼圈再次签到，但可以在其他鱼圈签到

#### 交互设计

| 操作 | 反馈 |
| ---- | ---- |
| 点击签到按钮 | 显示"签到成功！+1鱼币"，按钮变为"已签到" |
| 已签到状态 | 按钮灰色，显示"已签到" |
| 切换鱼圈 | 显示当前鱼圈的签到状态和鱼币余额 |

#### 业务规则

| 编号 | 规则 |
| ---- | ---- |
| BR-001 | 每个用户每天在每个鱼圈只能签到一次 |
| BR-002 | 签到获得的鱼币进入当前鱼圈的公共账户 |
| BR-003 | 签到状态在凌晨0点重置 |
| BR-004 | 私有鱼圈也可以签到 |

#### 异常处理

| 场景 | 处理方式 |
| ---- | ---- |
| 签到失败 | 显示"签到失败，请重试" |
| 网络异常 | 显示错误提示，允许重试 |

### 5.2 鱼币系统

#### 功能说明

功能目标：管理鱼圈的鱼币经济

- 在装饰商店显示当前鱼圈的鱼币余额

#### 业务规则

| 编号 | 规则 |
| ---- | ---- |
| BR-005 | 鱼币是鱼圈公共财产 |
| BR-006 | 鱼币余额存储在鱼圈文档中 |
| BR-007 | 鱼币余额实时更新 |

## 6. 验收标准

### Happy Path

*   AC-001
    *   Given 用户在摸鱼主界面且未签到
    *   When 用户点击签到按钮
    *   Then 显示"签到成功！+1鱼币"，鱼圈鱼币余额+1

*   AC-002
    *   Given 用户在摸鱼主界面且已签到
    *   When 用户查看签到按钮
    *   Then 按钮灰色，显示"已签到"

*   AC-003
    *   Given 用户在装饰商店且鱼币充足
    *   When 用户购买装饰
    *   Then 显示"购买成功！"，装饰自动显示在鱼缸

*   AC-004
    *   Given 用户在装饰商店
    *   When 用户查看兑换记录
    *   Then 显示所有兑换历史

*   AC-005
    *   Given 用户切换鱼圈
    *   When 查看签到状态
    *   Then 显示当前鱼圈的签到状态和鱼币余额

### Edge Case

*   AC-101
    *   Given 用户今日已在某鱼圈签到
    *   When 用户尝试再次签到
    *   Then 按钮灰色，显示"已签到"

*   AC-102
    *   Given 用户鱼币不足
    *   When 用户尝试购买装饰
    *   Then 显示"鱼币不足，无法购买"

*   AC-103
    *   Given 装饰已被购买
    *   When 用户查看装饰
    *   Then 显示"已拥有"标签

*   AC-104
    *   Given 用户在私有鱼圈
    *   When 用户签到
    *   Then 私有鱼圈账户+1鱼币

### Business Rule

*   AC-201
    *   Given 用户在某鱼圈签到
    *   When 系统处理
    *   Then 鱼币进入该鱼圈的公共账户

*   AC-202
    *   Given 用户购买装饰
    *   When 系统处理
    *   Then 鱼圈鱼币余额减少，装饰自动显示

*   AC-203
    *   Given 用户购买装饰
    *   When 系统处理
    *   Then 记录兑换信息（时间、用户、装饰、鱼币数）

*   AC-204
    *   Given 鱼圈成员查看鱼缸
    *   When 系统显示
    *   Then 显示所有已购买的装饰

## 8. 非功能需求

### 性能

*   签到响应时间 < 1秒
*   装饰购买响应时间 < 2秒
*   兑换记录加载时间 < 1秒

### 可用性

*   签到成功率 ≥ 99%
*   装饰购买成功率 ≥ 99%
*   兑换记录显示正确率 100%

### 安全性

*   签到状态基于服务端统计，防止客户端篡改
*   鱼币余额基于服务端计算，防止客户端篡改
*   装饰购买基于服务端验证，防止重复购买

## 9. 数据模型

### 鱼圈鱼币 (CircleCoin)

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| circleId | string | 鱼圈ID |
| balance | number | 鱼币余额 |
| updatedAt | Timestamp | 最后更新时间 |

### 签到记录 (SignInRecord)

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| userId | string | 用户ID |
| circleId | string | 鱼圈ID |
| signInDate | string | 签到日期（YYYY-MM-DD） |
| createdAt | Timestamp | 签到时间 |

### 装饰定义 (Decoration)

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| id | string | 装饰唯一标识 |
| name | string | 装饰名称 |
| type | string | 装饰类型（装饰物/背景/特效） |
| price | number | 价格（鱼币） |
| description | string | 装饰描述 |

### 鱼圈装饰 (CircleDecoration)

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| circleId | string | 鱼圈ID |
| decorationId | string | 装饰ID |
| purchasedBy | string | 购买者用户ID |
| purchasedAt | Timestamp | 购买时间 |

### 兑换记录 (ExchangeRecord)

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| circleId | string | 鱼圈ID |
| userId | string | 用户ID |
| decorationId | string | 装饰ID |
| cost | number | 消耗鱼币数 |
| createdAt | Timestamp | 兑换时间 |

### 存储路径

- 鱼圈鱼币：`circles/{circleId}` 文档内
- 签到记录：`circles/{circleId}/sign_records/{userId}_{date}`
- 装饰定义：`decorations/{decorationId}`（系统预置）
- 鱼圈装饰：`circles/{circleId}/decorations/{decorationId}`
- 兑换记录：`circles/{circleId}/exchange_records/{recordId}`
