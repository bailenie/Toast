# V1.1.0 开发路线图 (Development Roadmap)

## 0. 策略概览

*   **开发策略**: 以页面布局重构为地基 → 鱼圈管理优化为核心 → 摸鱼鱼交互重构为业务升级 → 签到与装饰系统为新功能扩展。遵循"先改后加"原则，优先完成布局和鱼圈管理改动，再重构摸鱼鱼交互，最后新增签到与装饰。
*   **MVP 目标**: 完成页面布局重构（左侧鱼圈栏 + 新导航）和鱼圈管理优化（创建鱼圈实时邀请机制 + 新用户引导），确保基础导航和多鱼圈切换可用。

---

## 1. 依赖分析

### 地基模块（必须最先完成）
- **页面布局重构** — 所有功能依赖新的左侧栏 + 主内容区 + 底部Tab架构

### 核心业务模块（MVP核心）
- **鱼圈管理优化** — 依赖新的页面布局，但新用户引导和实时邀请机制影响整个业务闭环

### 业务升级模块（优化现有功能）
- **摸鱼鱼交互重构** — 依赖新的摸鱼鱼页面布局，改动抽卡概率、成长体系、每日上限
- **宠物鱼成长体系重构** — 与摸鱼鱼交互强耦合，需同步修改

### 新增功能模块（锦上添花）
- **签到与装饰系统** — 依赖新的摸鱼鱼页面布局，新增数据模型最多

## 2. 里程碑规划

### Milestone 1: 布局重构 + 鱼圈管理（MVP）

> 目标：完成页面布局重构和鱼圈管理优化，实现多鱼圈切换和新用户引导的完整闭环。

#### Phase 1.1: 数据库 Schema 升级
*   **模块**: `Prisma Schema` / `Database Migration`
*   **说明**: 
    - Circle 模型新增 `icon`（鱼圈图标）、`coinBalance`（鱼币余额）字段
    - Circle 模型 `petFishExp` 重命名为 `petFishGrowth`
    - 新增 `Decoration` 模型（装饰定义，系统预置）
    - 新增 `CircleDecoration` 模型（鱼圈装饰）
    - 新增 `SignInRecord` 模型（签到记录）
    - 新增 `ExchangeRecord` 模型（兑换记录）
    - User 模型新增 `joinedCircleIds` 多鱼圈关联（替代 `joinedCircleId` 单鱼圈）
    - 删除私有鱼圈相关字段和关系
*   **开发顺序**: 1（所有后续功能的前置条件）
*   **提示词**: `"升级数据库Schema，Circle模型新增icon/coinBalance字段，petFishExp重命名为petFishGrowth，新增Decoration/CircleDecoration/SignInRecord/ExchangeRecord模型，User改为多鱼圈关联，删除私有鱼圈相关字段，执行Prisma迁移"`
*   **详细文档**: `specs/features/V1.1.0/数据库Schema升级_技术方案.md` / `specs/features/V1.1.0/数据库Schema升级_任务规划.md`

#### Phase 1.2: 页面布局重构
*   **模块**: `Common/LeftSidebar` / `Common/MainLayout` / `Common/BottomTabs`
*   **说明**:
    - 重构 `MainLayout`：顶部导航栏 + 左侧鱼圈栏（200px） + 主内容区
    - 新增 `LeftSidebar` 组件：鱼圈列表、[+] 按钮、底部"我的窝囊费"
    - 新增 `BottomTabs` 组件：蛐蛐间 / 摸鱼鱼 Tab 切换
    - 移除顶部导航栏的4个Tab，改为鱼圈名称 + 用户头像
    - 窝囊费作为独立页面从左侧栏底部进入
    - 路由结构调整：从 `/salary`, `/chat`, `/game`, `/circle` 改为 `/circle/:circleId/chat`, `/circle/:circleId/game`, `/salary`
*   **开发顺序**: 2
*   **提示词**: `"重构页面布局，新增左侧常驻鱼圈栏（200px宽）显示鱼圈列表和窝囊费入口，主内容区默认显示聊天室，底部Tab切换蛐蛐间和摸鱼鱼，顶部导航栏改为鱼圈名称+用户头像"`
*   **详细文档**: `specs/features/V1.1.0/页面布局重构_技术方案.md` / `specs/features/V1.1.0/页面布局重构_任务规划.md`

#### Phase 1.3: 鱼圈管理优化
*   **模块**: `Circle/LeftSidebar` / `Circle/CreateJoin` / `Circle/ManagementModal`
*   **说明**:
    - 左侧栏 [+] 按钮展开创建/加入鱼圈表单
    - 创建鱼圈改为实时邀请机制：生成邀请码 → 等待1人加入 → 自动激活（1小时有效期）
    - 鱼圈管理改为独立弹窗（入口在聊天窗口右上角）
    - 多鱼圈支持：用户可加入多个鱼圈，左侧栏显示所有已加入鱼圈
    - 未读消息数角标
*   **开发顺序**: 3
*   **提示词**: `"优化鱼圈管理，左侧栏[+]按钮展开创建/加入表单，创建鱼圈改为实时邀请机制（1小时有效期，至少1人加入），鱼圈管理改为独立弹窗，支持多鱼圈显示和未读消息角标"`
*   **详细文档**: `specs/features/V1.1.0/鱼圈管理优化_技术方案.md` / `specs/features/V1.1.0/鱼圈管理优化_任务规划.md`

#### Phase 1.4: 新用户引导
*   **模块**: `Auth/NewUserGuide`
*   **说明**:
    - 新用户注册成功后显示引导页面
    - 引导选项：加入同事的鱼圈 / 创建新鱼圈 / 暂时跳过
    - 跳过后只能使用窝囊费，其他功能显示"加入鱼圈解锁"
*   **开发顺序**: 4
*   **提示词**: `"开发新用户引导页面，注册成功后显示引导页面，提供加入鱼圈、创建鱼圈、暂时跳过三个选项，跳过后只能使用窝囊费"`
*   **详细文档**: `specs/features/V1.1.0/新用户引导_技术方案.md` / `specs/features/V1.1.0/新用户引导_任务规划.md`

### Milestone 2: 摸鱼鱼交互重构 + 签到装饰（功能完善）

> 目标：完成摸鱼鱼交互优化、宠物鱼成长体系重构、签到与装饰系统。

#### Phase 2.1: 摸鱼鱼交互优化
*   **模块**: `Game/MoyuButton` / `Game/CardDrop` / `Server/GameService`
*   **说明**:
    - 摸鱼交互变更：点击小鱼触发摸鱼动画（1秒，左→右→左）
    - 抽卡概率重新设计：30%不掉卡、30%重复卡、20%不重复普通、15%不重复稀有、5%不重复超稀有
    - 每日上限固定30次（凌晨0点重置，替代原随机15-45次）
    - 成长值获取改为固定1点/次（替代原5×卡片数量）
    - 宠物鱼进化形态调整为4个等级，升级所需成长值10→20→30
*   **开发顺序**: 5
*   **提示词**: `"重构摸鱼鱼交互，点击小鱼触发1秒摸鱼动画，重新设计抽卡概率（30%无卡/30%重复/20%普通/15%稀有/5%超稀有），每日上限固定30次凌晨重置，成长值改为固定1点/次"`
*   **详细文档**: `specs/features/V1.1.0/摸鱼鱼交互优化_技术方案.md` / `specs/features/V1.1.0/摸鱼鱼交互优化_任务规划.md`

#### Phase 2.2: 摸鱼鱼页面布局重构
*   **模块**: `Game/MoyuPage`
*   **说明**:
    - 摸鱼鱼页面改为左右布局：签到日历 + 治愈金鱼池 / 排行榜 + UNO图鉴
    - 签到日历卡片：本周签到日历、签到按钮、鱼币余额
    - 治愈金鱼池：宠物鱼、成长值进度条、今日配额进度条、装扮商城入口
    - 摸鱼排行榜：带头像的排行榜列表（前5名）
    - UNO摸鱼图鉴：收集进度条、卡牌统计
*   **开发顺序**: 6
*   **提示词**: `"重构摸鱼鱼页面为左右布局，第一行签到日历+治愈金鱼池，第二行摸鱼排行榜+UNO摸鱼图鉴，签到日历显示本周记录和鱼币余额，金鱼池展示宠物鱼和成长值"`
*   **详细文档**: `specs/features/V1.1.0/摸鱼鱼页面重构_技术方案.md` / `specs/features/V1.1.0/摸鱼鱼页面重构_任务规划.md`

#### Phase 2.3: 签到与装饰系统
*   **模块**: `Game/SignInCard` / `Game/DecorationShop` / `Server/SignInService` / `Server/DecorationService`
*   **说明**:
    - 每日签到：每个用户每天在每个鱼圈可签到一次，获得1鱼币
    - 鱼币系统：鱼币是鱼圈公共财产，存储在Circle文档

    - 兑换记录：记录鱼币兑换历史
*   **开发顺序**: 7
*   **提示词**: `"开发签到与装饰系统，每日签到获得1鱼币（鱼圈公共财产），装饰商店预置5款装饰（水草/气泡/石头/海星/珊瑚），购买后自动显示在鱼缸，支持兑换记录查询"`
*   **详细文档**: `specs/features/V1.1.0/签到与装饰系统_技术方案.md` / `specs/features/V1.1.0/签到与装饰系统_任务规划.md`

## 4. 关键技术决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 多鱼圈关联方式 | User 模型新增 `CircleMember` 关联表 | 支持一个用户加入多个鱼圈，保留加入时间等元数据 |
| 私有鱼圈处理 | 删除 `isPrivate` 字段和相关逻辑 | V1.1.0不再区分公私鱼圈 |
| 鱼圈切换方式 | URL 参数 `circleId` + 左侧栏点击 | 利用路由系统管理当前鱼圈上下文 |
| 未读消息实现 | Circle 模型记录 `lastMessageAt`，客户端对比 | 轻量级方案，避免额外的未读消息表 |
| 装饰数据初始化 | Prisma seed 脚本预置 | 系统数据，不需要API管理 |
| 实时邀请机制 | Socket.io 监听加入事件 + 定时器清理 | 利用现有Socket基础设施 |
| 鱼币存储 | Circle 模型 `coinBalance` 字段 | 鱼币是鱼圈公共财产，直接存在鱼圈文档 |
| 成长值重命名 | `petFishExp` → `petFishGrowth` + 数据迁移 | 语义更清晰，兼容现有数据 |
