# 鱼圈管理优化 — 邀请等待状态展示 PRD

## 1. 背景与目标

### 为什么做

当前创建鱼圈的流程是：用户点击创建 → 生成邀请码 → 显示弹窗 → 关闭弹窗。问题在于，关闭弹窗后用户就找不到自己创建的这个"等待中的群"了。用户需要重新打开弹窗才能看到邀请码，但没有直观的入口知道"我有一个群正在等人加入"。

### 用户价值

*   创建鱼圈后，侧边栏持续展示"等待中"的鱼圈，不再丢失邀请状态
*   实时倒计时提醒邀请码有效期，避免过期后才发现没人加入
*   一键复制邀请码，方便分享给同事
*   鱼圈激活后自动变为正常鱼圈，无需额外操作

### 成功标准

*   创建鱼圈后，用户能在侧边栏看到"等待中"的鱼圈
*   邀请码过期后，鱼圈自动从侧边栏消失
*   第2人加入后，鱼圈自动激活，侧边栏样式自动切换
*   创建鱼圈到侧边栏显示"等待中"状态的延迟 < 1秒

---

## 2. 范围定义

### 本次做

*   侧边栏展示"等待中"的鱼圈（与已激活鱼圈视觉区分）
*   显示过期倒计时（1小时有效期，每秒更新）
*   显示已邀请人数（"X/1 人加入"）
*   显示状态标签（"⏳ 等待加入"）
*   点击"等待中"鱼圈打开邀请弹窗（查看/复制邀请码）
*   邀请码过期后鱼圈从侧边栏自动消失
*   第2人加入后鱼圈自动激活，样式变为正常鱼圈
*   每个用户同时最多3个"等待中"的鱼圈
*   倒计时<10分钟时红色警示

### 本次不做

*   邀请码刷新/重新生成功能（过期后需重新创建鱼圈）
*   邀请链接分享（仅支持邀请码）
*   批量邀请功能
*   邀请记录查看（谁通过邀请码加入了）

## 3. 用户与场景

### 用户角色

| 角色 | 说明 |
| ---- | ---- |
| 群主（创建者） | 创建鱼圈后需要等待至少1人加入才能激活，是本功能的核心用户 |
| 被邀请者 | 通过邀请码加入鱼圈的用户，不是本功能的直接使用者 |

### 用户故事

| 编号 | 用户故事 |
| ---- | ---- |
| US-001 | 作为群主，我想在侧边栏看到"等待中"的鱼圈，以便知道有一个群正在等人加入 |
| US-002 | 作为群主，我想看到邀请码的过期倒计时，以便提醒同事尽快加入 |
| US-003 | 作为群主，我想点击"等待中"的鱼圈打开邀请弹窗，以便查看和复制邀请码 |
| US-004 | 作为群主，我想在第2人加入后鱼圈自动激活，以便立即开始使用 |
| US-005 | 作为群主，我想在邀请码过期后鱼圈自动消失，以便侧边栏保持整洁 |

## 4. 页面结构

### 4.1 侧边栏整体布局

```
左侧鱼圈栏 (200px)
│
├── ① 操作区 (顶部)
│   └── [+] 按钮
│
├── ② 已激活鱼圈列表
│   ├── 🐟 技术部摸鱼会 (高亮/未读角标)
│   └── 🐠 市场部早退党
│
├── ────────── 分隔线
│
├── ③ "等待中"折叠卡片 (默认折叠，最多3个)
│   └── [⏳ 等待加入  (3) ▶]  ← 折叠状态：一行标题 + 数量徽章
│       ├── 展开后：
│       │   ├── 🐟 躺平平  59:58  · 1/1人
│       │   ├── 🐠 产品组  59:34  · 1/1人
│       │   └── 🐟 小小产品  58:00  · 1/1人
│       └── 点击标题展开/收起
│
└── ④ 个人功能区 (底部)
    └── 💰 我的窝囊费
```

> 注意：未激活（等待中）的鱼圈不会出现在 ② 已激活鱼圈列表中，只显示在 ③ 折叠卡片内。

### 4.2 页面入口

| 功能模块 | 入口位置 | 交互方式 |
| ---- | ---- | ---- |
| "等待中"鱼圈展示 | 已激活鱼圈列表下方 → 折叠卡片 | 默认折叠，点击展开查看列表 |
| 邀请弹窗 | 展开后点击某个"等待中"鱼圈 | 打开弹窗，显示邀请码和倒计时 |
| 邀请码复制 | 邀请弹窗内 | 点击复制按钮 |

### 4.3 折叠卡片交互

| 状态 | 表现 |
| ---- | ---- |
| 无等待中鱼圈 | 不渲染任何内容 |
| 折叠状态（默认） | 一行标题：`⏳ 等待加入  [数量徽章]  ▶`，约 40px 高 |
| 展开状态 | 标题下方展开列表，每行显示：图标 + 名称 + 倒计时 + 人数 |
| 数量徽章 | 黄色圆角标签，显示等待中鱼圈数量 |

## 5. 功能详细设计

### 5.1 侧边栏"等待中"鱼圈展示

#### 功能说明

功能目标：在左侧鱼圈栏中持续展示"等待中"的鱼圈，让群主随时掌握邀请状态

#### 页面设计

功能入口：**已激活鱼圈列表下方的折叠卡片**（默认折叠，不占空间）

页面组成：

*   折叠卡片头部：标题"⏳ 等待加入" + 数量徽章 + 展开箭头
*   展开后显示等待中鱼圈列表，每行紧凑展示：图标、名称、倒计时、人数
*   位于已激活鱼圈列表下方、个人功能区上方，以分隔线与已激活鱼圈区分
*   无等待中鱼圈时不渲染任何内容

#### 视觉设计

**折叠卡片头部（默认状态）**：

| 元素 | 说明 |
| ---- | ---- |
| 分隔线 | 上方有 `border-t-2 border-ink` 与已激活鱼圈列表分隔 |
| 标题 | "⏳ 等待加入"，text-xs 字号 |
| 数量徽章 | 黄色圆角标签，显示等待中鱼圈数量 |
| 展开箭头 | ▶ 图标，展开时旋转 90° |
| 整体高度 | 约 40px，不展开时极省空间 |

**展开后的每行鱼圈**：

| 元素 | 说明 |
| ---- | ---- |
| 图标 | text-lg，半透明 |
| 名称 | text-xs，单行截断 |
| 倒计时 | 等宽数字字体，每秒更新 |
| 人数 | "X/1人"，紧凑显示 |
| 整体样式 | opacity: 0.7，表示"未激活"状态 |
| 行高 | py-2（比已激活鱼圈更紧凑） |

#### 交互设计

| 操作 | 反馈 |
| ---- | ---- |
| 创建鱼圈成功 | 侧边栏折叠卡片立即出现，数量徽章 +1 |
| 点击折叠卡片标题 | 展开/收起等待中鱼圈列表 |
| 展开后点击某个"等待中"鱼圈 | 打开邀请弹窗 |
| 邀请弹窗关闭 | 返回侧边栏，折叠卡片仍显示 |
| 倒计时归零 | 对应鱼圈从展开列表消失，数量徽章 -1 |
| 第2人加入成功 | 对应鱼圈从折叠卡片消失，出现在已激活鱼圈列表 |
| 同时有多个"等待中"鱼圈 | 按创建时间倒序排列（最新在最上面） |
| 无等待中鱼圈 | 折叠卡片不渲染 |

#### 排序规则

| 编号 | 规则 |
| ---- | ---- |
| BR-001 | "等待中"鱼圈按创建时间倒序排列（最新的在最上面） |
| BR-002 | "等待中"鱼圈始终显示在已激活鱼圈列表下方，以折叠卡片形式呈现 |

### 5.2 邀请弹窗

#### 功能说明

功能目标：点击"等待中"鱼圈后打开的弹窗，展示邀请码和相关信息

#### 页面设计

功能入口：**点击侧边栏"等待中"鱼圈**

页面组成：

*   标题：鱼圈名称
*   6位邀请码（大号显示，等宽数字字体）
*   过期倒计时（实时更新，格式"剩余 XX:XX:XX"）
*   已邀请人数："X/1 人加入"
*   复制邀请码按钮
*   关闭按钮（右上角）

#### 交互设计

| 操作 | 反馈 |
| ---- | ---- |
| 点击复制按钮 | 邀请码复制到剪贴板，显示"📋 邀请码已复制！" |
| 倒计时<10分钟 | 倒计时文字变为红色 |
| 倒计时归零 | 弹窗自动关闭，提示"邀请已过期，鱼圈已自动删除" |
| 关闭弹窗 | 返回侧边栏，"等待中"鱼圈仍显示（如果未过期） |

---

### 5.3 过期与激活机制

#### 功能说明

功能目标：管理邀请码的生命周期，包括过期清理和自动激活

#### 邀请码过期

| 编号 | 规则 |
| ---- | ---- |
| BR-003 | 邀请码有效期1小时（从创建时开始计算） |
| BR-004 | 过期后鱼圈从侧边栏自动消失 |
| BR-005 | 过期后鱼圈自动删除（数据库清理） |
| BR-006 | 如用户正在查看邀请弹窗，过期时弹窗自动关闭并提示 |

#### 自动激活

| 编号 | 规则 |
| ---- | ---- |
| BR-007 | 至少1人加入（总人数达到2人）后鱼圈自动激活 |
| BR-008 | 激活后"等待中"样式立即变为正常鱼圈样式 |
| BR-009 | 激活后倒计时消失，鱼圈可正常使用 |

#### 数量限制

| 编号 | 规则 |
| ---- | ---- |
| BR-010 | 每个用户同时最多有3个"等待中"的鱼圈 |
| BR-011 | 达到3个上限时，创建鱼圈操作被阻止 |
| BR-012 | 提示文案："你已经有3个等待中的鱼圈了，等它们过期或有人加入后再创建吧！" |
| BR-013 | 折叠卡片默认为折叠状态，不占用额外垂直空间 |

## 6. 验收标准

### Happy Path

*   AC-001: Given 用户创建了一个新鱼圈，When 创建成功后查看侧边栏，Then 侧边栏折叠卡片出现，数量徽章显示正确数量，展开后可看到该鱼圈的名称、倒计时、人数
*   AC-002: Given 用户点击折叠卡片标题展开，Then 看到等待中鱼圈列表，每行显示图标、名称、倒计时、人数
*   AC-003: Given 用户展开折叠卡片后点击某个"等待中"鱼圈，When 邀请弹窗打开，Then 显示鱼圈名称、6位邀请码（大号）、过期倒计时、"0/1 人加入"、复制按钮
*   AC-004: Given 用户在邀请弹窗中点击复制按钮，When 复制成功，Then 邀请码复制到剪贴板，显示"📋 邀请码已复制！"提示
*   AC-005: Given 第2人通过邀请码加入"等待中"鱼圈，When 加入成功，Then 鱼圈从折叠卡片消失，出现在已激活鱼圈列表中，倒计时消失
*   AC-006: Given 用户关闭邀请弹窗，When 返回侧边栏，Then 折叠卡片仍显示（如果还有等待中的鱼圈）

### Edge Case

*   AC-101: Given 邀请码倒计时归零，When 系统检查过期，Then 对应鱼圈从折叠卡片消失（数量徽章 -1），如用户正在查看弹窗则关闭弹窗并提示"邀请已过期，鱼圈已自动删除"
*   AC-102: Given 用户已有3个"等待中"的鱼圈，When 尝试创建第4个鱼圈，Then 创建被阻止，提示"你已经有3个等待中的鱼圈了，等它们过期或有人加入后再创建吧！"
*   AC-103: Given 展开折叠卡片后，某个"等待中"鱼圈倒计时<10分钟，When 用户查看，Then 倒计时文字变为红色警示
*   AC-104: Given 用户创建了多个"等待中"的鱼圈，When 展开折叠卡片，Then 鱼圈按创建时间倒序排列（最新的在最上面）
*   AC-105: Given "等待中"鱼圈的邀请码已过期，When 用户尝试通过该邀请码加入，Then 提示"邀请码已过期，请联系群主重新生成"

### Business Rule

*   AC-201: Given 任何情况下，When 邀请码有效期计算，Then 有效期为1小时（从创建时开始）
*   AC-202: Given 任何情况下，When 检查"等待中"鱼圈数量上限，Then 每个用户同时最多3个
*   AC-203: Given 任何情况下，When 鱼圈激活条件判断，Then 至少需要2人（创建者+1人）才能激活
*   AC-204: Given 任何情况下，When "等待中"鱼圈排序，Then 按创建时间倒序排列
*   AC-205: Given 任何情况下，When 倒计时显示，Then 使用等宽数字字体，每秒更新
