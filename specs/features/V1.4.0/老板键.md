# 老板键

> 版本：V1.4.0 | 来源：feedback-0626 | 推翻重做（替换 V1.0.0 原有老板键设计）

## 1. 背景与目标

### 为什么做
"摸鱼"行为的核心风险是被领导或同事发现。老板键提供一键切换伪装界面的能力，让用户在他人接近时瞬间隐藏摸鱼内容。新增导航栏按钮入口、两个可切换的虚拟页面、默认页面设置等能力。

### 用户价值
- 按一下 Esc 瞬间进入伪装页面，操作零延迟
- 两个虚拟页面（产品验收记录表、AI 知识学习页面）均可交互，伪装更真实
- 页面标题和浏览器图标同步切换，增强伪装可信度
- 可设置默认伪装页面，贴合个人工作场景
- 连续快速两下 Esc 退出伪装，方便快捷

## 2. 范围定义

### 本次做
- 顶部导航栏新增老板键按钮
- Esc 键全局监听：按一下进入老板模式
- 连续快速两下 Esc 退出老板模式
- 两个虚拟页面：产品验收记录表 + AI 知识学习页面
- 两个页面可互相切换
- 页面标题切换（真实 ↔ "工作周报 - 在线文档"）
- 浏览器图标（Favicon）切换（🐟 ↔ 📄）
- 默认老板模式页面设置（头像下拉菜单入口）
- 退出后所有状态保留
- 进入时关闭所有打开的弹窗/侧边栏
- 虚拟页面前端假数据，可滚动、可交互

### 本次不做
- 防误触机制（不需要，按一下进入/快速两下退出的设计已避免误触）
- 在线表格伪装页面（替换为新的两个虚拟页面）
- 伪装界面数据持久化到后端
- 手机端老板键
- 自定义伪装内容
- 自定义页面标题或图标

## 3. 用户与场景

### 触发方式

| 方式 | 行为 | 说明 |
|------|------|------|
| 导航栏按钮 | 点击进入 / 点击退出 | 顶部导航栏常驻按钮 |
| Esc 键 | 按一下进入 | 全局监听，页面获得焦点时有效 |
| Esc 键 | 连续快速两下退出 | 两次按键间隔 < 1500ms |
| 页面按钮 | 点击退出 | 虚拟页面上的"退出老板模式"按钮 |

### 用户故事

| 编号 | 用户故事 |
| ---- | ---- |
| US-001 | 作为用户，我想按一下 Esc 快速进入伪装页面，以便领导走近时瞬间隐藏 |
| US-002 | 作为用户，我想在伪装页面之间切换，以便应对不同的窥视场景 |
| US-003 | 作为用户，我想连续快速两下 Esc 或点击按钮退出伪装，以便快速恢复使用 |
| US-004 | 作为用户，我想设置默认伪装页面，以便每次进入时直接显示最合适的伪装内容 |
| US-005 | 作为用户，我希望退出后所有进度和草稿都保留，以便无缝继续摸鱼 |

### 4.1 老板模式整体布局

进入老板模式后，页面全屏覆盖为伪装界面，包含：

- **顶部标签栏**：两个页面标签可切换（"产品验收记录表" / "AI 知识学习"）
- **主内容区**：当前选中页面的内容
- **右下角**："退出老板模式"按钮（不显眼但可找到）

## 5. 交互设计

### 5.1 进入老板模式

| 操作 | 反馈 |
| ---- | ---- |
| 按一下 Esc | 立即切换到默认伪装页面，关闭所有弹窗/侧边栏 |
| 点击导航栏老板键按钮 | 同上 |

### 5.2 老板模式内交互

| 操作 | 反馈 |
| ---- | ---- |
| 点击顶部标签切换页面 | 切换至对应虚拟页面，内容即时显示 |
| 在验收表中点击行 | 行高亮选中，可查看详情 |
| 在 AI 学习页面点击课程卡片 | 卡片展开，显示虚假课程详情 |
| 滚动页面 | 正常滚动 |

### 5.3 退出老板模式

| 操作 | 反馈 |
| ---- | ---- |
| 连续快速两下 Esc（间隔 < 1500ms） | 退出伪装，恢复进入前的真实界面 |
| 点击右下角"退出老板模式"按钮 | 同上 |
| 点击导航栏老板键按钮 | 同上 |

### 5.4 状态切换

| 状态项 | 真实界面 | 老板模式 |
| ---- | ---- | ---- |
| 页面标题 | 吐司 - {当前页面} | 工作周报 - 在线文档 |
| 浏览器图标 | 🐟 | 📄 |
| 二级弹窗 | 进入时全部关闭 | 无 |

## 6. 验收标准

### Happy Path

- **AC-001**: Given 用户在应用主界面，When 按下一次 Esc 键或点击导航栏老板键按钮，Then 立即跳转到老板模式默认虚拟页面，页面标题变为"工作周报 - 在线文档"
- **AC-002**: Given 用户在老板模式下，When 连续快速按下两次 Esc 键（间隔 < 1500ms），Then 退出老板模式，恢复进入前的真实界面，所有状态保留
- **AC-003**: Given 用户在老板模式下，When 点击右下角"退出老板模式"按钮或导航栏老板键按钮，Then 退出老板模式，恢复真实界面
- **AC-004**: Given 用户在老板模式的"产品验收记录表"页面，When 点击顶部标签切换到"AI知识学习"，Then 显示 AI 知识学习页面的完整内容
- **AC-005**: Given 用户在头像下拉菜单中，When 选择"老板模式默认页面"并设为"AI知识学习页面"后保存，Then 后续触发老板模式直接进入 AI 知识学习页面
- **AC-006**: Given 新用户从未设置过默认老板模式页面，When 首次触发老板模式，Then 默认进入"产品验收记录表"

### Edge Cases

- **AC-007**: Given 用户真实界面有弹窗打开（如卡片掉落弹窗、鱼圈管理弹窗），When 触发老板模式，Then 弹窗关闭，切换至虚拟页面
- **AC-008**: Given 用户在摸鱼动画播放过程中，When 触发老板模式，Then 动画中断，切换至虚拟页面；退出后摸鱼状态恢复至触发前的状态
- **AC-009**: Given 用户只按了一下 Esc（未快速连按），When 在老板模式下，Then 不退出老板模式（必须快速连按两下才退出）
- **AC-010**: Given 用户在老板模式下，When 刷新浏览器页面，Then 页面恢复为真实界面（老板模式不跨会话保留）

### Business Rules

- **AC-011**: Given 老板模式的两个虚拟页面，When 用户浏览和交互，Then 所有数据均为前端假数据，用户的编辑操作仅当次会话有效，退出或刷新后不保留
- **AC-012**: Given 用户在老板模式下，When 查看浏览器标签页标题，Then 显示"工作周报 - 在线文档"；退出后恢复原标题
- **AC-013**: Given 用户在老板模式下，When 查看浏览器标签页图标，Then 显示 📄；退出后恢复原图标 🐟
