# 蛐蛐间变更

> 版本：V1.4.0 | 来源：feedback-0626

## 1. 背景与目标

### 为什么做
当前左侧鱼圈栏缺少未读消息提示，用户无法快速识别哪个鱼圈有新消息；消息列表未按活跃度排序，不够直观。聊天窗口顶部文案可优化为更友好的引导语。

### 用户价值
- 快速定位有未读消息的鱼圈，不错过同事的吐槽
- 最活跃的鱼圈排在最上面，减少切换成本
- 聊天窗口顶部文案更有温度

---

## 2. 范围定义

### 本次做
- 左侧鱼圈栏新增未读消息红点提示
- 鱼圈列表按最新消息发送时间倒序排列
- 聊天窗口顶部文案替换为新文案
- 去掉"连接中..."状态提示文字

### 本次不做
- 未读消息数字角标
- 手动拖拽排序鱼圈
- 聊天消息本身的展示规则变更
- 修改其他状态提示文案

---

## 3. 功能详细说明

### 3.1 消息列表未读提示与排序

#### 未读标记规则
- 鱼圈中有用户未查看的新消息时，左侧栏该鱼圈显示红点
- 用户点击进入该鱼圈后，红点消失
- 用户当前正在查看的鱼圈收到新消息不显示红点

#### 排序规则
- 按每个鱼圈最新一条消息的发送时间倒序排列（最新有消息的排最上面）
- 没有消息记录的鱼圈排在最后，之间按加入时间排序

### 3.2 聊天窗口文案优化

#### 新文案
> 有什么想吐槽的大胆吐出来吧！5MIN瞬时阅后即焚！请文明用语。

#### 保留与移除
- 鱼圈名称**保留**显示
- "连接中..."状态提示**移除**，不再显示

---

## 4. 验收标准

### Happy Path

- **AC-001**: Given 用户加入多个鱼圈且某个鱼圈有新消息而用户不在其中，When 用户查看左侧栏，Then 该鱼圈显示红点未读标记
- **AC-002**: Given 用户在左侧栏点击有未读标记的鱼圈，When 进入该鱼圈，Then 未读标记消失
- **AC-003**: Given 用户查看左侧鱼圈列表，When 列表渲染完成，Then 鱼圈按最新消息发送时间倒序排列（最新有消息的排在最上面）
- **AC-004**: Given 用户进入某个鱼圈的蛐蛐间，When 查看聊天窗口顶部区域，Then 显示"有什么想吐槽的大胆吐出来吧！5MIN瞬时阅后即焚！请文明用语。"

### Edge Cases

- **AC-005**: Given 用户当前正在查看某个鱼圈，When 该鱼圈收到新消息，Then 左侧栏该鱼圈不显示未读标记（用户正在查看中）
- **AC-006**: Given 用户进入蛐蛐间，When 查看聊天窗口顶部状态区域，Then 不显示"连接中..."等连接状态文字

### Business Rules

- **AC-007**: Given 鱼圈列表按最新消息时间倒序排列，When 存在没有消息记录的鱼圈，Then 这些鱼圈排在列表最后，彼此之间按加入时间排序
- **AC-008**: Given 用户进入蛐蛐间，When 查看聊天窗口，Then 当前鱼圈名称仍然正常显示在窗口中
