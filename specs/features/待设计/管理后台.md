# 管理后台（Admin Dashboard）PRD

## 1. 背景与目标

### 为什么做

摸鱼圈作为一个社交平台，需要管理员对用户和内容进行监管。管理后台提供用户管理、鱼圈管理、数据统计和系统配置等功能，确保平台健康运行，及时处理违规行为。

### 用户价值

*   管理员可查看所有用户和鱼圈的状态
*   可快速禁用违规用户、解散违规鱼圈
*   数据统计面板帮助了解平台运营状况
*   游戏参数可在线调整，无需重新部署

### 成功标准

*   管理员可正常访问后台
*   用户禁用/启用操作成功率 100%
*   鱼圈解散操作成功率 100%
*   数据统计实时更新

---

## 2. 范围定义

### 本次做

*   管理员身份验证（邮箱白名单）
*   用户管理：查看用户列表、禁用/启用用户
*   鱼圈管理：查看所有鱼圈、强制解散违规鱼圈
*   数据统计：用户总数、鱼圈总数、每周摸鱼趋势图
*   系统配置：UNO卡片掉率倍数、每日摸鱼次数上限

### 本次不做

*   管理员角色分级（超级管理员、普通管理员）
*   内容审核（聊天消息审查）
*   用户数据导出
*   操作日志审计
*   通知推送系统
*   数据备份/恢复

### 5.3 用户管理

#### 功能说明

功能目标：查看所有注册用户，支持禁用/启用操作

#### 页面设计

功能入口：管理后台底部

页面组成：

*   标题："全天候注册雇员表 ({用户数} 个账号)"
*   用户表格：
    *   雇员昵称（含头像emoji）
    *   Email
    *   基础薪资
    *   打卡圈（当前鱼圈ID）
    *   状态（正常/禁闭中）
    *   快捷拉黑按钮

#### 用户状态

| 状态 | 标签 | 颜色 |
| ---- | ---- | ---- |
| 正常 | ✅ 正常 | 绿色 |
| 已禁用 | 🚫 禁闭中 | 红色 |

#### 交互设计

| 操作 | 反馈 |
| ---- | ---- |
| 点击"封禁"按钮 | 弹出确认框："是否确定要【禁用/拉黑】用户 [{昵称}]？" |
| 确认禁用 | 用户状态变为"禁闭中"，按钮变为"重放" |
| 点击"重放"按钮 | 弹出确认框："是否确定要【解除禁用】用户 [{昵称}]？" |
| 确认解除 | 用户状态变为"正常"，按钮变为"封禁" |
| 点击"刷新数据源" | 重新加载用户列表 |

#### 业务规则

| 编号 | 规则 |
| ---- | ---- |
| BR-006 | 管理员不能禁用自己（按钮置灰保护） |
| BR-007 | 禁用用户后，该用户登录时显示封禁提示页 |
| BR-008 | 用户列表显示所有注册用户 |

#### 异常处理

| 场景 | 处理方式 |
| ---- | ---- |
| 操作失败 | 控制台输出错误，显示错误提示 |
| 数据加载失败 | 控制台输出错误 |

### 5.5 系统配置

#### 功能说明

功能目标：在线调整游戏参数，无需重新部署

#### 页面设计

功能入口：管理后台中部左侧

页面组成：

*   标题："全网游戏配置台 🎮"
*   副标题："在线调整UNO掉落权重系数与摸鱼防沉迷"
*   配置项：
    *   UNO随机稀有卡掉落率倍数（滑块，0.5x ~ 3.0x）
    *   限制单天点击安全额区间上限（滑块，20 ~ 100次）
*   保存按钮："同步配置至服务器"

#### 配置参数

| 参数 | 类型 | 范围 | 默认值 | 步长 | 说明 |
| ---- | ---- | ---- | ---- | ---- | ---- |
| cardDropRate | number | 0.5 ~ 3.0 | 1.0 | 0.1 | UNO稀有卡掉落率倍数 |
| maxMoyuLimit | number | 20 ~ 100 | 40 | 5 | 每日摸鱼次数上限 |

#### 交互设计

| 操作 | 反馈 |
| ---- | ---- |
| 拖动滑块 | 实时显示当前值 |
| 点击"同步配置至服务器" | 显示"正在应用缓存..."，保存成功后提示"摸鱼游戏控制参数调整保存成功！" |

#### 业务规则

| 编号 | 规则 |
| ---- | ---- |
| BR-011 | 配置修改后立即生效 |
| BR-012 | 配置保存到服务器（当前为本地模拟） |

#### 异常处理

| 场景 | 处理方式 |
| ---- | ---- |
| 保存失败 | 控制台输出错误 |

## 6. 验收标准

### Happy Path

*   AC-001

    *   Given 管理员登录
    *   When 查看导航栏
    *   Then 显示紫色"监控后台"Tab

*   AC-002

    *   Given 管理员在管理后台
    *   When 点击"刷新数据源"
    *   Then 用户列表和鱼圈列表重新加载

*   AC-003

    *   Given 管理员在用户管理
    *   When 点击某用户的"封禁"按钮并确认
    *   Then 用户状态变为"禁闭中"

*   AC-004

    *   Given 管理员在用户管理
    *   When 点击已禁用用户的"重放"按钮并确认
    *   Then 用户状态变为"正常"

*   AC-005

    *   Given 管理员在鱼圈管理
    *   When 点击某鱼圈的解散按钮并确认
    *   Then 鱼圈从列表移除

*   AC-006

    *   Given 管理员在系统配置
    *   When 调整参数并点击"同步配置至服务器"
    *   Then 配置保存成功，显示成功提示

### Edge Case

*   AC-101

    *   Given 普通用户登录
    *   When 查看导航栏
    *   Then 不显示"监控后台"Tab

*   AC-102

    *   Given 管理员在用户管理
    *   When 尝试禁用自己
    *   Then 按钮置灰，显示"置灰(保护)"

*   AC-103

    *   Given 管理员在鱼圈管理
    *   When 尝试解散私有鱼圈
    *   Then 显示"抱歉，此为保留的个人专属水池，不可解散毁灭！"

### Business Rule

*   AC-201

    *   Given 管理员禁用用户
    *   When 被禁用用户尝试登录
    *   Then 登录成功但显示封禁提示页

*   AC-202

    *   Given 管理员解散鱼圈
    *   When 鱼圈内成员重新登录
    *   Then 成员自动回到私有鱼圈

*   AC-203

    *   Given 管理员调整摸鱼次数上限为50
    *   When 用户摸鱼
    *   Then 用户每日摸鱼上限变为50次
