## 6. 关键决策

### 6.1 卡片归属

**决策**：卡片为鱼圈共享资产（非用户私有）
**理由**：

1. 现有数据模型 `@@unique([circleId, cardId])` 已支持
2. 与其他资产（鱼币、宠物鱼、装饰）保持一致
3. 逻辑更简单，避免"用户退出后卡片归属"的复杂问题

### 6.2 级联删除策略

**决策**：使用数据库级联删除（`ON DELETE CASCADE`）
**理由**：

1. 确保数据一致性，避免孤立记录
2. 减少应用层代码复杂度
3. 性能更好，单次数据库操作
